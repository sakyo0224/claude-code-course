# 仮想マシンを使わないメリット

**Claude Code（VMなし）**
- ファイルに **直接アクセス** → 動作が速い
- テキストベース処理 → トークン消費が少ない
- ローカルツールとの連携が自在
- PCのリソースを無駄遣いしない

**Cowork（VMあり）**
- スクリーンショット → 画像認識 → 操作 の繰り返しで遅い
- 30分以上のタスクで速度低下・クラッシュの報告あり
- PCがスリープすると動作が止まる

[図：Claude Code=直線矢印（高速）、Cowork=迂回矢印（低速）の図解]

# settings.json ── 許可と禁止の設定

```json
{
  "permissions": {
    "allow": ["Read", "Write", "Edit"],
    "deny": ["Bash(rm *)"]
  }
}
```

| 設定 | 意味 |
|---|---|
| allow | 自動で許可する操作 |
| deny | 絶対に実行させない操作 |

> 「ファイルの読み書きはOK、削除はNG」のように制御できます

# パーミッションモード一覧

| モード | 動作 | おすすめ対象 |
