# 調査レポート：Claude Cowork vs Claude Code / 機能一覧 / Pro契約手順

調査日：2026年4月6日

---

## 1. Claude Cowork vs Claude Code 詳細比較

### 1-1. 基本的な位置づけ

| 項目 | Claude Code | Claude Cowork |
|------|-------------|---------------|
| 対象ユーザー | ターミナル操作に慣れた開発者 | 非技術者を含むすべてのユーザー |
| インターフェース | CLI（コマンドライン）/ デスクトップアプリのCodeタブ | デスクトップアプリのCoworkタブ |
| 主な用途 | コーディング、リポジトリ操作、開発作業全般 | リサーチ、ドキュメント編集、スプレッドシート作業、デスクトップ操作 |
| 実行環境 | ローカルマシンに直接アクセス | サンドボックス化された仮想マシン（VM）内で一部実行 |
| 提供状態 | 正式リリース済み | **リサーチプレビュー（研究段階）** |

### 1-2. 仮想マシン（VM）を使うCoworkのデメリット

Coworkは、コード実行をサンドボックス化された隔離環境（VM）で行いますが、ファイルへの実際の変更はユーザーのデスクトップに直接反映されるという仕組みです。

**VMによる制限・デメリット：**

- **トークン消費が非常に大きい**：Coworkは自律的に計画→確認→再計画を繰り返すため、「フォルダを整理して」のような簡単な指示でも数分で数千トークンを消費する。スクリーンショットの画像処理（1000x1000ピクセルで約1,334トークン）が頻繁に発生し、消費量がさらに増加する
- **操作の遅さ**：スクリーンショットの取得→画像認識→操作指示という手順を繰り返すため、直接ファイル操作するClaude Codeよりも遅い
- **長時間タスクの不安定性**：30分以上のタスクで速度低下、一時停止、クラッシュが報告されている
- **環境の制約**：デスクトップアプリが開いている必要があり、パソコンがスリープすると動作が止まる
- **セッション間のメモリ非保持**：スタンドアロンセッション間ではメモリ（文脈情報）が引き継がれない
- **セッション共有不可**：他のユーザーとセッションを共有できない
- **モバイル/Web非対応**：セッションはデスクトップアプリ上のみで、Web版やモバイルには同期されない

**VMを使わないClaude Codeのメリット：**

- **直接ファイルアクセス**：ファイルシステムに直接読み書きするため高速
- **トークン効率が良い**：テキストベースの操作のため、画像処理によるトークン消費が発生しない
- **リソース効率**：VM起動・管理のオーバーヘッドがない
- **セッション継続性**：リモートセッションを使えばパソコンを閉じても作業が続く
- **複数セッション管理**：Git worktreeで並列作業が可能
- **開発者向け最適化**：Bash、ファイル編集、Git操作、PR管理が直接統合

### 1-3. Windowsでの安定性

**Claude Code デスクトップアプリのWindows対応（2026年4月3日追加）：**

- 2026年4月3日にComputer Use（コンピュータ操作）機能がWindowsに対応
- Pro/Maxプラン契約者が利用可能
- ブラウザ操作、マウス/キーボード制御、画面操作が可能
- Windows固有の修正：PowerShellツールのダブルクォート・空白文字処理の改善
- Windows版ではAccessibility/Screen Recording権限の設定が不要（macOSでは必要）
- Gitのインストールが必須（macOSはデフォルトで入っている）

**Cowork のWindows対応：**

- 同じく2026年4月3日にWindows対応を追加
- macOS版と同等の機能が利用可能

**Windows環境での注意点：**

- まだリサーチプレビュー段階であり、一般提供の正式なタイムラインは未発表
- Anthropicは「機密データを扱うアプリケーションと併用しないこと」を推奨
- macOS版に比べて出たばかりのため、不具合が出る可能性はやや高い

### 1-4. トークン消費効率の比較

| 項目 | Claude Code | Cowork |
|------|-------------|--------|
| 平均消費（開発者1日あたり） | 約6ドル（90%のユーザーは12ドル以下） | タスクにより大きく変動（数倍になることも） |
| スクリーンショット処理 | 基本なし（Computer Use有効時を除く） | 頻繁に発生（1枚あたり約1,334トークン） |
| 節約のコツ | 具体的な指示を出す、/compactで文脈圧縮 | 関連タスクをまとめて1つの指示にする |

### 1-5. セキュリティモデルの違い

| 項目 | Claude Code | Cowork |
|------|-------------|--------|
| パーミッション制御 | 5段階のモード（後述）で細かく制御可能 | ファイルアクセス許可＋アプリ許可 |
| サンドボックス | Bashツールにサンドボックスあり | コード実行はVM内で隔離 |
| 監査ログ | Team/Enterpriseで対応 | **監査ログ非対応（Audit Logs、Compliance API、Data Exportsに記録されない）** |
| プロンプトインジェクション対策 | 多層防御あり | 対策あり（ただしインターネットアクセスによるリスクが高い） |
| 破壊的操作 | 許可制（パーミッションモードで制御） | 破壊的操作（ファイル削除など）の可能性あり |

### 1-6. できることの範囲

| 機能 | Claude Code | Cowork |
|------|-------------|--------|
| コード編集・開発 | 得意 | 基本的な開発は可能 |
| ターミナルコマンド実行 | 得意 | 直接は不可 |
| Git/GitHub操作 | 得意 | 限定的 |
| ドキュメント作成・編集 | 可能 | 得意 |
| スプレッドシート操作 | 不可（MCP連携で一部可能） | 得意 |
| Webブラウザ操作 | Chrome拡張で可能 | 画面操作で可能 |
| デスクトップアプリ操作 | Computer Use有効時のみ | 得意 |
| メール/カレンダー操作 | コネクタ連携で可能 | 画面操作で可能 |
| リサーチ・調査 | 可能 | 得意 |
| スケジュール実行 | クラウドで定期実行可能 | 不可 |
| リモート実行 | パソコンを閉じても継続可能 | パソコンが起動している必要あり |

### 1-7. Coworkのリサーチプレビュー状態のリスク

- **安定性**：予告なく機能変更・仕様変更が行われる可能性がある
- **規制対応不可**：監査ログ非対応のため、規制対象のワークロードには使用不可
- **データ事故のリスク**：「テスト中に11GBのファイルを誤って消費した」というユーザー報告あり。バックアップの作成が強く推奨される
- **破壊的操作への注意**：Claudeが指示を誤解し、ファイルを削除する可能性がゼロではない
- **セキュリティ**：インターネットアクセスによるプロンプトインジェクションのリスクが高い
- **サポート体制**：正式リリース前のため、サポートが限定的な可能性がある

## 2. Claude Code デスクトップアプリ 全機能リスト

### 2-1. ファイル操作系

| 機能 | 説明 |
|------|------|
| ファイル読み取り（Read） | 任意のファイルを読み込み、内容を理解 |
| ファイル書き込み（Write） | 新しいファイルを作成 |
| ファイル編集（Edit） | 既存ファイルの一部を差し替え修正 |
| ファイル検索（Glob/Grep） | パターンマッチや正規表現でファイル/内容を検索 |
| @メンション | プロンプトに`@ファイル名`で文脈としてファイルを追加 |
| ドラッグ&ドロップ | 画像、PDF、その他のファイルをプロンプトにドロップして添付 |
| ビジュアルdiffレビュー | 変更箇所をファイル単位で確認、行単位でコメント可能 |

### 2-2. コマンド実行系

| 機能 | 説明 |
|------|------|
| Bashツール | ターミナルコマンドを実行（サンドボックス対応） |
| Git操作 | commit、push、PR作成、ブランチ管理など |
| パッケージマネージャ | npm、pip等のインストール・実行 |
| テスト実行 | テストスイートの実行と結果確認 |
| ビルド・デプロイ | プロジェクトのビルドとデプロイ |
| バックグラウンドタスク | Ctrl+Bでタスクをバックグラウンドに送る |

### 2-3. Computer Use（コンピュータ操作）

| 機能 | 説明 |
|------|------|
| アプリ操作 | デスクトップアプリを開いて操作 |
| 画面操作 | マウスクリック、キーボード入力、スクロール |
| スクリーンショット | 画面のキャプチャと認識 |
| アプリ権限管理 | アプリごとにView only / Click only / Full controlの3段階 |
| ブラウザ操作 | Chrome拡張連携で高精度なWeb操作 |

### 2-4. パーミッションモード（権限制御）

| モード | 設定キー | 説明 |
|--------|----------|------|
| Ask permissions（確認モード） | `default` | ファイル編集・コマンド実行前に毎回確認。初心者におすすめ |
| Auto accept edits（編集自動承認） | `acceptEdits` | ファイル編集は自動承認、コマンドは確認 |
| Plan mode（計画モード） | `plan` | コード分析と計画立案のみ。ファイル変更・コマンド実行なし |
| Auto mode（自動モード） | `auto` | バックグラウンドの安全チェック付きで自動実行。Team/Enterprise/APIプラン向け |
| Bypass permissions（権限省略） | `bypassPermissions` | すべての確認を省略。サンドボックス環境専用 |

### 2-5. 設定・カスタマイズ系

| 機能 | 説明 |

### 2-8. MCP サーバー連携

- **リモートMCPコネクタ**：GitHub、Slack、Linear、Notion、Google Calendar等との連携
- **カスタムMCPサーバー**：settings.jsonで手動設定可能
- **MCPプロンプト**：MCPサーバーが公開するプロンプトを`/mcp__<server>__<prompt>`形式で利用可能
- **コネクタ管理**：デスクトップアプリの設定画面またはプロンプトの「+」ボタンから

### 2-9. マルチセッション・並列作業

- **並列セッション**：複数のセッションを同時に実行（Git worktreeで自動分離）
- **リモートセッション**：Anthropicクラウドで長時間タスクを実行（パソコンを閉じてもOK）
- **SSHセッション**：リモートサーバー上で作業
- **Dispatch連携**：スマートフォンからタスクを送信
- **「Continue in」メニュー**：ローカルセッションをリモートやIDEに移行
- **複数リポジトリ**：リモートセッションでは複数リポジトリを同時に扱える

### 2-10. エージェント機能

- **サブエージェント**：特定のタスクに特化したエージェントを定義・管理（`/agents`）
- **スキル**：拡張機能を読み込み、`/`コマンドで呼び出し
- **プラグイン**：スキル、エージェント、フック、MCPサーバー、LSP設定をパッケージとして管理
- **Agent Teams**：複数エージェントの協調動作（`--teammate-mode`）
- **バンドルスキル**：simplify、batch、debug、loop、claude-api

### 2-11. プレビュー・検証機能

- **ライブプレビュー**：開発サーバーを起動して埋め込みブラウザで確認
- **自動検証（Auto-verify）**：編集後に自動でスクリーンショット、DOM検査、エラーチェック
- **PR監視**：GitHub PRのCIステータスを表示、自動修正・自動マージ対応
- **コードレビュー**：diff画面でClaudeにコードレビューを依頼

### 2-12. その他

- **音声入力**：20言語対応のPush-to-Talk（/voice）
- **スケジュール実行**：cronのように定期的にタスクを実行（クラウド上）
- **Remote Control**：claude.aiやモバイルアプリから操作
- **GitHub Actions連携**：PRコメントに応答するCI統合
- **Slack App連携**：Slackからタスクを送信
- **チャンネル機能**：外部通知を受け取るリスニング機能（リサーチプレビュー）
- **エンタープライズ設定**：MDM、SSO、管理コンソールによる組織管理

## 3. Proプラン契約の手順

### 3-1. 契約に必要なもの

- **メールアドレス**：Googleアカウントまたは任意のメールアドレス
- **クレジットカードまたはデビットカード**：決済手段として必須
- **サポート地域の電話番号**：認証に必要な場合あり
- ※2026年4月現在、30日間の無料トライアルが提供されている場合があります

### 3-2. 契約手順（ステップバイステップ）

**ステップ1：claude.aiにアクセス**
- ブラウザで https://claude.ai にアクセス

**ステップ2：アカウント作成**
- 「Continue with email」でメールアドレスを入力、または「Continue with Google」でGoogleアカウントを使用
- メール認証の場合は、受信したメールのリンクをクリック

**ステップ3：電話番号認証（初回のみ）**
- 電話番号を入力してSMS認証コードを受け取る
- 認証コードを入力して完了

**ステップ4：設定画面を開く**
- 画面左下の自分のイニシャル（アイコン）をクリック
- 「Settings」を選択

**ステップ5：Billing（課金）ページへ移動**
- Settings内の「Billing」を選択
- 「Upgrade plan」をクリック

**ステップ6：プランを選択**
- 「Get Pro plan」をクリック（月額20ドル）
- 請求期間を選択：月額（$20/月）または年額（$17/月、年一括払い）

**ステップ7：支払い情報を入力**
- クレジットカード/デビットカード番号を入力
- 「Subscribe」をクリック

**ステップ8：完了確認**
- 画面に「Pro」の表示が出れば契約完了
- すぐにClaude Codeを含む全Pro機能が利用可能

### 3-3. 契約後すぐにデスクトップアプリが使えるか

**はい、すぐに使えます。**

- Pro契約完了後、即座にClaude Codeの全機能にアクセス可能
- デスクトップアプリは https://claude.com/download からダウンロード
- アプリをインストールし、Pro契約したアカウントでログインすればすぐに利用開始

### 3-4. 想定される問題と対策

| 問題 | 原因 | 対策 |
|------|------|------|
| カード決済失敗 | 海外決済の制限、残高不足 | 事前にカード会社に海外決済を有効化するよう連絡。予備のカードを用意 |
| メール認証の遅延 | メールサーバーの遅延、迷惑メール扱い | 迷惑メールフォルダを確認。数分待つ。別のメールアドレスを使用 |
| SMS認証コードが届かない | 電話番号の形式誤り、通信障害 | 国番号（+81）を正しく入力。Wi-Fi通話をオフにする |
| Googleログインエラー | ブラウザのポップアップブロック | ポップアップを許可する。別のブラウザで試す |
| デスクトップアプリが起動しない | OS要件未達、インストール不完了 | アプリを再ダウンロード。OS要件を確認（Windows 10以上、macOS 12以上） |
| Git未インストール（Windows） | Codeタブ使用にGitが必要 | https://git-scm.com/downloads/win からGitをインストールし、アプリを再起動 |

### 3-5. セミナーでの所要時間の見積もり（8人同時契約）

| 手順 | 所要時間目安 |
|------|-------------|
| サイトアクセス＋アカウント作成 | 3〜5分 |
| メール/SMS認証 | 2〜5分（遅延含む） |
| Pro契約（カード入力〜完了） | 3〜5分 |
| デスクトップアプリのダウンロード・インストール | 3〜5分 |
| アプリでのログイン・初期設定 | 2〜3分 |
| トラブル対応の余裕 | 5〜10分 |
| **合計** | **約20〜30分** |

**セミナーでのおすすめ進行：**

- 事前にメールアドレスとクレジットカードを用意するよう参加者に伝える
- 手順を画面共有しながら全員で同時に進める
- つまずいた人には個別に声をかけながら全体を進行する
- 無料トライアル（30日間）がある場合は、その旨を案内すると契約のハードルが下がる

## 補足：セミナーでの説明ポイント

### 初心者に伝えるべきClaude Code の3つの特徴

1. **開発者向けの強力なAIアシスタント**：コードを書く、読む、修正する、テストする、デプロイする ── 開発のあらゆる工程を支援
2. **5段階の権限制御**：最初は「確認モード」で安心して使い始め、慣れたら「自動モード」に切り替えられる
3. **マルチ環境対応**：ローカル、クラウド、リモートサーバー、スマートフォンからの操作 ── 場所を選ばず使える

### Coworkとの使い分けのまとめ

- **コードを書く・開発する** → Claude Code
- **調査する・文書を作る・デスクトップ操作する** → Cowork
- **安定性・信頼性を重視** → Claude Code（正式リリース済み）
- **非技術者が使う** → Cowork（ただしリサーチプレビューであることに注意）
