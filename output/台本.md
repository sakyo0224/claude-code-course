## セクション2：AIとClaude Codeとは（5分）

### スライド5：AIとは何か（約50秒）

**話す言葉：**
「AIって聞くと、なんだか難しそうなイメージがありませんか？映画に出てくるロボットとか、プログラミングとか。でも実は、みなさんが普段やっていることと変わらないんです。メールで誰かにお願いごとを書きますよね。『この資料まとめといて』って。AIに対しても、まったく同じです。日本語でお願いごとを書くだけ。プログラミングの知識はゼロで大丈夫です。」

**補足メモ：**
- 「AIは難しい」という先入観を最初に取り除く
- 日常の「メールでお願いする」行為と結びつける

---

### スライド6：チャット型 vs エージェント型（約60秒）

**話す言葉：**
「AIツールには大きく2つのタイプがあります。1つが『チャット型』、もう1つが『エージェント型』です。わかりやすく言うと──チャット型は『レシピを教えてくれる人』です。『この料理どうやって作るの？』って聞くと、手順を教えてくれる。でも作るのは自分ですよね。一方、エージェント型は『料理を作ってくれるシェフ』です。材料を渡して『これでパスタ作って』と言えば、完成品が出てくる。今日やるClaude Code はこの『シェフ』の方です。ファイルを渡したら、読んで、作業して、結果を保存するところまで全部やってくれます。」

**補足メモ：**
- 比喩は「レシピ vs シェフ」が直感的に伝わりやすい
- 余裕があれば「カーナビ vs 自動運転車」も補足で使える
- チャット型の代表例として、ブラウザ版のClaudeやChatGPTをイメージしてもらう

---

### スライド7：Claude Code デスクトップアプリの特徴（約50秒）

**話す言葉：**
「Claude Code のデスクトップアプリは、パソコンの中にあるファイルに直接触れるのが一番の特徴です。ブラウザで使うAIだと、ファイルの中身をコピーして貼り付けて、出力をまたコピーして……という手間がかかりますよね。デスクトップアプリなら、ファイルをポンと渡すだけ。しかもWindows対応がつい先日、4月3日に追加されたばかりなので、お使いのパソコンがMacでもWindowsでも大丈夫です。」

**補足メモ：**
- 「ファイルに直接触れる」がブラウザ版との最大の違いであることを強調
- Windows対応が最新情報であることに触れると「新しいツール」感が出る

### 【演習2タイム】（約5分）

**声かけのタイミング：**

- **開始直後（0:00）：** 「では始めましょう。ファイルをアプリの画面にドラッグ＆ドロップしてみてください。」
- **30秒後：** 「ドラッグ＆ドロップがうまくいかない方は、ファイルをデスクトップに移してからもう一度試してみてください。それでもダメな場合は手を挙げてくださいね。」
- **1分30秒後：** 「ファイルを渡せたら、スライドに書いてある指示をコピーして入力してください。」
- **2分30秒後：** 「結果が出てきた方、どうですか？ちゃんと整理されていますか？」

### スライド34：パーミッションモード（約60秒）

**話す言葉：**
「Claude Code には3つの許可モードがあります。default は毎回確認を求めるモード。acceptEdits にすると、ファイル編集は自動でOKになって、確認はコマンド実行のときだけ。plan は読み取り専用で、計画だけ立てるモードです。おすすめは acceptEdits です。送信ボタンの横にモードセレクターがあるので、クリックするだけで切り替えられます。これだけで、作業スピードが一気に上がります。なお、完全に自動で動く Auto モードもありますが、これは Team プラン以上が必要です。今日の Pro プランでは acceptEdits がベストです。」

### スライド35：ハーネスエンジニアリングとは（約90秒）

**話す言葉：**

ChatGPTに上手に質問する技術を"プロンプトエンジニアリング"と言いますよね。ハーネスエンジニアリングはその上位概念で、1回の質問だけじゃなく、AIが動くシステム全体を設計することです。これを知っているかどうかで、AIの活用度がまったく変わります。」

**補足メモ：**
- 「プロンプトエンジニアリングはハーネスの一部」という関係を強調
