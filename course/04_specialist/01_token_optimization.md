## 本文

### トークンとは何か — コストの根本を理解する

Claude Code の利用料は「トークン」と呼ばれる単位で計算されます。トークンとは、テキストをモデルが処理する際の最小単位です。日本語の場合、おおむね 1 文字が 1〜2 トークンに相当します。英語では 1 単語が約 1 トークン程度です。

コストは大きく **入力トークン**（あなたが送る情報）と **出力トークン**（Claude が返す情報）に分かれます。一般に出力トークンは入力トークンの数倍のコストがかかるため、「Claude に長文を書かせすぎない」ことが最初の最適化ポイントです。

### コンテキストウィンドウの肥大化パターン

セッションが長くなると、過去のやり取りがすべてコンテキストとして送信され続けます。以下のパターンでコストが急増します。

1. **会話の蓄積**: 50 回以上のやり取りが積み重なると、毎回のリクエストに過去の全履歴が含まれる
2. **大きなファイルの丸読み**: `Read` で数千行のファイルを読み込むと、その内容がコンテキストに残り続ける
3. **不要なツール出力**: `git log` や `find` の結果が大量に返されてコンテキストを圧迫する
4. **画像の貼り付け**: スクリーンショット 1 枚で約 1,334 トークン（1000x1000px の場合）を消費する

### 具体的な最適化テクニック

**テクニック 1: `/compact` による会話の圧縮**

```
/compact 現在の作業状態を保持しつつ圧縮してください
```

`/compact` コマンドは、過去のやり取りを要約してコンテキストを縮小します。引数に「何を残すべきか」を指示することで、重要な文脈を失わずに圧縮できます。目安として、やり取りが 20〜30 回を超えたら実行しましょう。

**テクニック 2: 自動コンパクトのしきい値設定**

`settings.json` に以下を追加すると、コンテキストが一定割合に達した際に自動で圧縮が走ります。

```json
{
  "contextCompression": {
    "autoCompactThreshold": 75
  }
}
```

75% を超えたら自動圧縮する設定です。これにより、手動で `/compact` を実行する手間が省けます。

※ 設定項目は将来のバージョンで変更される可能性があります。公式ドキュメントで最新の設定を確認してください。

**テクニック 3: ファイル読み込みの範囲指定**

ファイルの全行を読み込む代わりに、必要な範囲だけを読み込みます。

```
# 悪い例: 3000行のファイルを丸ごと読み込む
「app.py を読んで」

# 良い例: 必要な行だけを指定する
「app.py の 100〜150行目を読んで」
```

**テクニック 4: セッション分割戦略**

1 つの大きなタスクを複数のセッションに分けて実行します。

```
セッション 1: 設計と計画 → 計画をファイルに保存
セッション 2: 実装（計画ファイルを読み込んで開始）
セッション 3: テストとデバッグ
```

セッション間の引き継ぎには、計画や進捗をファイルに書き出しておくのが効果的です。

**テクニック 5: `/cost` と `/context` で現状を可視化する**

```
/cost     # 現在のセッションのトークン消費量を確認
/context  # コンテキストウィンドウの使用状況を可視化
```

この 2 つのコマンドを定期的に実行し、「今どれくらい消費しているか」を意識する習慣をつけましょう。

**テクニック 6: CLAUDE.md を簡潔に保つ**

CLAUDE.md はセッション開始時に毎回読み込まれます。内容が冗長だと、それだけで数百〜数千トークンを常に消費します。

```markdown
# 良い CLAUDE.md の例（簡潔）
- TypeScript + React プロジェクト
- テスト: `npm test` を使う
- コーディング規約: ESLint の設定に従う

# 悪い CLAUDE.md の例（冗長）
- このプロジェクトは2024年に開始され、TypeScriptとReactを使用しています。
  当初はJavaScriptで書かれていましたが、2024年にTypeScriptへ移行しました。
  （以下、背景説明が延々と続く…）
```

### Pro/Max プランでの消費目安

Anthropic の公開データによると、Claude Code ユーザーの 1 日あたりの平均消費は約 6 ドル相当で、90% のユーザーは 12 ドル以下に収まっています。ただし、大きなファイルを頻繁に扱ったり、長時間セッションを続けたりすると、これを大きく超える場合があります。

## やってみよう（ハンズオン）

### ステップ 1: 現在のコスト状況を確認する

Claude Code を起動して、いくつかのやり取りを行ったあとに以下を実行してください。

```
/cost
/context
```

表示された数字を記録しておきましょう。

### ステップ 2: `/compact` の効果を計測する

```
/compact 作業の進捗状態だけを保持してください
```

実行後にもう一度 `/context` を実行し、どれだけコンテキストが圧縮されたかを比較してください。

### ステップ 3: settings.json に自動圧縮を設定する

`~/.claude/settings.json` を開き、`autoCompactThreshold` を追加してください。設定後、長いセッションで自動圧縮が発動するか確認しましょう。

---

## まとめ
- トークン消費の主な原因は「会話の蓄積」「大きなファイルの丸読み」「不要なツール出力」の 3 つ
- `/compact` と自動圧縮設定で、コンテキストの肥大化を防ぐ
- `/cost` と `/context` で定期的に消費状況を確認する習慣をつける
- CLAUDE.md は簡潔に保ち、常時消費されるトークンを最小化する
