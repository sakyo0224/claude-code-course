## 本文

### ワークフロー設計の全体像

業務ワークフローの自動化は、以下の 5 つのフェーズで設計します。

```
1. 分析    → 現在の業務プロセスを可視化する
2. 設計    → 自動化する範囲と人間の判断ポイントを決める
3. 構築    → Claude Code の機能を組み合わせて実装する
4. テスト  → 小規模で試行し、問題点を洗い出す
5. 運用    → 本格運用を開始し、継続的に改善する
```

### フェーズ 1: 業務プロセスの分析

まず、自動化したい業務プロセスを「入力」「処理」「出力」「判断」の 4 要素に分解します。

```
例: 週次レポート作成プロセス

入力:
  - 各部署の CSV データ（売上、在庫、顧客数など）
  - 前週のレポート（前週比の計算に使用）

処理:
  - データの集計と分析
  - 前週比の計算
  - グラフの生成
  - レポート文面の作成

出力:
  - Word 形式の週次レポート
  - 経営陣への配信メール

判断（人間が行う）:
  - レポート内容の最終確認
  - 異常値があった場合の深掘り分析の要否
```

### フェーズ 2: 自動化範囲の設計

すべてを自動化するのではなく、「自動化すべき部分」と「人間が判断すべき部分」を明確に分けます。

**自動化に適した処理**
- 定型的なデータ収集・集計
- テンプレートに基づく文書生成
- ファイル形式の変換
- 定期的な監視とアラート

**人間の判断が必要な処理**
- 異常値の解釈と対応判断
- 顧客への送信内容の最終確認
- セキュリティ上の重要な判断
- ビジネス戦略に関わる意思決定

### フェーズ 3: ワークフローの構築

Claude Code で利用できる機能を組み合わせて、ワークフローを構築します。

**構成要素の選択基準**

| 要件 | 使用する機能 |
|------|------------|
| 定期実行 | `/schedule`（スケジュール実行） |
| 外部サービス連携 | MCP コネクタ / カスタム MCP サーバー |
| ファイル処理 | Claude Code の Read/Write/Edit |
| 品質チェック | Hooks（PostToolUse） |
| 通知 | MCP コネクタ（Slack、メール） |
| 人間の確認 | パーミッションモード（default） |

**実践例: 週次レポート自動生成ワークフロー**

```
スケジュール（毎週月曜 9:00）
    │
    ▼
ステップ 1: データ収集
    Claude Code が /data フォルダの CSV を読み込む
    │
    ▼
ステップ 2: 分析と集計
    前週比を計算し、異常値を検出
    │
    ▼
ステップ 3: レポート生成
    テンプレートに基づいて Word 形式のレポートを生成
    │
    ▼
ステップ 4: 人間の確認 ← ここで一旦停止
    Slack に通知を送り、担当者がレポートを確認
    │
    ▼
ステップ 5: 配信
    確認後、メールで経営陣に配信
```

このワークフローを Claude Code で実装する場合、以下のように設計します。

```markdown
# workflow/weekly-report/CLAUDE.md

## 週次レポート自動生成ワークフロー

### 手順
1. /data/weekly/ フォルダ内の最新 CSV ファイルを読み込む
2. 各指標（売上、在庫、顧客数）を集計する
3. 前週のレポート（/reports/previous/）と比較して前週比を計算する
4. 10% 以上の変動がある指標をハイライトする
5. テンプレート（/templates/weekly-report.md）に基づいてレポートを生成する
6. /reports/current/ にレポートを保存する
7. 完了したら Slack の #reports チャンネルに通知する

### テンプレートの変数
- {{period}}: レポート期間
- {{summary}}: 経営サマリー
- {{highlights}}: 注目すべき変動
- {{details}}: 各指標の詳細
```

### `/schedule` によるスケジュール実行

`/schedule` コマンドを使って、ワークフローを定期的に実行できます。

```
/schedule "毎週月曜 9:00 に weekly-report ワークフローを実行"
```

スケジュール実行はクラウド上で行われるため、あなたのパソコンが起動していなくても動作します。

### MCP コネクタを活用した連携

外部サービスとの連携には、MCP コネクタを使用します。

```
# Slack への通知
Claude Code → MCP コネクタ（Slack）→ Slack チャンネル

# Google Calendar からの予定取得
Google Calendar → MCP コネクタ → Claude Code

# GitHub の PR 管理
Claude Code → MCP コネクタ（GitHub）→ GitHub
```

### ヒューマンインザループの設計

自動化ワークフローの中で、人間の確認や判断が必要なポイントを「ヒューマンインザループ」として設計します。

```json
// パーミッション設定で、特定の操作前に確認を求める
{
  "permissions": {
    "allow": [
      "Read",
      "Glob",
      "Grep",
      "Edit",
      "Write"
    ],
    "deny": [
      "Bash(git push *)"
    ]
  }
}
```

さらに、Hooks を使って重要な判断ポイントで通知を送ることもできます。

```bash
#!/bin/bash
# scripts/notify-for-review.sh
# レポート生成完了後に Slack で通知
curl -X POST "$SLACK_WEBHOOK_URL" \
  -d "{\"text\": \"週次レポートが生成されました。確認をお願いします。\nファイル: /reports/current/weekly-report.md\"}"
```

### ワークフローのテストと改善

本格運用の前に、必ず以下のテストを実施してください。

1. **ドライラン**: 実データの代わりにテストデータで全工程を通す
2. **エッジケース**: データが空の場合、異常値が大量にある場合などを確認
3. **障害テスト**: 外部サービスが応答しない場合の挙動を確認
4. **パフォーマンステスト**: 実データ量での処理時間とトークン消費を測定

## やってみよう（ハンズオン）

### ステップ 1: 業務プロセスを分解する

あなたの日常業務から、繰り返し行っているプロセスを 1 つ選び、「入力」「処理」「出力」「判断」に分解してください。

### ステップ 2: ワークフローの CLAUDE.md を作成する

分解した業務プロセスを、Claude Code が実行できる手順として CLAUDE.md に記述してください。

### ステップ 3: 小規模でテストする

テストデータを用意し、ワークフローの最初の 2〜3 ステップだけを実行してみてください。問題があれば手順を修正します。

---

## まとめ
- 業務ワークフローの自動化は「分析 → 設計 → 構築 → テスト → 運用」の 5 フェーズで進める
- 自動化すべき部分と人間が判断すべき部分を明確に分ける
- `/schedule` でクラウド上の定期実行、MCP コネクタで外部サービス連携を実現する
- ヒューマンインザループを適切に配置し、重要な判断は人間が行う
