## 本文

### Claude の3つのモデルファミリー

Claude Code では、目的に応じて異なるAIモデルを選択できます。モデルとは、AIの「頭脳」にあたる部分で、モデルによって得意分野や処理速度が異なります。

| モデル | 特徴 | たとえると |
|--------|------|----------|
| **Haiku（ハイク）** | 高速・軽量。シンプルなタスク向き | 自転車：軽くて速い。近距離の移動に最適 |
| **Sonnet（ソネット）** | バランス型。日常業務の多くに対応 | 自動車：速度と積載量のバランスが良い |
| **Opus（オーパス）** | 最高性能。複雑なタスクに強い | トラック：重い荷物も運べるが、燃費は高い |

### 各モデルの詳細

#### Haiku — 速さ重視の軽量モデル

Haiku は、処理速度が最も速く、利用枠の消費も少ないモデルです。

**得意なこと：**
- 簡単な質問への回答
- 短い文章の作成・修正
- ファイルの簡単な変換作業
- 「はい/いいえ」で答えられるような単純な判断

**不向きなこと：**
- 長文の分析や複雑な推論
- 大量のデータを横断的に処理する作業

#### Sonnet — 日常業務の万能選手

Sonnet は、速度と能力のバランスに優れたモデルです。Claude Code のデフォルト（初期設定）は Sonnet が選ばれていることが多く、ほとんどの日常業務はこれで十分にこなせます。

**得意なこと：**
- レポートや文書の作成
- データの集計・分析
- メール文面の生成
- 中程度の複雑さを持つタスク全般

#### Opus — 最高品質を求めるとき

Opus は、最も高い能力を持つモデルです。複雑な分析や、高い品質が求められる文書作成に向いています。ただし、処理に時間がかかり、利用枠の消費も多くなります。

**得意なこと：**
- 複雑な分析レポートの作成
- 大量のデータを横断的に分析する
- 高い精度が求められる作業
- 長い文章の構成・校正

### モデルの切り替え方

#### /model コマンド

```
/model
```

と入力すると、利用可能なモデルの一覧が表示され、選択できます。特定のモデルを直接指定することもできます。

```
/model sonnet
```

#### キーボードショートカット

`Cmd+P`（macOS）または `Meta+P`（Windows）を押すと、モデル選択画面がすぐに開きます。

#### /fast コマンド

```
/fast on
```

高速モードをオンにすると、自動的に軽量なモデルが選択されます。速度重視のタスクで便利です。

```
/fast off
```

で元に戻ります。

### 推論の「がんばり度」を調整する

モデルの切り替えとは別に、`/effort` コマンドで推論の深さ（「がんばり度」）を調整できます。

```
/effort low    — 素早く簡潔に回答（トークン消費が少ない）
/effort medium — 標準的な回答
/effort high   — じっくり考えて丁寧に回答
/effort max    — 最大限の推論を行う
/effort auto   — タスクの複雑さに応じて自動調整（おすすめ）
```

「がんばり度」は、同じモデルでも回答の質と速度に影響します。簡単な質問なら `low`、重要な分析なら `high` というように使い分けると、利用枠を効率的に使えます。

### タスク別おすすめ設定

| タスク | おすすめモデル | がんばり度 |
|--------|-------------|----------|
| 「今日の日付を教えて」のような簡単な質問 | Haiku | low |
| メール文面の作成 | Sonnet | medium |
| CSVデータの集計 | Sonnet | medium |
| 50件のPDF分析レポート | Sonnet | high |
| 年度末の経営分析レポート | Opus | high |
| 複雑な比較表の作成 | Opus | high |

### コストを意識したモデル選択

利用枠（Pro プランの場合、5時間あたり約45メッセージ）を効率よく使うために、以下を意識しましょう。

1. **デフォルトは Sonnet で十分**：ほとんどの作業は Sonnet で問題なく処理できます
2. **簡単なタスクは Haiku + low effort**：利用枠の消費を最小限に抑えられます
3. **Opus は本当に必要なときだけ**：品質の違いが明確に求められるタスクに限定しましょう
4. **/cost で消費量を確認**：定期的に `/ cost` コマンドで利用状況を確認しましょう

## やってみよう（ハンズオン）

### ステップ1：現在のモデルを確認する

```
/model
```

現在選択されているモデルを確認しましょう。

### ステップ2：モデルによる回答の違いを体験する

同じ質問を異なるモデルで試してみましょう。

まず Haiku で試します。

```
/model haiku
```

```
日本の四季の特徴を簡潔に説明してください。
```

次に Sonnet で同じ質問をします。

```
/model sonnet
```

```
日本の四季の特徴を簡潔に説明してください。
```

回答の詳しさや表現の違いを比べてみてください。

### ステップ3：がんばり度の違いを体験する

同じモデル（Sonnet）で、がんばり度を変えて試してみましょう。

```
/effort low
```

```
効率的な時間管理の方法を教えてください。
```

```
/effort high
```

```
効率的な時間管理の方法を教えてください。
```

回答の深さや具体性の違いを確認してください。

### ステップ4：元に戻す

テストが終わったら、標準設定に戻しておきましょう。

```
/model sonnet
/effort auto
```

## まとめ
- Claude には **Haiku（速い）** / **Sonnet（バランス）** / **Opus（高性能）** の3つのモデルファミリーがある
- ほとんどの日常業務は **Sonnet** で十分。簡単なタスクは **Haiku**、複雑な分析は **Opus** と使い分ける
- **/model** コマンドでモデルを切り替え、**/effort** コマンドで推論の深さを調整できる
- 利用枠を効率的に使うには、タスクの難易度に合わせてモデルとがんばり度を選ぶことが大切
