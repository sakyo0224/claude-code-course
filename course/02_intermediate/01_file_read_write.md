## 本文

### Claude Code のファイル操作とは

Claude Code は、あなたのパソコン上のファイルに直接アクセスして読み書きする機能を持っています。これは通常のチャット型 AI にはない、エージェント型 AI ならではの強力な特徴です。

Claude Code がファイルを扱うときに使う主なツールは次の3つです。

| ツール名 | 役割 | たとえると |
|---------|------|----------|
| **Read** | ファイルの中身を読む | 書類を手に取って目を通す |
| **Write** | 新しいファイルを作成する | 白紙に一から書き起こす |
| **Edit** | 既存ファイルの一部を修正する | 書類の一部を消して書き直す |

### Read — ファイルを読む

Read ツールは、指定されたファイルの中身を読み取ります。テキストファイルはもちろん、CSV（表形式データ）、PDF、画像ファイルなど、さまざまな形式に対応しています。

Claude Code に「このファイルを読んで」と伝えると、内部で Read ツールが呼び出され、ファイルの内容を理解した上で回答してくれます。

**Read が活躍する場面の例：**
- 「この報告書の内容を要約して」
- 「このCSVファイルのデータを分析して」
- 「この画像に何が写っているか教えて」

### Write — 新しいファイルを作る

Write ツールは、まったく新しいファイルをゼロから作成します。指定したパス（保存場所）にファイルを生成してくれます。

**Write が活躍する場面の例：**
- 「議事録のテンプレートを作って」
- 「売上データをまとめたCSVファイルを作成して」
- 「プレゼン資料の構成案をテキストファイルにして」

### Edit — 既存ファイルを修正する

Edit ツールは、すでに存在するファイルの一部分だけを書き換えます。ファイル全体を上書きするのではなく、変更が必要な箇所だけをピンポイントで修正するのが特徴です。

これは、書類の一行だけを訂正するようなイメージです。ファイルの他の部分はそのまま残ります。

**Edit が活躍する場面の例：**
- 「この文書の3行目の日付を4月6日に変更して」
- 「CSVファイルのヘッダー行を英語に変えて」
- 「設定ファイルのこの値だけ変更して」

### Write と Edit の使い分け

初心者の方が迷いやすいのが、Write と Edit のどちらを使うべきかという点です。基本的な判断基準は次のとおりです。

- **新しいファイルを作りたい** → Write
- **既存のファイルを少し直したい** → Edit
- **既存のファイルを大幅に書き直したい** → Write（全体を上書き）

実は、Claude Code はあなたの指示内容から自動的に最適なツールを選んでくれます。「このファイルの〇〇を修正して」と言えば Edit を、「新しくファイルを作って」と言えば Write を選びます。ツール名を直接指定する必要はありません。

### ファイル指定の方法

Claude Code にファイルを伝える方法はいくつかあります。

1. **パスを直接書く**：「/Users/yourname/Documents/report.txt を読んで」
2. **相対パスを使う**：「./data/sales.csv を分析して」
3. **@メンションを使う**：入力欄に `@` を打つとファイル候補が表示されます
4. **ドラッグ&ドロップ**：デスクトップアプリでは、ファイルを直接ドラッグして渡せます

特に便利なのが **@メンション** と **ドラッグ&ドロップ** です。パスを手入力する手間が省けるので、ぜひ活用してください。

### パーミッション（許可）について

Claude Code がファイルを操作する際、初期設定では操作ごとに確認ダイアログが表示されます。これは「パーミッション（許可）」という安全機能です。

- **Read（読み取り）** は、基本的に確認なしで実行されます
- **Write（書き込み）** や **Edit（編集）** は、実行前に「この操作を許可しますか？」と聞かれます

「Allow（許可）」を押すと操作が実行され、「Always allow（常に許可）」を押すと、同じ種類の操作を今後は自動で実行するようになります。

## やってみよう（ハンズオン）

### ステップ1：ファイルを読み取る

まず、テスト用のファイルを Claude Code に読んでもらいましょう。パソコンのデスクトップに適当なテキストファイルがあれば、それを使います。なければ以下のように入力してください。

```
デスクトップに test.txt というファイルを作って、中身は「Claude Code のファイル操作テストです。今日は良い天気ですね。」にしてください。
```

ファイルができたら、次の指示を試してみましょう。

```
デスクトップの test.txt を読んで、内容を教えてください。
```

Claude Code が Read ツールを使ってファイルの中身を表示してくれるはずです。

### ステップ2：ファイルを編集する

次に、作成したファイルの内容を書き換えてみましょう。

```
デスクトップの test.txt の「良い天気」を「素晴らしい天気」に変更してください。
```

Claude Code が Edit ツールを使って、該当箇所だけを変更します。変更前と変更後の差分（何が変わったか）も表示されるので、確認してみてください。

### ステップ3：新しいファイルを作成する

最後に、少し実用的なファイルを作ってもらいましょう。

```
デスクトップに weekly_report_template.txt というファイルを作ってください。内容は、週次報告書のテンプレートで、「報告日」「担当者」「今週の成果」「来週の予定」「課題・相談事項」の項目を含めてください。
```

作成されたファイルを開いて、中身を確認してみましょう。

## まとめ
- Claude Code は **Read（読む）** / **Write（書く）** / **Edit（直す）** の3つのツールでファイルを操作する
- どのツールを使うかは、指示内容から Claude Code が自動的に判断してくれる
- **@メンション** や **ドラッグ&ドロップ** を活用すると、ファイル指定が楽になる
- ファイル操作時にはパーミッション（許可）の確認が表示されるので、内容を確認してから承認する
