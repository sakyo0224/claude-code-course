## 本文

### ビジネスシナリオ

あなたは、企画部門で新しいプロジェクト管理ツールの導入を検討しています。上司から「主要な5つのツールを比較して、おすすめを提案してほしい」と依頼されました。

通常であれば、各社のWebサイトを巡回し、料金・機能・口コミを調べ、比較表をExcelでまとめ、レポートを書くという作業に、丸1日（6〜8時間）かかるところです。

Claude Code を活用すれば、あなたが集めた情報を効率的に整理・分析し、見やすい比較表とレポートを短時間で作成できます。

### 競合調査のワークフロー

競合調査は、以下のステップで進めます。

#### ステップ1：調査対象と比較項目を決める

まず、何を調べ、何を比較するかを明確にします。

**調査対象：** 比較する製品やサービスのリスト
**比較項目：** 料金、主要機能、対応プラットフォーム、サポート体制、ユーザー評価など

#### ステップ2：情報を収集する

各社のWebサイトやレビューサイトから情報を集めます。集めた情報はテキストファイルやCSVに記録しておきます。

**情報収集のポイント：**
- 公式サイトの料金ページ
- 機能一覧ページ
- 導入事例ページ
- レビューサイトの評価

#### ステップ3：Claude Code で情報を整理・比較する

集めた情報を Claude Code に渡して、構造化された比較表を作成してもらいます。

#### ステップ4：分析レポートを作成する

比較表をもとに、メリット・デメリットの分析、おすすめの選定理由を含むレポートを生成してもらいます。

### 効果的な調査指示の出し方

**1. 比較項目を事前に決めて伝える**

```
以下の項目で比較表を作成してください。
- 月額料金（1ユーザーあたり）
- 無料プランの有無
- 最大ユーザー数
- 日本語対応
- モバイルアプリ対応
- 主要な連携サービス
- サポート対応時間
- 導入実績（目安）
```

**2. 評価基準を明示する**

```
各項目について以下の基準で評価してください。
- ◎（優れている）：業界標準を大幅に上回る
- ○（良い）：業界標準以上
- △（普通）：業界標準レベル
- ×（課題あり）：業界標準を下回る
```

**3. レポートの対象読者を伝える**

```
このレポートの読者は、IT知識が中程度の経営層です。
技術的な専門用語は避け、ビジネス上のメリット・デメリットを中心に記述してください。
```

### 比較表のフォーマット例

Claude Code に作成してもらう比較表のフォーマットを指定しましょう。

**Markdown 形式（報告書向け）：**

```markdown
| 項目 | ツールA | ツールB | ツールC |
|------|---------|---------|---------|
| 月額料金 | $10/人 | $15/人 | $8/人 |
| 無料プラン | あり | なし | あり（3人まで） |
```

**CSV 形式（Excelで開く場合）：**

```
項目,ツールA,ツールB,ツールC
月額料金,$10/人,$15/人,$8/人
無料プラン,あり,なし,あり（3人まで）
```

### 注意点：情報の鮮度と正確性

競合調査で特に注意すべきは、情報の鮮度です。

- **料金はよく変わる**：公式サイトで最新の料金を必ず確認しましょう
- **機能は随時追加される**：半年前の情報は古くなっている可能性があります
- **Claude Code の知識には限界がある**：AIの学習データに含まれていない最新情報は把握できません

そのため、Claude Code には「情報の整理・比較・レポート作成」を任せ、「最新情報の収集」は自分で行うか、Web検索と組み合わせるのが最も効果的です。

## やってみよう（ハンズオン）

### ステップ1：調査プロジェクトを準備する

```
デスクトップに competitive_research というフォルダを作成してください。

その中に以下のファイルを作成してください。

1. research_plan.md — 調査計画書
以下の内容を含めてください。

# プロジェクト管理ツール比較調査

## 調査目的
社内のプロジェクト管理を効率化するため、最適なツールを選定する

## 調査対象（5製品）
1. Asana
2. Trello
3. Notion
4. Monday.com
5. Backlog

## 比較項目
- 月額料金（1ユーザーあたり、10人利用時）
- 無料プランの有無と制限
- 主要機能（タスク管理、ガントチャート、カンバンボード、ファイル共有）
- 日本語対応
- モバイルアプリ
- 主要な外部サービス連携
- カスタマーサポート（日本語対応、対応時間）
- 導入しやすさ（学習コスト）

## 選定基準
- 予算：1人あたり月額2,000円以内
- 必須：日本語対応、モバイルアプリあり
- 重視：操作の簡単さ（ITリテラシーが高くない社員でも使える）
```

### ステップ2：収集した情報を記録する

実際の業務では各社のサイトを調べますが、ここでは Claude Code の知識をもとに情報を整理してもらいます。

```
research_plan.md の調査計画に基づいて、5つのプロジェクト管理ツール（Asana、Trello、Notion、Monday.com、Backlog）について知っている情報をもとに、各ツールの情報を個別ファイルにまとめてください。

competitive_research/data/ フォルダを作成し、各ツールのファイルを保存してください。
- data/asana.md
- data/trello.md
- data/notion.md
- data/monday.md
- data/backlog.md

各ファイルには、比較項目に対応する情報をすべて含めてください。
情報の確実性が低い場合は「要確認」と明記してください。
```

### ステップ3：比較表を作成する

```
data/ フォルダ内の5つのファイルを読み込んで、以下の2つの比較表を作成してください。

1. competitive_research/comparison_table.md
— Markdown形式の詳細比較表。すべての比較項目を網羅。

2. competitive_research/comparison_table.csv
— CSV形式（UTF-8 BOM付き）の比較表。Excelで開けるように。

比較表には、各項目について ◎○△× の評価も付けてください。
評価基準：
- ◎：業界トップクラス or 調査計画の選定基準を大幅に満たす
- ○：十分なレベル or 選定基準を満たす
- △：一部制限あり or 選定基準をギリギリ満たす
- ×：非対応 or 選定基準を満たさない
```

### ステップ4：分析レポートを作成する

```
比較表の内容をもとに、以下のフォーマットで分析レポートを作成してください。
ファイル名：competitive_research/analysis_report.md

# プロジェクト管理ツール比較分析レポート

## エグゼクティブサマリー
（結論を3行でまとめる。おすすめツールとその理由）

## 比較表（サマリー版）
（主要項目だけの簡潔な比較表）

## 各ツールの詳細評価

### ツール名
- 強み（3点）
- 弱み（2点）
- こんな組織に向いている

（5つのツールすべてについて記載）

## 総合評価ランキング
（選定基準に基づいた順位付けと根拠）

## 推奨
### 第1推奨：○○
（選んだ理由を3点）

### 第2推奨：○○
（第1推奨が合わなかった場合の代替案）

## 注意事項
- 本レポートの情報は調査時点のものです
- 料金は変更される可能性があります
- 導入前に無料トライアルで実際の使い勝手を確認することをおすすめします

## 次のステップ
（導入に向けた具体的なアクションプラン）

レポートの対象読者は、IT知識が中程度の経営層です。
専門用語は避け、ビジネス観点でのメリット・デメリットを中心に書いてください。
```

### ステップ5：成果物を確認する

```
competitive_research フォルダ内のすべてのファイルの一覧を表示し、各ファイルの概要（行数、主な内容）を教えてください。

また、analysis_report.md の品質について、以下の観点で自己評価してください。
- 論理的な一貫性
- 比較の公平性
- 読みやすさ
- 実用性（意思決定に使えるか）
```

## まとめ
- 競合調査は「計画→収集→整理→分析→レポート」の流れで進める
- Claude Code には「情報の整理・比較・レポート作成」を任せ、「最新情報の収集・確認」は自分で行う
- 比較項目と評価基準を事前に決めておくことで、公平で一貫性のある比較表が作れる
- レポートの読者層を伝えると、適切な粒度と表現で書いてもらえる
- 丸1日かかっていた競合調査を、1〜2時間に短縮できる可能性がある
