## 本文

### Bashツールとは

「Bash（バッシュ）」とは、パソコンに文字で命令を伝える仕組み（シェル）の一種です。普段、マウスでフォルダを開いたりファイルをコピーしたりしている操作を、文字のコマンド（命令文）で実行できます。

Claude Code の「Bashツール」は、このターミナルコマンドを Claude Code が代わりに実行してくれる機能です。つまり、あなたがターミナル操作を知らなくても、日本語で「こういうことをして」と伝えれば、Claude Code が適切なコマンドに変換して実行してくれます。

### Bashツールでできること

Bashツールを使うと、ファイル操作だけではカバーできない幅広い作業が可能になります。

| 分類 | できること | 具体例 |
|------|----------|--------|
| ファイル管理 | フォルダの作成・移動・コピー | 「data フォルダをバックアップして」 |
| データ処理 | ファイルの結合・変換・並べ替え | 「CSVファイルを日付順に並べ替えて」 |
| 情報取得 | システム情報の確認 | 「ディスクの空き容量を教えて」 |
| ツール実行 | インストール済みアプリの操作 | 「Python スクリプトを実行して」 |
| Git操作 | バージョン管理の操作 | 「変更をコミットして」 |

### Read/Write/Edit との違い

前の記事で学んだ Read / Write / Edit はファイルの中身を扱うツールでした。Bashツールはそれよりもっと広い範囲の操作をカバーします。

**たとえで言うと：**
- Read / Write / Edit = 書類を読む・書く・直す（机の上の作業）
- Bashツール = オフィス全体を動き回る（コピー機を使う、棚を整理する、電話をかけるなど）

### Bashツールの実行フロー

Claude Code がBashツールを使うとき、以下のような流れで処理されます。

1. あなたが日本語で指示を出す
2. Claude Code が適切なコマンドに変換する
3. 実行前に確認ダイアログが表示される（初期設定の場合）
4. 「Allow（許可）」を押すとコマンドが実行される
5. 実行結果が表示され、Claude Code がその結果を解釈して報告する

### サンドボックスによる安全対策

Bashツールには「サンドボックス」と呼ばれる安全対策が施されています。サンドボックスとは、危険な操作がパソコン全体に影響しないように、実行環境を隔離する仕組みのことです。

たとえば、ネットワークへの不正なアクセスや、システムファイルの改ざんなど、危険性のある操作は自動的にブロックされます。

### 安全に使うためのポイント

Bashツールは強力な機能なので、以下のポイントを押さえて安全に使いましょう。

1. **確認ダイアログを必ず読む**：実行前に「何をしようとしているか」が表示されます。意味がわからないコマンドは「Deny（拒否）」を選びましょう
2. **重要なファイルはバックアップを取る**：大量のファイル操作を行う前に、元のデータを別の場所にコピーしておくと安心です
3. **小さく試してから大きく実行する**：「まず1ファイルで試して」から「全ファイルに適用して」という順番で進めましょう
4. **削除系の操作は慎重に**：ファイルの削除やフォルダの消去は、元に戻すのが難しいことがあります

### バックグラウンド実行

時間のかかるコマンドを実行する場合、Claude Code はバックグラウンド（裏側）で処理を進めることができます。大量のファイル処理やデータ変換など、待ち時間が長くなりそうな作業で便利です。

`Ctrl + B` を押すと、実行中のタスクをバックグラウンドに送ることができます。バックグラウンドで走っている間も、別の質問や指示を Claude Code に送ることができます。

## やってみよう（ハンズオン）

### ステップ1：フォルダを作成・整理する

Bashツールの基本的な使い方を体験しましょう。

```
デスクトップに bash_test というフォルダを作って、その中に「2024年」「2025年」「2026年」という3つのサブフォルダを作成してください。
```

Claude Code が `mkdir` コマンドを使ってフォルダを作成してくれます。確認ダイアログが表示されたら、コマンドの内容を確認して「Allow」を押してください。

### ステップ2：システム情報を確認する

```
このパソコンの OS のバージョン、ディスクの空き容量、今日の日付を教えてください。
```

Claude Code がいくつかのコマンドを実行して、情報を集めて報告してくれます。

### ステップ3：ファイルの一括処理

前のハンズオンで作った search_test フォルダがあれば、以下を試してみましょう。

```
search_test フォルダ内のすべてのCSVファイルを、bash_test/2026年 フォルダにコピーしてください。
```

コピーが完了したら、確認のために次の指示も出してみましょう。

```
bash_test/2026年 フォルダの中身を一覧表示してください。
```

### ステップ4：実行結果を確認・活用する

```
bash_test フォルダ全体のフォルダ構造をツリー形式で表示してください。
```

フォルダの階層構造が見やすく表示されるはずです。

## まとめ
- **Bashツール** は、ターミナルコマンドを Claude Code が代わりに実行してくれる機能
- ファイル管理、データ処理、システム情報の取得など、幅広い操作に対応する
- 実行前に確認ダイアログが表示されるので、内容を確認してから許可する
- 重要なファイルはバックアップを取り、小さく試してから大きく実行するのが安全に使うコツ
