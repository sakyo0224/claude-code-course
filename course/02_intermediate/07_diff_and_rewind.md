# 変更管理 — /diff と /rewind を使いこなす

## この記事で学ぶこと
- /diff コマンドで変更箇所を確認する方法を覚える
- /rewind コマンドで安全に作業を巻き戻す方法を理解する
- 変更管理の習慣を身につけ、安心して作業を進められるようになる

## 前提知識

## 本文

### 変更管理とは

Claude Code で作業をすると、ファイルの作成・編集・削除といった変更が行われます。こうした変更の履歴を追跡し、必要に応じて元に戻せるようにすることを「変更管理」と呼びます。

日常的な例で言えば、Word の「元に戻す」（Ctrl+Z）や、ゲームの「セーブデータ」をロードする機能に似ています。ただし、Claude Code の変更管理はもっと強力で、複数のファイルにまたがる変更もまとめて管理できます。

### /diff — 何が変わったかを確認する

`/diff` コマンドは、Claude Code が行った変更の内容を一覧で表示する機能です。どのファイルのどの部分がどう変わったかを、視覚的に確認できます。

```
/diff
```

と入力すると、変更されたファイルの一覧が表示されます。各ファイルについて、以下の情報が確認できます。

- **追加された行**：緑色で表示される（新しく書き加えた部分）
- **削除された行**：赤色で表示される（元の内容から消えた部分）
- **変更されたファイル名**：どのファイルが影響を受けたか

### /diff を使うタイミング

`/diff` は、以下のタイミングで使うと効果的です。

1. **作業の途中で確認する**：「ここまでの変更を確認させて」
2. **作業完了後に検証する**：「全部の変更内容を見せて」
3. **意図しない変更がないかチェックする**：「変更してはいけないファイルに影響が出ていないか確認したい」

特に、大量のファイルを一括処理した後は、必ず `/diff` で変更内容を確認する習慣をつけましょう。

### /rewind — 変更を巻き戻す

`/rewind` コマンドは、Claude Code が行った変更を巻き戻す（元に戻す）機能です。「元に戻す」のゲームのセーブデータ版だと考えてください。

```
/rewind
```

と入力すると、会話のどの時点まで戻すかを選択する画面が表示されます。

**巻き戻しの3つの選択肢：**

| 選択肢 | 内容 | 使いどころ |
|--------|------|----------|
| コードと会話の両方を復元 | ファイルの変更も会話の内容も元に戻す | 完全にやり直したいとき |
| 会話のみ復元 | 会話を戻すが、ファイルの変更はそのまま | 会話の流れだけ修正したいとき |
| コードのみ復元 | ファイルの変更を元に戻すが、会話はそのまま | ファイルだけ元に戻したいとき |

### /rewind を使うタイミング

`/rewind` は、以下のような場面で活躍します。

- **予期しない変更があった場合**：「さっきの変更はやめて、元に戻して」
- **方針を変えたい場合**：「別のアプローチで最初からやり直したい」
- **途中の手順をやり直したい場合**：「ステップ3の前まで戻して」

### Escape キーによる素早い巻き戻し

`/rewind` コマンドの代わりに、**Escape キーを2回連続で押す**ことでも巻き戻しモードに入ることができます。急いで操作を取り消したいときに便利です。

### 変更管理のベストプラクティス

安心して作業を進めるために、以下の習慣を身につけましょう。

**1. 大きな作業の前にバックアップを取る**

```
作業を始める前に、data フォルダのバックアップを backup/ に作成してください。
```

**2. こまめに /diff で確認する**

一気に全部やってもらうのではなく、ステップごとに確認しながら進めましょう。

```
まずステップ1だけ実行してください。
```
（実行後）
```
/diff
```
（確認後）
```
OKです。ステップ2に進んでください。
```

**3. 怪しいと思ったらすぐ /rewind**

「あれ、何か変だな」と感じたら、すぐに `/rewind` で戻しましょう。早い段階で戻すほど、影響範囲が小さく済みます。

**4. /branch で分岐を作る**

実験的な作業をするときは、`/branch` コマンドで会話の分岐を作っておくと、安全に試行錯誤できます。

```
/branch 実験：別のフォーマットで出力を試す
```

うまくいかなければ元の分岐に戻ればよいので、気軽に試せます。

## やってみよう（ハンズオン）

### ステップ1：変更を加えてから /diff で確認する

まず、テスト用のファイルに変更を加えてもらいましょう。

```
デスクトップに diff_test.txt というファイルを作成してください。内容は以下のとおりです。

営業部 月次報告
報告日：2026年3月1日
売上：500万円
目標達成率：95%
コメント：あと一歩で目標達成でした。
```

次に、内容を変更してもらいます。

```
diff_test.txt の報告日を4月1日に、売上を620万円に、目標達成率を110%に変更してください。コメントも「目標を大幅に達成できました。」に変えてください。
```

変更が完了したら、`/diff` で確認してみましょう。

```
/diff
```

どの行が変わったか、色分けで確認できるはずです。

### ステップ2：/rewind で変更を巻き戻す

先ほどの変更を巻き戻してみましょう。

```
/rewind
```

「コードと会話の両方を復元」を選択してください。diff_test.txt の内容が元に戻っていることを確認しましょう。

```
diff_test.txt の内容を表示してください。
```

報告日が3月1日、売上が500万円に戻っていれば成功です。

### ステップ3：ステップバイステップで確認しながら進める

今度は、慎重に進める方法を実践してみましょう。

```
diff_test.txt の報告日だけ4月1日に変更してください。
```

```
/diff
```

問題なければ次に進みます。

```
diff_test.txt の売上を620万円に変更してください。
```

```
/diff
```

このように、一つずつ変更を確認しながら進めるのが、安全な作業の進め方です。

## まとめ
- **/diff** は変更内容を視覚的に確認するコマンド。大きな作業の後は必ず確認する
- **/rewind** は変更を巻き戻すコマンド。コード、会話、またはその両方を元に戻せる
- **Escape キー2回** で素早く巻き戻しモードに入れる
- 大きな作業の前にバックアップを取り、こまめに確認しながら進めるのが安全な習慣
