# ファイル検索 — Glob と Grep をマスターする

## この記事で学ぶこと
- Glob と Grep の違いと使い分けを理解する
- ファイル名で検索する方法（Glob）を覚える
- ファイルの中身で検索する方法（Grep）を覚える

## 前提知識
- [ファイル操作の基本 — Read / Write / Edit](01_file_read_write.md)

## 本文

### なぜファイル検索が大切なのか

仕事で使うパソコンには、何百、何千ものファイルが保存されています。「あのデータ、どこに保存したっけ？」と探し回った経験は、誰にでもあるのではないでしょうか。

Claude Code には、大量のファイルの中から目的のものを素早く見つけ出す検索機能があります。検索には2種類のツールが用意されています。

| ツール名 | 検索対象 | たとえると |
|---------|---------|----------|
| **Glob** | ファイルの名前やパス | 図書館で本のタイトルから探す |
| **Grep** | ファイルの中身（テキスト） | 本の中に書かれた言葉から探す |

### Glob — ファイル名で探す

Glob（グロブ）は、ファイル名のパターンを指定して検索するツールです。「グロブパターン」と呼ばれる特殊な記号を使って、条件に合うファイルを一括で見つけ出します。

**よく使うパターン：**

| パターン | 意味 | 例 |
|---------|------|-----|
| `*` | 何でも（任意の文字列） | `*.csv` → すべてのCSVファイル |
| `**` | フォルダの階層を超えて検索 | `**/*.pdf` → すべてのフォルダ内のPDFファイル |
| `?` | 任意の1文字 | `report_?.txt` → report_1.txt、report_A.txt など |

たとえば「このフォルダ内のすべてのExcelファイルを探して」と指示すると、Claude Code は Glob ツールを使って `**/*.xlsx` のようなパターンで検索を行います。

### Grep — ファイルの中身で探す

Grep（グレップ）は、ファイルの中に書かれている文字列で検索するツールです。「このフォルダ内のファイルの中で『売上合計』という言葉が書かれているものを探して」といった検索ができます。

Grep は「正規表現（せいきひょうげん）」という高度な検索パターンにも対応しています。正規表現とは、検索条件を柔軟に指定できる特別な書き方です。

**日常的に使う検索パターンの例：**

| やりたいこと | 指示の例 |
|------------|---------|
| 特定の単語を含むファイルを探す | 「"売上報告" という文字を含むファイルを探して」 |
| 特定の人名が出てくるファイルを探す | 「"田中" の名前が出てくるファイルを一覧にして」 |
| メールアドレスが含まれるファイルを探す | 「メールアドレスが記載されたファイルを見つけて」 |

### Glob と Grep の組み合わせ

この2つのツールは組み合わせることで、さらに強力になります。

**例：特定のフォルダ内のCSVファイルから「東京」という文字が含まれるものを探す**

```
data フォルダ内のCSVファイルの中で、「東京」という文字が含まれているファイルを探してください。
```

Claude Code は、まず Glob でCSVファイルを見つけ、次に Grep でその中から「東京」を含むものを絞り込んでくれます。

### 検索結果の活用

ファイルを見つけただけでは仕事は終わりません。検索結果を次のアクションにつなげることが大切です。

たとえば、次のような一連の指示ができます。

```
1. reports フォルダ内で「2024年度」を含むファイルを探して
2. 見つかったファイルの内容を要約して
3. 要約をまとめたレポートを作成して
```

このように、検索→読み取り→加工を一度の会話の中で行えるのが、Claude Code の強みです。

### 検索のコツ

効率よく検索するためのコツをいくつかご紹介します。

1. **まず範囲を絞る**：「パソコン全体」ではなく「このフォルダの中で」と指定すると、検索が速くなります
2. **ファイルの種類を指定する**：「すべてのファイル」ではなく「CSVファイルの中で」と限定すると精度が上がります
3. **キーワードは具体的に**：「売上」よりも「2024年度第3四半期売上」のように具体的にすると、必要なファイルだけが見つかります

## やってみよう（ハンズオン）

### ステップ1：テスト用ファイルを準備する

まず、検索のテスト用にいくつかのファイルを作りましょう。

```
デスクトップに search_test というフォルダを作って、以下の3つのファイルを作成してください。

1. sales_tokyo.csv — 内容：「店舗,売上\n東京本店,5000000\n渋谷店,3200000」
2. sales_osaka.csv — 内容：「店舗,売上\n大阪本店,4200000\n梅田店,2800000」
3. memo.txt — 内容：「来週の東京出張の際に、東京本店の売上データを確認すること」
```

### ステップ2：ファイル名で検索する（Glob）

```
search_test フォルダ内のCSVファイルを一覧表示してください。
```

2つのCSVファイル（sales_tokyo.csv と sales_osaka.csv）が表示されるはずです。memo.txt は含まれません。

### ステップ3：ファイルの中身で検索する（Grep）

```
search_test フォルダ内のすべてのファイルから「東京」を含むものを探してください。
```

今度は3つすべてのファイルが見つかるはずです（sales_tokyo.csv のデータ内と memo.txt の本文に「東京」が含まれるため）。

### ステップ4：検索結果をもとに作業する

```
search_test フォルダ内のCSVファイルをすべて読んで、全店舗の売上を合計した表を新しいCSVファイルとして作成してください。
```

検索→読み取り→集計→ファイル作成を一度の指示で実行してくれます。

## まとめ
- **Glob** はファイル名やパターンで検索する（「どんな名前のファイルがあるか？」）
- **Grep** はファイルの中身のテキストで検索する（「どのファイルにこの言葉が書かれているか？」）
- 2つを組み合わせると、膨大なファイルの中から必要な情報を効率的に見つけられる
- 検索結果をそのまま次の作業（要約、集計、レポート作成など）につなげられるのが Claude Code の強み
