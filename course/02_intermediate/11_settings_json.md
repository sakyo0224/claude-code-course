## 本文

### settings.json とは

settings.json は、Claude Code の動作を細かく制御するための設定ファイルです。JSON（ジェイソン）という構造化されたデータ形式で書かれています。

CLAUDE.md が「こういうルールで作業してね」という自然言語の指示書だとすれば、settings.json は「この操作は許可、この操作は禁止」という厳格なルールブックです。

**CLAUDE.md と settings.json の違い：**

| 項目 | CLAUDE.md | settings.json |
|------|-----------|---------------|
| 書き方 | 自由な文章 | JSON 形式（構造化） |
| 内容 | 作業のルールや好み | 動作の許可/禁止、技術設定 |
| 強制力 | Claude Code が「参考にする」 | システムが「強制する」 |
| たとえ | 業務マニュアル | セキュリティポリシー |

### settings.json の配置場所

settings.json にも、CLAUDE.md と同様に複数の配置場所があります。

| 場所 | パス | 適用範囲 |
|------|------|---------|
| ユーザー設定 | `~/.claude/settings.json` | すべてのプロジェクト |
| プロジェクト設定 | `.claude/settings.json`（プロジェクトフォルダ内） | そのプロジェクトのみ |

ユーザー設定は個人の好みを、プロジェクト設定はプロジェクト固有のルールを設定するのに使います。

### 基本的な書き方

JSON は、波括弧 `{}` の中に「キー（項目名）」と「値（設定内容）」をコロン `:` でつないで書く形式です。

```json
{
  "permissions": {
    "allow": [
      "Read",
      "Glob",
      "Grep"
    ],
    "deny": [
      "Bash(rm -rf *)",
      "Bash(sudo *)"
    ]
  }
}
```

上の例では、「Read、Glob、Grep は常に許可する」「rm -rf（一括削除）と sudo（管理者権限）は常に拒否する」という設定をしています。

### よく使う設定項目

#### 1. パーミッション設定（permissions）

最もよく使う設定です。どの操作を自動許可し、どの操作を禁止するかを指定します。

```json
{
  "permissions": {
    "allow": [
      "Read",
      "Edit",
      "Write",
      "Glob",
      "Grep",
      "Bash(git *)",
      "Bash(ls *)"
    ],
    "deny": [
      "Bash(rm -rf *)",
      "Bash(sudo *)",
      "Bash(curl * | bash)"
    ]
  }
}
```

**allow（許可リスト）に追加すると便利なもの：**
- `Read` — ファイル読み取り（確認なしで自動実行）
- `Edit` — ファイル編集
- `Write` — ファイル作成
- `Glob` — ファイル名検索
- `Grep` — ファイル内容検索
- `Bash(git *)` — Git 関連のコマンド
- `Bash(ls *)` — フォルダの中身を一覧表示

**deny（拒否リスト）に追加すべきもの：**
- `Bash(rm -rf *)` — ファイルの一括削除
- `Bash(sudo *)` — 管理者権限でのコマンド実行
- `Bash(curl * | bash)` — 外部スクリプトのダウンロードと実行

#### 2. 環境変数（env）

Claude Code に環境変数を渡す設定です。

```json
{
  "env": {
    "CLAUDE_CODE_MAX_OUTPUT_TOKENS": "16000"
  }
}
```

#### 3. MCP サーバー設定（mcpServers）

MCP（Model Context Protocol）サーバーとの連携を設定します。これは外部ツールやサービスとの接続に使います。中級の段階ではまだ使う機会は少ないかもしれませんが、設定できることを知っておくと将来役立ちます。

### 設定画面からの変更

settings.json を直接編集する以外に、Claude Code の設定画面（UI）からも変更できます。

```
/config
```

と入力すると、設定画面が開きます。ここからグラフィカルに設定を変更でき、変更内容は自動的に settings.json に反映されます。

JSON の書き方に慣れていない方は、まず `/config` コマンドからの設定変更をおすすめします。

### 設定の優先順位

複数の場所に settings.json がある場合、以下の優先順位で適用されます。

1. **プロジェクト設定**（最優先）
2. **ユーザー設定**

プロジェクト設定にある項目はユーザー設定よりも優先されます。

## やってみよう（ハンズオン）

### ステップ1：現在の設定を確認する

まず、現在の設定内容を確認しましょう。

```
/config
```

設定画面が開いたら、どんな項目が設定されているか見てみましょう。

### ステップ2：パーミッション設定を追加する

よく使う操作を自動許可する設定を追加してみましょう。Claude Code に以下のように依頼します。

```
~/.claude/settings.json を読んで、現在の設定内容を教えてください。その上で、Read、Glob、Grep を常に許可する設定を追加してください。
```

Claude Code が settings.json を更新してくれます。更新内容を確認しましょう。

### ステップ3：許可設定の効果を確認する

設定を追加したら、効果を確認してみましょう。

```
デスクトップのファイルを一覧表示してください。
```

以前は確認ダイアログが出ていた操作が、自動で実行されるようになっているはずです。

### ステップ4：危険な操作の拒否設定を追加する

安全のために、拒否設定も追加しましょう。

```
settings.json の deny リストに Bash(rm -rf *) と Bash(sudo *) を追加してください。
```

これにより、危険なコマンドは Claude Code が実行しようとしてもブロックされるようになります。

## まとめ
- **settings.json** は Claude Code の動作を厳格にコントロールする設定ファイル
- **CLAUDE.md** は「参考にしてもらうルール」、**settings.json** は「強制するルール」という違いがある
- **permissions** 設定で、自動許可（allow）と自動拒否（deny）を設定できる
- `/config` コマンドを使えば、設定画面からグラフィカルに変更できる
- よく使う読み取り系の操作を allow に、危険な操作を deny に設定すると、安全かつ快適に使える
