# パーミッション設定を深く理解する

## この記事で学ぶこと
- パーミッションモードの全体像と各モードの特徴を理解する
- 自分の業務スタイルに合ったモードを選べるようになる
- 段階的にパーミッションを緩和していく方法を身につける

## 前提知識
- [settings.json で自分好みにカスタマイズ](11_settings_json.md)

## 本文

### パーミッションモードとは

パーミッション（permission）とは「許可」という意味です。Claude Code には、操作を実行する前にどの程度の確認を求めるかを決める「パーミッションモード」が用意されています。

これは自動車の運転支援システムにたとえるとわかりやすいかもしれません。

| モード | たとえ |
|--------|------|
| 確認モード（default） | すべての操作で運転手に確認を取る |
| 編集自動承認モード（acceptEdits） | ハンドル操作は自動、急ブレーキは確認 |
| 計画モード（plan） | ナビが経路を提案するだけで、運転手が操作 |
| 自動モード（auto） | ほぼ自動運転。危険時のみ手動に切り替え |
| 権限省略モード（bypassPermissions） | 完全自動運転。人間の介入なし |

### 各モードの詳細

#### 確認モード（default）— 初心者におすすめ

Claude Code の初期設定です。ファイルの読み取り以外のすべての操作（ファイル編集、コマンド実行など）で確認ダイアログが表示されます。

**メリット：**
- すべての操作を事前に確認できるので、最も安全
- Claude Code が何をしようとしているかを学べる

**デメリット：**
- 確認ダイアログが頻繁に表示されるため、作業が中断される
- 大量のファイルを処理する際に手間がかかる

#### 編集自動承認モード（acceptEdits）— 中級者におすすめ

ファイルの編集（Edit、Write）は自動で承認し、ターミナルコマンド（Bash）の実行だけ確認を求めるモードです。

**メリット：**
- ファイル操作の流れが中断されないので作業が効率的
- コマンド実行はまだ確認が入るので安全性も確保

**デメリット：**
- ファイルの変更を事前に確認できない（事後に /diff で確認する必要がある）

**切り替え方法：**
```
Shift+Tab を1回押す
```

#### 計画モード（plan）— 慎重に進めたいとき

読み取り専用モードです。ファイルの分析と計画立案はできますが、実際の変更は行いません。

**メリット：**
- ファイルが一切変更されないので安全
- 方針を検討する段階で有用

**切り替え方法：**
```
Shift+Tab を2回押す、または /plan コマンド
```

#### 自動モード（auto）— 長時間タスク向け

安全分類器（セーフティクラシファイア）が各操作のリスクを自動判定し、安全な操作は自動で実行、危険な操作はブロックするモードです。

**仕組み：**
1. Claude Code が操作を実行しようとする
2. 別の AI（安全分類器）がその操作のリスクを瞬時に判定
3. 安全と判定されれば自動実行、危険と判定されれば停止

**自動でブロックされる操作の例：**
- 大量のファイル削除
- 外部スクリプトのダウンロードと実行
- 本番環境へのデプロイ
- 管理者権限が必要なコマンド

**利用条件：**
- Team、Enterprise、または API プランが必要（Pro/Max では利用不可）
- 設定画面で事前に有効化する必要がある

#### 権限省略モード（bypassPermissions）— 隔離環境専用

すべての確認を省略し、完全に自律で動作するモードです。安全チェックも行われません。

**重要な注意：** このモードは、仮想マシン（VM）やコンテナなど、隔離された環境でのみ使用することが強く推奨されています。通常の業務パソコンでは使わないでください。

### 段階的にパーミッションを緩和するおすすめの進め方

Claude Code を使い始めたばかりの方には、以下の段階的なアプローチをおすすめします。

**ステップ1：まず確認モード（default）で慣れる**

最初の1〜2週間は確認モードで使い、Claude Code がどんな操作をしようとするかを観察しましょう。

**ステップ2：安全な操作を settings.json で自動許可する**

慣れてきたら、安全とわかった操作を settings.json の allow リストに追加します。

```json
{
  "permissions": {
    "allow": [
      "Read",
      "Edit",
      "Write",
      "Glob",
      "Grep",
      "Bash(git status)",
      "Bash(git diff *)",
      "Bash(ls *)"
    ],
    "deny": [
      "Bash(rm -rf *)",
      "Bash(sudo *)"
    ]
  }
}
```

**ステップ3：編集自動承認モード（acceptEdits）に移行する**

ファイル操作に慣れたら、acceptEdits モードに切り替えます。ファイルの変更は自動承認されるので作業がスムーズになります。

**ステップ4：必要に応じてモードを切り替える**

作業の性質によってモードを使い分けましょう。

- 通常の作業 → acceptEdits
- 慎重に進めたい作業 → plan → default
- 大量処理 → （Team/Enterprise なら）auto

### セッション中のモード確認と切り替え

現在のパーミッションモードは、画面下部のステータスバーに表示されています。`Shift+Tab` を押すたびに `default` → `acceptEdits` → `plan` と順番に切り替わります。

## やってみよう（ハンズオン）

### ステップ1：パーミッションモードを切り替えてみる

`Shift+Tab` を押して、モードが切り替わる様子を確認しましょう。

1回目：`default` → `acceptEdits`（画面下部の表示が変わる）
2回目：`acceptEdits` → `plan`
3回目：`plan` → `default`

### ステップ2：確認モードでの操作を体験する

まず、`default` モードに戻してください。

```
デスクトップに permission_test.txt というファイルを作成してください。内容は「パーミッションテスト」としてください。
```

確認ダイアログが表示されることを確認してください。「Allow」を押して実行します。

### ステップ3：編集自動承認モードでの操作を体験する

`Shift+Tab` を1回押して `acceptEdits` モードに切り替えてから、同じような操作を試みます。

```
permission_test.txt の内容を「パーミッションテスト — acceptEdits モードで変更しました」に更新してください。
```

今度は確認ダイアログが表示されず、自動でファイルが変更されるはずです。

変更後、`/diff` で内容を確認しましょう。

```
/diff
```

### ステップ4：計画モードでの操作を体験する

`Shift+Tab` をもう1回押して `plan` モードに切り替えます。

```
permission_test.txt の内容をさらに変更してください。
```

計画モードでは、変更内容が「提案」として表示されるだけで、実際のファイルは変更されません。

確認したら、`Shift+Tab` で `default` モードに戻しておきましょう。

## まとめ
- Claude Code には **5つのパーミッションモード** があり、作業の安全性と効率のバランスを調整できる
- 初心者は **default（確認モード）** から始め、慣れたら **acceptEdits（編集自動承認モード）** に移行するのがおすすめ
- **settings.json** の allow/deny リストで、個別の操作ごとに許可・拒否を設定できる
- **Shift+Tab** でモードを素早く切り替えられる
- 段階的にパーミッションを緩和していくことで、安全性を保ちながら効率を上げられる
