# 計画モードで大きなタスクに取り組む

## この記事で学ぶこと
- 計画モード（Plan mode）の仕組みと目的を理解する
- 大きなタスクを安全に進めるための考え方を身につける
- /plan コマンドと /ultraplan コマンドの使い方を覚える

## 前提知識
- [CLAUDE.md 実践 — 効果的なルール設計](05_claude_md_advanced.md)

## 本文

### 計画モードとは

計画モード（Plan mode）は、Claude Code を「読み取り専用」にして、ファイルの変更やコマンドの実行を一切行わないモードです。代わりに、タスクの分析と作業計画の立案だけを行います。

たとえるなら、建物を建てる前に設計図を作る工程です。いきなり工事を始めるのではなく、まず全体像を把握し、手順を決めてから着手することで、手戻りを防ぎ、効率よく作業を進められます。

### なぜ計画が重要なのか

小さなタスク（「この文書の誤字を直して」など）は、計画なしですぐに実行しても問題ありません。しかし、以下のような場合は計画を立てることが重要です。

- **複数のファイルにまたがる作業**：50件のCSVデータを集計してレポートにまとめる
- **手順が複雑な作業**：データの変換→集計→グラフ作成→報告書にまとめる
- **影響範囲が大きい作業**：フォルダ構成の変更や大量ファイルのリネーム
- **やり直しが難しい作業**：データの一括変換や削除を伴う作業

計画を先に確認することで、「思っていたのと違う」という事態を未然に防げます。

### /plan コマンドの使い方

計画モードに切り替える方法は2つあります。

**方法1：/plan コマンドを使う**

```
/plan 月次売上レポートを作成するための手順を考えてください
```

`/plan` に続けてタスクの内容を書くと、Claude Code はファイルを分析し、作業の手順を計画として提示します。

**方法2：Shift+Tab でモードを切り替える**

`Shift+Tab` キーを押すと、パーミッションモードが `default` → `acceptEdits` → `plan` と順番に切り替わります。`plan` モードに切り替えたあとに指示を出すと、実行ではなく計画だけが返ってきます。

### 計画の確認と実行

Claude Code が提示した計画には、通常以下の情報が含まれます。

1. **タスクの理解**：何を求められているかの解釈
2. **現状の分析**：関連ファイルの確認結果
3. **作業ステップ**：具体的な手順の一覧
4. **注意点**：想定されるリスクや確認事項

計画を確認して問題がなければ、モードを `default` や `acceptEdits` に戻してから「この計画を実行してください」と指示します。修正が必要であれば「ステップ3はこうしてほしい」と伝えて、計画を調整できます。

### /ultraplan — より大規模なタスクの計画

特に大きなタスク（たとえば「過去1年分のデータを全部整理し直す」など）には `/ultraplan` コマンドが便利です。

```
/ultraplan 年度末のデータ整理と来期の準備作業を計画してください
```

`/ultraplan` を使うと、より詳細で構造化された計画がブラウザ上で表示されます。計画の各ステップを確認・修正してから実行に移せるため、大規模な作業でも安心して進められます。

### 計画モードの活用パターン

**パターン1：見積もりを取る**

```
/plan このフォルダ内のPDFファイルをすべて読み取ってCSVに変換するとしたら、どれくらいの作業量になりますか？
```

実行する前に作業量を把握できるので、「思ったより大変だった」という事態を避けられます。

**パターン2：リスクを事前に確認する**

```
/plan このフォルダの構成を変更したいのですが、影響を受けるファイルと注意点を教えてください
```

変更を加える前に影響範囲を確認できます。

**パターン3：複数案を比較する**

```
/plan 売上データの集計方法として、以下の2パターンを比較検討してください。
A案：月別に集計
B案：四半期ごとに集計
```

それぞれの利点と欠点を整理してもらえます。

## やってみよう（ハンズオン）

### ステップ1：計画モードを体験する

まず、計画モードの基本を体験しましょう。

```
/plan デスクトップの bash_test フォルダ内のすべてのCSVファイルを読み込み、全店舗の売上を集計した年間サマリーレポートを作成する手順を計画してください。
```

Claude Code が提示する計画を読んでみてください。どんなファイルを読み、どんな順序で処理し、どんなファイルを作成するかが示されるはずです。

### ステップ2：計画を修正する

提示された計画に対して、修正を依頼してみましょう。

```
計画に以下の要件を追加してください。
- 集計結果にはグラフ用のデータも含めること
- 最終レポートは Markdown 形式にすること
- 作業前にバックアップを取ること
```

計画が更新されることを確認してください。

### ステップ3：計画を実行する

計画に納得したら、モードを切り替えて実行しましょう。

`Shift+Tab` で `default` モードに戻してから、次のように指示します。

```
先ほどの計画を実行してください。
```

Claude Code が計画に沿って、ステップバイステップで作業を進めてくれます。

## まとめ
- **計画モード** は、Claude Code を読み取り専用にして作業計画だけを立てるモード
- 複雑なタスクや影響範囲が大きい作業では、まず計画を立てることで手戻りを防げる
- `/plan` で基本的な計画を、`/ultraplan` で大規模な計画を立てられる
- 計画を確認・修正してから実行に移すことで、安全かつ効率的に作業を進められる
