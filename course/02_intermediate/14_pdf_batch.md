## 本文

### ビジネスシナリオ

あなたは、経理部門で取引先からの請求書処理を担当しています。毎月、20〜50件の請求書がPDFで届き、そこから取引先名・請求金額・請求日・振込期限などの情報を手作業で経理システムに入力しています。

この入力作業には毎月約8時間かかり、さらに転記ミスのリスクもあります。Claude Code を使えば、PDF からの情報抽出を自動化し、この作業を大幅に効率化できます。

### Claude Code の PDF 読み取り機能

Claude Code は、PDF ファイルの内容を直接読み取ることができます。テキストが含まれた PDF はもちろん、スキャンされた画像形式の PDF も読み取れます（ただし、画像形式の場合は読み取り精度が下がることがあります）。

**対応しているPDFの種類：**
- テキストベースの PDF（パソコンで作成された請求書など）
- スキャンされた PDF（紙の書類をスキャンしたもの）
- 画像が含まれた PDF（ロゴや表が入っている請求書など）

**注意点：**
- 大きな PDF（10ページ以上）は、ページ範囲を指定して読み込む必要があります
- 手書き文字の読み取りは精度が低くなる場合があります

### PDF 一括処理の進め方

複数の PDF を処理する場合、以下の流れが効果的です。

1. **フォルダにPDFをまとめる**：処理対象のPDFを1つのフォルダに集める
2. **1件で試す**：まず1件のPDFで抽出がうまくいくか確認する
3. **抽出項目を確定する**：どの情報を取り出すかを明確にする
4. **一括処理を実行する**：全件まとめて処理する
5. **結果を検証する**：抽出結果に誤りがないか確認する

### 抽出精度を上げるコツ

**1. 抽出項目を具体的に指定する**

```
悪い例：「請求書の情報を取り出して」
良い例：「各PDFから以下の項目を抽出してください。
- 請求書番号
- 取引先名（正式名称）
- 請求日（YYYY/MM/DD形式）
- 請求金額（税込、数値のみ）
- 振込期限（YYYY/MM/DD形式）
- 振込先口座情報」
```

**2. 出力形式を統一する**

```
「抽出結果はCSVファイルに出力してください。
- 1行目はヘッダー：請求書番号,取引先名,請求日,請求金額,振込期限
- 日付は YYYY/MM/DD 形式
- 金額は半角数字のみ（カンマなし）
- 文字コードは UTF-8（BOM付き）」
```

**3. 不明な項目の扱いを決めておく**

```
「PDFから該当項目が読み取れない場合は "不明" と記入してください」
```

### 結果の検証方法

PDFからの自動抽出は便利ですが、100%の精度は保証されません。特に以下のケースでは注意が必要です。

- 手書きの記載がある場合
- 印刷が不鮮明な場合
- 通常と異なるレイアウトの請求書
- 外国語が混在する場合

抽出結果は必ず目視で確認し、特に金額と日付については重点的にチェックしましょう。

## やってみよう（ハンズオン）

### ステップ1：練習用の請求書データを作成する

実際のPDFは用意が難しいので、テキスト形式の模擬請求書を作って練習しましょう。

```
デスクトップに pdf_practice というフォルダを作成し、その中に invoices というサブフォルダを作ってください。

invoices フォルダ内に、以下の5件の請求書ファイル（テキスト形式）を作成してください。実際の業務ではPDFですが、練習用にテキストファイルで模擬します。

1. invoice_001.txt — 株式会社ABC商事、請求金額 550,000円（税込）、請求日 2026/03/15、振込期限 2026/04/30、請求書番号 INV-2026-001
2. invoice_002.txt — 有限会社デザインワークス、請求金額 330,000円（税込）、請求日 2026/03/18、振込期限 2026/04/30、請求書番号 INV-2026-002  
3. invoice_003.txt — 山田コンサルティング合同会社、請求金額 1,100,000円（税込）、請求日 2026/03/20、振込期限 2026/05/15、請求書番号 INV-2026-003
4. invoice_004.txt — 株式会社テクノソリューションズ、請求金額 275,000円（税込）、請求日 2026/03/22、振込期限 2026/04/30、請求書番号 INV-2026-004
5. invoice_005.txt — グローバルサプライ株式会社、請求金額 825,000円（税込）、請求日 2026/03/25、振込期限 2026/05/15、請求書番号 INV-2026-005

各ファイルは実際の請求書に近いフォーマット（レイアウト）で作成してください。会社の住所、電話番号、振込先口座なども含めてリアルなものにしてください。
```

### ステップ2：1件で試す

まず、1件だけ情報抽出を試してみましょう。

```
pdf_practice/invoices/invoice_001.txt を読んで、以下の情報を抽出してください。

- 請求書番号
- 取引先名
- 請求日
- 請求金額（税込）
- 振込期限

抽出結果を表形式で表示してください。
```

正しく抽出されているか確認してください。

### ステップ3：全件を一括処理する

1件で問題なければ、全件をまとめて処理しましょう。

```
pdf_practice/invoices/ フォルダ内のすべての請求書ファイルを読み込んで、以下の情報を抽出し、CSVファイルにまとめてください。

抽出項目：
- 請求書番号
- 取引先名
- 請求日（YYYY/MM/DD形式）
- 請求金額（税込、数値のみ、カンマなし）
- 振込期限（YYYY/MM/DD形式）

出力ファイル名：pdf_practice/invoice_summary.csv
文字コード：UTF-8（BOM付き、Excelで開けるように）
```

### ステップ4：集計レポートを作成する

抽出したデータをもとに、集計レポートも作ってもらいましょう。

```
invoice_summary.csv のデータをもとに、以下のレポートを pdf_practice/payment_schedule.md として作成してください。

- 請求金額の合計
- 振込期限別の支払い予定金額（4月末期限と5月中旬期限に分けて）
- 金額が大きい順の請求書一覧
- 支払いスケジュールの要約
```

### ステップ5：突合チェックの練習

実際の業務では、抽出したデータと社内の発注データを突き合わせることも多いです。

```
pdf_practice フォルダに purchase_orders.csv を作成してください。以下の発注データを含めてください。

発注番号,取引先名,発注金額,発注日
PO-001,株式会社ABC商事,500000,2026/03/01
PO-002,有限会社デザインワークス,330000,2026/03/05
PO-003,山田コンサルティング合同会社,1000000,2026/03/10
PO-004,株式会社テクノソリューションズ,275000,2026/03/12
PO-005,グローバルサプライ株式会社,825000,2026/03/15

次に、invoice_summary.csv と purchase_orders.csv を突き合わせて、請求金額と発注金額に差異がある取引先を特定し、差異レポートを作成してください。
```

## まとめ
- Claude Code は PDF を直接読み取り、必要な情報を自動抽出できる
- 一括処理の前に必ず1件で試し、抽出結果を確認してから全件処理に進める
- 抽出項目と出力形式を具体的に指定すると、精度が高くなる
- 抽出結果は必ず目視で検証する（特に金額と日付）
- 請求書処理は月8時間→30分程度に短縮できる可能性があり、転記ミスも大幅に減る
