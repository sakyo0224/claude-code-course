- 基本的な CLAUDE.md を書けるようになる

## 本文

### CLAUDE.md とは何か

CLAUDE.md は、Claude Code に対して「このプロジェクトではこういうルールで作業してください」と伝えるための設定ファイルです。

たとえるなら、新入社員に渡す「引き継ぎ資料」のようなものです。一度書いておけば、Claude Code を起動するたびに最初から説明し直す必要がなくなります。

CLAUDE.md がない場合、Claude Code は毎回まっさらな状態でスタートします。「日本語で回答して」「ですます調で書いて」といったルールも、毎回伝え直す必要があります。CLAUDE.md があれば、こうしたルールは自動的に適用されます。

### CLAUDE.md の仕組み

CLAUDE.md は、Claude Code が起動するときに最初に読み込まれます。読み込まれた内容は会話全体を通じて保持され、すべての回答やファイル操作に影響します。

ファイル形式は Markdown（マークダウン）という、テキストに簡単な装飾をつけられる形式です。特別なソフトは不要で、メモ帳やテキストエディタで編集できます。

### CLAUDE.md の配置場所

CLAUDE.md は、用途に応じて3つの場所に置くことができます。

| 配置場所 | 適用範囲 | 使いどころ |
|---------|---------|----------|
| `~/.claude/CLAUDE.md` | すべてのプロジェクト共通 | 「常に日本語で回答して」など、個人の好み |
| プロジェクトのルートフォルダ | そのプロジェクト内全体 | プロジェクト固有のルール |
| サブフォルダ内 | そのフォルダ内の作業のみ | フォルダ固有の細かいルール |

`~` はホームディレクトリ（ユーザーフォルダ）を意味します。macOS なら `/Users/あなたの名前/`、Windows なら `C:\Users\あなたの名前\` です。

**優先順位のポイント：** 複数の CLAUDE.md が存在する場合、作業中のフォルダに近いものほど優先度が高くなります。たとえば、グローバル設定で「英語で書いて」と設定し、プロジェクトの CLAUDE.md で「日本語で書いて」と設定した場合、そのプロジェクト内では日本語が優先されます。

### 基本的な書き方

CLAUDE.md に書く内容は、自由形式のテキストです。ただし、効果的に機能させるために、以下のような項目を含めるとよいでしょう。

**最低限書いておくとよい内容：**

1. **回答の言語とスタイル**：「日本語で回答する」「丁寧な文体（です・ます調）にする」
2. **プロジェクトの概要**：「このフォルダは月次報告書の作成に使う」
3. **ファイルの命名規則**：「ファイル名は YYYYMMDD_内容.拡張子 にする」
4. **やってほしくないこと**：「既存のファイルを許可なく削除しない」

### CLAUDE.md のテンプレート

以下は、業務利用で使える基本テンプレートです。

```markdown
# プロジェクトルール

## 基本方針
- 回答は必ず日本語で行うこと
- 丁寧な文体（です・ます調）を使うこと
- 専門用語を使う場合はわかりやすい説明を添えること

## プロジェクト概要
- このフォルダは〇〇業務で使用するファイルを管理しています
- 主なファイル形式：CSV、Excel、PDF

## ファイル操作のルール
- 新しいファイルを作るときは、ファイル名を「YYYYMMDD_内容.拡張子」の形式にすること
- 既存のファイルを削除する前に必ず確認すること
- バックアップは backup/ フォルダに保存すること

## 注意事項
- 個人情報を含むファイルの内容を会話の中に表示しないこと
- 大量のファイル操作を行う前に、作業計画を提示すること
```

### /init コマンドで自動生成

Claude Code には、CLAUDE.md を自動生成してくれるコマンドがあります。

```
/init
```

と入力すると、現在のプロジェクトの構成を分析し、基本的な CLAUDE.md のひな形を作ってくれます。まずはこれで生成してから、自分好みにカスタマイズするのがおすすめです。

## やってみよう（ハンズオン）

### ステップ1：グローバル CLAUDE.md を作成する

まず、すべてのプロジェクトに共通で適用されるルールを設定しましょう。

```
~/.claude/ フォルダに CLAUDE.md を作成してください。内容は以下のとおりです。

- 回答は必ず日本語で行うこと
- 丁寧な文体（です・ます調）を使うこと
- 提案は箇条書きでまとめること
```

### ステップ2：プロジェクト用 CLAUDE.md を作成する

次に、特定のフォルダ用のルールを設定します。デスクトップの bash_test フォルダ（前回作成したもの）を使います。

```
デスクトップの bash_test フォルダに CLAUDE.md を作成してください。内容は以下のとおりです。

# bash_test プロジェクトルール

## 概要
- このフォルダは Claude Code の学習用テストデータを管理しています

## ルール
- ファイル名は日本語を使わず、英数字とアンダースコアのみにすること
- CSVファイルのヘッダー行は日本語で統一すること
- 新しいファイルを作成するときは、年度フォルダの中に保存すること
```

### ステップ3：ルールが反映されるか確認する

CLAUDE.md を作成したら、新しい会話を始めて（`/clear` を使って）ルールが反映されるか確認してみましょう。

```
/clear
```

会話をクリアした後、次のように指示してみてください。

```
bash_test フォルダに売上サンプルデータのCSVファイルを作ってください。
```

CLAUDE.md のルールどおり、英数字のファイル名で、ヘッダーが日本語のCSVファイルが年度フォルダ内に作成されるか確認してみましょう。

## まとめ
- **CLAUDE.md** は Claude Code に「ルール」や「好み」を伝えるための設定ファイル
- 一度書いておけば、起動するたびに自動で読み込まれるので毎回伝え直す必要がない
- 配置場所は3段階あり、グローバル → プロジェクト → サブフォルダの順で細かく設定できる
- `/init` コマンドを使うと、ひな形を自動生成できる
