## 本文

### ハーネスエンジニアリングとは

CLAUDE.md を効果的に設計することは、「ハーネスエンジニアリング」と呼ばれる考え方の中核をなします。

ハーネスとは、もともと「手綱」や「馬具」を意味する言葉です。馬をコントロールするための道具になぞらえて、AI エージェントの動作を安全に・正確に・自分好みにコントロールするための仕組みづくり全体を「ハーネスエンジニアリング」と呼びます。

> **補足**：ハーネスエンジニアリングの詳細（Hooks、パーミッション設計、サブエージェントの活用など）は、上級セクションで学びます。

**ハーネスエンジニアリングの構成要素：**

| 要素 | 役割 | たとえると |
|-----|------|----------|
| CLAUDE.md | AI に「何をすべきか」を教える | 業務マニュアル |
| settings.json | AI に「何をしていいか」を決める | 権限設定 |
| Hooks | AI の動作を「いつチェックするか」を仕込む | 品質チェック |
| パーミッション | AI の「行動範囲」を制限する | ガードレール |

この記事では、中でも最も使用頻度が高く効果的な CLAUDE.md の設計を深掘りします。

### 効果的な CLAUDE.md の5つの原則

#### 原則1：具体的に書く

あいまいな指示よりも、具体的な指示のほうが Claude Code は正確に動いてくれます。

```markdown
# 悪い例
- きれいな文章を書くこと

# 良い例
- 一文は60文字以内を目安にすること
- 箇条書きは5項目以内にまとめること
- 漢字の使用率は30%程度に抑えること
```

#### 原則2：やるべきことと、やってはいけないことの両方を書く

「やってほしいこと」だけでなく「やってほしくないこと」も明記すると、想定外の動作を防げます。

```markdown
## やるべきこと
- ファイルを作成する前に、作成計画を提示すること
- データの加工が完了したら、結果の概要を報告すること

## やってはいけないこと
- 元のデータファイルを上書きしないこと（必ず別ファイルとして保存）
- 個人情報を含むデータの内容を会話に出力しないこと
- 一度に10個以上のファイルを同時に作成しないこと
```

#### 原則3：作業手順を定義する

繰り返し行う作業がある場合、手順を CLAUDE.md に書いておくと、毎回同じ品質で作業してもらえます。

```markdown
## 月次レポート作成手順
1. data/ フォルダから当月のCSVファイルをすべて読み込む
2. 売上合計、前月比、前年同月比を計算する
3. 結果を reports/YYYYMM_monthly_report.csv として保存する
4. 要約コメントを reports/YYYYMM_summary.txt として保存する
5. 作成したファイルの一覧と概要を報告する
```

#### 原則4：フォルダ構成を説明する

プロジェクトのフォルダ構成を記載しておくと、Claude Code がどこに何があるかを迷わず判断できます。

```markdown
## フォルダ構成
- data/ — 元データ（CSV、Excel）を保存する場所。手を加えない
- reports/ — 生成したレポートの保存先
- templates/ — テンプレートファイルの保存先
- backup/ — バックアップの保存先
- temp/ — 一時ファイル。作業終了後に削除してよい
```

#### 原則5：出力フォーマットを指定する

レポートや表の形式を指定しておくと、毎回統一されたフォーマットで出力してもらえます。

```markdown
## レポートのフォーマット
- ファイル形式：CSV（UTF-8、BOM付き）
- 日付の表記：YYYY年MM月DD日
- 金額の表記：3桁ごとにカンマ区切り、円記号を先頭に付ける
- ヘッダー行：日本語で統一
- 小数点：第2位まで表示
```

### よくある失敗パターン

#### 失敗1：ルールが多すぎる

CLAUDE.md に何十ものルールを詰め込むと、重要なルールが埋もれてしまうことがあります。本当に大切なルールを5〜10個に絞りましょう。

#### 失敗2：矛盾するルールがある

「簡潔に書くこと」と「詳しく説明すること」のように、矛盾するルールがあると Claude Code が混乱します。書いたら一度全体を見直して、矛盾がないか確認しましょう。

#### 失敗3：更新しない

プロジェクトの進行とともに、必要なルールも変わります。定期的に CLAUDE.md を見直し、不要なルールを削除したり新しいルールを追加したりしましょう。

### 業務別テンプレート集

**経理・会計チーム向け：**

```markdown
# 経理業務ルール

## 基本方針
- 金額は必ず半角数字で、3桁カンマ区切り
- 日付は YYYY/MM/DD 形式で統一
- 消費税率は10%で計算（軽減税率8%の場合は明記）

## データ処理ルール
- 元データ（CSVやExcel）は絶対に上書きしない
- 加工後のデータは processed/ フォルダに保存
- ファイル名に処理日を含める（例：20260406_processed_sales.csv）
```

**営業チーム向け：**

```markdown
# 営業業務ルール

## 基本方針
- 顧客名は正式名称を使用（株式会社は省略しない）
- 個人名は姓のみ記載し、フルネームは出力しない

## メール作成ルール
- 件名は20文字以内で要件がわかるようにする
- 敬語は「貴社」「ご担当者様」を使用
- 本文は300文字以内を目安にまとめる
```

### ステップ1：現在の CLAUDE.md を改善する

前回作成した bash_test フォルダの CLAUDE.md を、本記事の原則に基づいて改善しましょう。

```
bash_test フォルダの CLAUDE.md を読んで、今の内容に以下を追加してください。

## フォルダ構成
- 2024年/ — 2024年度のデータ
- 2025年/ — 2025年度のデータ
- 2026年/ — 2026年度のデータ（現在の作業対象）
- backup/ — バックアップ保存先

## 作業ルール
- データ加工前に必ず元ファイルを backup/ にコピーすること
- 新規ファイルの作成前に、作成するファイルの概要を報告すること
- 一度に3つ以上のファイルを作成する場合は、作業計画を先に提示すること

## やってはいけないこと
- 元データの上書き
- backup/ フォルダ内のファイルの削除
```

### ステップ2：ルールが機能するか検証する

会話をクリアしてから、ルールが正しく動くか確認しましょう。

```
/clear
```

```
bash_test フォルダに、以下の4つのCSVファイルを作成してください。
- 第1四半期の売上データ
- 第2四半期の売上データ  
- 第3四半期の売上データ
- 第4四半期の売上データ
```

CLAUDE.md のルールにより、Claude Code は「3つ以上のファイルを作成するので、まず作業計画を提示します」と報告してくるはずです。

### ステップ3：/init で自動生成を体験する

別のフォルダで `/init` コマンドを試してみましょう。

```
デスクトップに init_test というフォルダを作って、その中にサンプルのCSVファイルとテキストファイルをいくつか作成してください。そのあと、そのフォルダで /init を実行して CLAUDE.md を自動生成してみましょう。
```

自動生成された CLAUDE.md の内容を確認し、必要に応じて自分のルールを追加してみてください。

## まとめ
- 「ハーネスエンジニアリング」とは、AI の動作を安全にコントロールするための仕組みづくりのこと
- 効果的な CLAUDE.md の5つの原則：具体的に書く、禁止事項も書く、作業手順を定義する、フォルダ構成を説明する、出力フォーマットを指定する
- ルールは多すぎず（5〜10個）、矛盾なく、定期的に見直すことが大切
- `/init` コマンドでひな形を自動生成し、そこからカスタマイズするのが効率的
