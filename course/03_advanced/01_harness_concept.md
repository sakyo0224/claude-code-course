## 本文

### ハーネスとは何か

AI エージェントの世界で「ハーネス（Harness）」とは、**エージェントの動作を制御するインフラ全体**のことを指します。馬に乗るときの手綱をイメージしてください。手綱があるから、馬を安全にコントロールできます。AI エージェントにも同じ仕組みが必要です。

公式的には、次のように定式化されます。

> **コーディングエージェント ＝ AI モデル ＋ ハーネス**

つまり、Claude のような AI モデルが「頭脳」だとすると、ハーネスは「体制・ルール・安全装置」にあたります。どんなに優秀な頭脳でも、適切な制御なしに動かせば事故が起きます。ハーネスエンジニアリングとは、この制御の仕組みを設計・構築する技術です。

HashiCorp 創業者の Mitchell Hashimoto 氏は、こう表現しています。

> 「エージェントがミスをするたびに、そのミスを二度と繰り返さないような仕組みをエンジニアリングすること」

### プロンプトエンジニアリングとの違い

プロンプトエンジニアリングとハーネスエンジニアリングは、よく混同されますが、対象とする範囲が異なります。

| 観点 | プロンプトエンジニアリング | ハーネスエンジニアリング |
|------|------------------------|----------------------|
| **対象** | 1 回の入力（プロンプト） | エージェントを取り巻くシステム全体 |
| **範囲** | 「何を聞くか」の最適化 | ツール管理、権限制御、エラー回復、可観測性 |
| **比喩** | 「右に曲がれ」という指示 | 道路、ガードレール、標識、信号システム全体 |
| **持続性** | 1 セッション内 | セッションをまたいで永続 |
| **決定論性** | 確率的（モデルの応答に依存） | 決定論的（設定ファイルで強制） |

プロンプトエンジニアリングはハーネスエンジニアリングの**一部**です。良いプロンプトは重要ですが、それだけでは長期的に安定した運用はできません。

### Claude Code におけるハーネスの構成要素

Claude Code では、以下の 7 つの要素がハーネスを構成しています。

| 構成要素 | 役割 | 具体例 |
|---------|------|--------|
| **CLAUDE.md** | プロジェクト固有のコンテキストと指示 | コーディング規約、テスト手順の記述 |
| **settings.json** | 決定論的な動作の強制 | パーミッション（allow/deny）、モデル選択 |
| **Hooks** | ライフサイクルの各ポイントで自動実行されるスクリプト | 編集前にリント実行、コミット前にテスト実行 |
| **Permission settings** | 操作の許可/拒否/確認のルール | `allow: ["Bash(git *)"]` |
| **Sub-agents** | タスクの分離とコンテキスト管理 | レビュー担当、テスト担当の役割分担 |
| **Skills** | 必要に応じて知識とツールを段階的に開示 | ドキュメント処理スキルの追加 |
| **MCP サーバー** | 外部サービスとの接続 | Slack、GitHub、Notion との連携 |

### なぜ今ハーネスエンジニアリングが重要なのか

AI エージェントの能力が向上するにつれ、「何でもできてしまう」ことがリスクになります。ハーネスエンジニアリングは、このリスクを管理しながらエージェントの能力を最大限に引き出すための技術です。

特に Claude Code では、ファイルの読み書き、コマンドの実行、Git 操作など、システムに直接影響する操作が可能です。適切なハーネスなしに Auto モードで動かすと、意図しないファイル削除やデータ流出が起こりえます。逆に、ハーネスを正しく設計すれば、安全に自走させることができ、生産性が飛躍的に向上します。

## やってみよう（ハンズオン）

### ステップ 1：自分のプロジェクトのハーネスを棚卸しする

まず、現在のプロジェクトでどのハーネス要素が設定されているかを確認しましょう。

```bash
# プロジェクトルートに移動して確認
ls -la CLAUDE.md .claude/settings.json .claude/ 2>/dev/null
```

### ステップ 2：ハーネスチェックリストを作成する

以下のチェックリストで、自分のプロジェクトの状態を確認してください。

- [ ] CLAUDE.md にプロジェクトのルールを記述しているか
- [ ] settings.json で allow/deny パーミッションを設定しているか
- [ ] 危険な操作（`rm -rf *`, `sudo *`）を deny に追加しているか
- [ ] Hooks で自動チェック（リント、テスト）を設定しているか
- [ ] 必要な MCP サーバーを接続しているか

### ステップ 3：最小限のハーネスを構築する

まだハーネスを設定していない場合は、以下の最小構成から始めましょう。

```json
// .claude/settings.json
{
  "permissions": {
    "allow": [
      "Read",
      "Edit",
      "Write",
      "Glob",
      "Grep",
      "Bash(git *)",
      "Bash(ls *)"
    ],
    "deny": [
      "Bash(rm -rf *)",
      "Bash(sudo *)"
    ]
  }
}
```

この設定だけで、ファイル操作と Git コマンドは自動承認され、危険なコマンドはブロックされます。

## まとめ
- ハーネスとは、AI エージェントの動作を制御するインフラ全体のこと
- プロンプトエンジニアリングは「1 回の指示」、ハーネスエンジニアリングは「システム全体の設計」
- Claude Code のハーネスは、CLAUDE.md / settings.json / Hooks / Permissions / Sub-agents / Skills / MCP の 7 要素で構成される
- 適切なハーネスがあれば、安全にエージェントを自走させられる
