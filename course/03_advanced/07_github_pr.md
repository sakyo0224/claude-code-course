# PR 作成とコードレビュー自動化

## この記事で学ぶこと
- Claude Code を使った PR（プルリクエスト）の効率的な作成方法
- コードレビューの自動化と diff ビューの活用
- PR のライフサイクル全体を Claude Code で管理する方法

## 前提知識
- Git / GitHub 連携の基本（→ [Git / GitHub 連携の基本](06_git_basics.md)）
- GitHub CLI（`gh`）が認証済みであること

## 本文

### Claude Code による PR 作成

Claude Code は、変更内容を分析して PR を自動で作成できます。タイトル、説明文、レビュー依頼まで一貫して行えるため、PR 作成の手間が大幅に削減されます。

**基本的な PR 作成：**
```
この変更内容でPRを作成して
```

Claude Code は以下を自動で行います。
1. `git diff` で変更内容を分析
2. `git log` で関連するコミット履歴を確認
3. PR タイトルと説明文を生成
4. `gh pr create` で PR を作成

**詳細な指示を添えた PR 作成：**
```
PRを作成して。レビュアーに@tanaka を指定して、labelsに"enhancement"を追加して
```

### PR 説明文のカスタマイズ

Claude Code が生成する PR 説明文のテンプレートを CLAUDE.md で指定できます。

```markdown
# PR 作成ルール
- PRの説明文は以下のフォーマットに従うこと
  ## 概要
  （変更の目的を1-2文で）
  ## 変更内容
  （変更点を箇条書きで）
  ## テスト
  （テスト方法と結果）
  ## スクリーンショット
  （UIの変更がある場合）
```

### ビジュアル diff レビュー

Claude Code のデスクトップアプリには、変更箇所をファイル単位で確認できる**ビジュアル diff ビューア**が搭載されています。

**diff の確認：**
```
/diff
```

このコマンドで、変更されたファイルの一覧と差分が表示されます。各行にコメントを付けることもできます。

### Claude Code によるコードレビュー

Claude Code は、既存の PR に対してコードレビューを行うこともできます。

**PR のレビューを依頼する：**
```
PR #42 のコードレビューをして。セキュリティとパフォーマンスの観点でチェックして
```

Claude Code は以下の観点でレビューします。
- コードの正確性（バグの可能性）
- セキュリティ上の問題（SQL インジェクション、XSS など）
- パフォーマンスの問題（N+1 クエリ、不要なループなど）
- コードスタイルの一貫性
- テストの網羅性

**セキュリティレビュー専用のコマンド：**
```
/security-review
```

このコマンドで、現在の変更に対するセキュリティ脆弱性の分析が実行されます。

### PR のライフサイクル管理

Claude Code は PR の作成だけでなく、そのライフサイクル全体を管理できます。

**レビューコメントへの対応：**
```
PR #42 のレビューコメントに対応して修正して
```

Claude Code は以下を行います。
1. `gh pr view 42` で PR の情報を取得
2. `gh api repos/owner/repo/pulls/42/comments` でレビューコメントを取得
3. 各コメントの指摘事項を理解して修正
4. 修正をコミットしてプッシュ

**CI の結果確認と修正：**
```
PR #42 のCIが失敗しているので修正して
```

Claude Code は CI のログを確認し、エラーの原因を特定して修正します。

**PR のマージ：**
```
PR #42 をマージして
```

```bash
# Claude Code が実行するコマンド
gh pr merge 42 --squash
```

### PR 監視の自動化

Claude Code のデスクトップアプリには、PR の CI ステータスを監視する機能があります。

- CI が失敗した場合に自動で原因を分析
- 修正提案の自動生成
- レビュー承認後の自動マージ（設定により）

### 効率的な PR ワークフローの例

以下は、Claude Code を活用した一連の PR ワークフローです。

```
1. 「ユーザー登録機能を実装して」
   → Claude Code がブランチ作成 → 実装 → テスト → コミット

2. 「PRを作成して」
   → PR作成（タイトル、説明文、レビュアー指定を自動化）

3. 「PR #15 のレビューコメントに対応して」
   → レビュー指摘の修正 → 追加コミット → プッシュ

4. 「CIが通ったらマージして」
   → CI監視 → 成功後にマージ
```

### settings.json での GitHub 操作パーミッション

```json
{
  "permissions": {
    "allow": [
      "Bash(gh pr create *)",
      "Bash(gh pr view *)",
      "Bash(gh pr list *)",
      "Bash(gh pr checks *)",
      "Bash(gh pr merge *)",
      "Bash(gh api *)"
    ]
  }
}
```

## やってみよう（ハンズオン）

### ステップ 1：作業ブランチで変更を加える

```
mainから新しいブランチを作成して、READMEに機能一覧のセクションを追加して
```

### ステップ 2：PR を作成する

```
この変更内容でPRを作成して
```

作成された PR の URL を確認し、GitHub 上で内容を確認してください。

### ステップ 3：コードレビューを依頼する

作成した PR に対して、Claude Code でレビューを実行します。

```
さっき作成したPRのコードレビューをして
```

## まとめ
- Claude Code は変更内容を分析して、PR のタイトル・説明文を自動生成する
- `/diff` コマンドでビジュアルな差分確認ができる
- レビューコメントへの対応、CI 失敗の修正、マージまで一貫して管理可能
- `/security-review` でセキュリティ観点のレビューも自動化できる
- CLAUDE.md で PR 説明文のテンプレートを指定すると、チームの規約に沿った PR が作成される
