## 本文

### Auto モードとは

Auto モード（自動モード）は、Claude Code のパーミッションモードの一つで、**安全分類器（Classifier）による自動チェック付きで操作を自動実行**するモードです。ユーザーが毎回「許可」を押す必要がなくなるため、長時間のタスクや反復作業に特に有効です。

通常の `default` モードでは、ファイル編集やコマンド実行のたびに許可を求められます。これは安全ですが、複雑なタスクでは何十回も「許可」を押すことになり、「許可疲れ」が発生します。Auto モードはこの問題を解決するために、2025 年後半に導入されました。

### 安全分類器の仕組み

Auto モードの核心は、**2 段階の安全分類器**です。

**Stage 1：高速フィルター**
- すべての操作に対して、まず高速な Yes/No 判定を行います
- 明らかに安全な操作（ファイルの読み取り、`git status` の実行など）はここで自動承認されます
- 少しでも疑わしい操作は Stage 2 に送られます

**Stage 2：詳細判断（Chain-of-Thought）**
- Stage 1 でフラグされた操作に対して、思考連鎖による詳細な判断を行います
- 操作の文脈、ユーザーの意図、潜在的なリスクを総合的に評価します
- この段階で安全と判断されれば自動承認、危険と判断されればブロックされます

**プロンプトインジェクション対策**として、分類器にはユーザーメッセージとツール呼び出しのみが渡され、ツールの実行結果（外部からのデータ）は除外されます。これにより、悪意あるデータが分類器の判断を操作するリスクを軽減しています。

### デフォルトの許可/ブロック基準

**自動的に許可される操作：**
- 作業ディレクトリ内のファイルの読み取り・編集・作成
- ロックファイルに宣言された依存関係のインストール
- 読み取り専用の HTTP リクエスト
- 現在のブランチへの `git push`
- テストやリントの実行

**自動的にブロックされる操作：**
- `curl | bash` のような外部コードの直接実行
- 機密データの外部への送信
- 本番環境へのデプロイやマイグレーション
- 大量のファイル削除
- IAM やリポジトリ権限の変更
- `main` ブランチへの `force push`

### 利用条件（重要）

Auto モードには明確な利用条件があります。ご注意ください。

| 条件 | 詳細 |
|------|------|
| **対応プラン** | Team、Enterprise、または API のみ |
| **非対応プラン** | **Pro / Max では利用不可** |
| **対応モデル** | Claude Sonnet 4.6 または Opus 4.6 |
| **対応プロバイダ** | Anthropic API のみ（AWS Bedrock / Google Vertex / Foundry は不可） |
| **管理者設定** | Team/Enterprise では管理者が事前に有効化している必要がある |

Pro プランをご利用の方は、Auto モードの代わりに `acceptEdits` モードと `settings.json` の `permissions.allow` を組み合わせることで、近い使用感を実現できます。

### 設定方法

**デスクトップアプリの場合：**
1. 設定画面を開く
2. 「Auto mode」を有効化する
3. プロンプト送信ボタン横のモードセレクターから「Auto mode」を選択

**CLI の場合：**
```bash
claude --permission-mode auto
```

**セッション中の切り替え：**
- `Shift+Tab` でモードを巡回（ただし Auto モードはこの巡回には含まれないことがあります）
- モードセレクターから直接選択

### Auto モードを安全に使うためのガイドライン

1. **まず小さなタスクから試す**
   - 最初から大規模なリファクタリングを Auto モードで実行するのは避けましょう
   - テストの追加、ドキュメントの更新など、影響範囲の小さいタスクから始めてください

2. **settings.json の deny リストを充実させる**
   - Auto モードでも deny リストは有効です
   - 絶対に実行してほしくない操作は deny に追加しておきましょう

3. **Git のブランチを活用する**
   - Auto モードで作業する前に、必ず作業ブランチを切りましょう
   - 問題が起きても `main` ブランチには影響しません

4. **Hooks と組み合わせる**
   - PostToolUse でテストを自動実行する Hooks を設定しておくと、Auto モードの安全性がさらに向上します

### Pro プランでの代替策

Pro プランでは Auto モードは使えませんが、以下の組み合わせで効率的な作業が可能です。

```json
// .claude/settings.json
{
  "permissions": {
    "allow": [
      "Read", "Edit", "Write", "Glob", "Grep",
      "Bash(git status)", "Bash(git diff *)",
      "Bash(git add *)", "Bash(git commit *)",
      "Bash(npm test)", "Bash(npm run lint)",
      "Bash(npm run build)", "Bash(ls *)"
    ],
    "deny": [
      "Bash(rm -rf *)", "Bash(sudo *)",
      "Bash(git push --force *)"
    ]
  }
}
```

この設定で `acceptEdits` モード（`Shift+Tab` で切り替え）を使えば、日常的な操作のほとんどは自動承認されます。

## やってみよう（ハンズオン）

### ステップ 1：現在のパーミッションモードを確認する

```bash
# CLI の場合
claude --help | grep permission
```

デスクトップアプリの場合は、プロンプト送信ボタン横のモードセレクターを確認してください。

### ステップ 2：acceptEdits モードで作業してみる

まずは Auto モードの前段階として、`acceptEdits` モードを試しましょう。

1. `Shift+Tab` で `acceptEdits` モードに切り替え
2. 「このファイルにコメントを追加して」のような簡単な指示を出す
3. ファイル編集が自動承認されることを確認

### ステップ 3：Auto モードを試す（Team/Enterprise/API の場合）

Team/Enterprise/API プランの方は、以下の手順で Auto モードを試してください。

1. 設定画面で Auto mode を有効化
2. モードセレクターから「Auto mode」を選択
3. 「テストファイルを 3 つ追加して」のような安全なタスクを実行
4. すべての操作が自動承認されることを確認
5. 意図的に `rm -rf` を含む指示を出し、ブロックされることを確認

## まとめ
- Auto モードは安全分類器付きの自走モードで、許可疲れを解消する
- 2 段階の安全分類器（高速フィルター + 詳細判断）で操作のリスクを評価する
- **利用条件：Team / Enterprise / API プランのみ（Pro / Max は不可）**
- Pro プランでは `acceptEdits` + `settings.json` の allow/deny で代替可能
- 小さなタスクから始め、Git ブランチと Hooks を併用して安全性を確保する
