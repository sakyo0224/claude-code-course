## この記事で学ぶこと
- サブエージェントの概念と、メインエージェントとの違い
- スキルの仕組みと、カスタムスキルの作成方法
- プラグインによるパッケージ管理

## 前提知識

## 本文

### サブエージェントとは

サブエージェント（Sub-agent）とは、メインの Claude Code エージェントから分離された、**特定のタスクに特化した子エージェント**です。人間の組織に例えると、プロジェクトマネージャー（メインエージェント）が、専門家（サブエージェント）にタスクを委任するようなイメージです。

**サブエージェントを使うメリット：**
- **コンテキストの分離**：サブエージェントは独自のコンテキストを持つため、メインエージェントのコンテキストウィンドウを消費しない
- **専門性の付与**：特定の分野（セキュリティレビュー、テスト作成など）に特化した指示を設定できる
- **並列実行**：複数のサブエージェントを並行して実行できる

### サブエージェントの管理

サブエージェントの管理は `/agents` コマンドで行います。

```
/agents
```

このコマンドで、以下の操作ができます。
- サブエージェントの一覧表示
- 新しいサブエージェントの作成
- 既存のサブエージェントの編集・削除

### サブエージェントの活用例

**例 1：コードレビュー専門エージェント**

セキュリティとパフォーマンスに特化したレビューを行うサブエージェントを作成します。

```
コードレビューエージェントを作成して。以下の観点でレビューを行うように設定して：
- SQLインジェクション、XSSなどのセキュリティ脆弱性
- N+1クエリ、不要なループなどのパフォーマンス問題
- 型安全性の確認
```

**例 2：テスト作成専門エージェント**

テストコードの作成に特化したサブエージェントです。

```
テスト作成エージェントを作成して。以下のルールに従って：
- Jest を使用
- 各関数に対して正常系と異常系のテストを作成
- モックは最小限に抑える
- カバレッジ80%以上を目指す
```

**例 3：ドキュメント作成エージェント**

API ドキュメントの作成に特化したサブエージェントです。

```
ドキュメントエージェントを作成して。コードを分析してJSDoc形式のコメントと
API仕様書を自動生成するように設定して
```

### Agent Teams（エージェントチーム）

複数のサブエージェントを協調させる機能が Agent Teams です。`--teammate-mode` オプションを使用します。

これにより、たとえば以下のような協調動作が可能になります。
- 実装エージェントがコードを書く
- レビューエージェントがコードをチェック
- テストエージェントがテストを作成
- これらが並列で動作し、互いの結果を参照する

### スキルとは

スキル（Skills）は、Claude Code に追加機能を提供する**拡張モジュール**です。スラッシュコマンドとして呼び出せます。

**ビルトインスキルの例：**

| スキル | コマンド | 説明 |
|--------|---------|------|
| Simplify | `/simplify` | コードの品質改善・簡素化 |
| Batch | `/batch` | 一括処理 |
| Debug | `/debug` | デバッグ支援 |
| Loop | `/loop` | 定期的なコマンド繰り返し実行 |
| Claude API | `/claude-api` | Claude API の利用支援 |

**スキルの確認：**
```
/skills
```

**スキルの検索：**
```
/find-skills
```

新しいスキルを探してインストールできます。

### カスタムスキルの概念

スキルは、特定のタスクに対する「知識 + ツール + 手順」のパッケージです。たとえば「請求書処理スキル」は以下を含みます。

- PDF を読み取るための知識
- データ抽出のルール
- CSV への出力手順
- エラーハンドリングの方法

### プラグイン

プラグイン（Plugin）は、スキル・エージェント・Hooks・MCP サーバー・LSP 設定を**1 つのパッケージにまとめたもの**です。

```
/plugin
```

このコマンドでプラグインの管理を行います。プラグインにより、複雑な設定を一括でインストール・共有できます。

たとえば「React 開発プラグイン」には以下が含まれます。
- React コンポーネント作成スキル
- ESLint/Prettier の Hooks 設定
- テスト作成エージェント
- Storybook 連携の MCP サーバー

### SubagentStop Hook

サブエージェントの出力を自動で検証するために、`SubagentStop` Hook を設定できます。

```json
{
  "hooks": {
    "SubagentStop": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "echo 'サブエージェント完了: ' && npm test"
          }
        ]
      }
    ]
  }
}
```

この設定により、サブエージェントがタスクを完了するたびにテストが自動実行されます。

## やってみよう（ハンズオン）

### ステップ 1：スキル一覧を確認する

```
/skills
```

利用可能なスキルの一覧を確認してください。

### ステップ 2：Simplify スキルを試す

プロジェクト内のファイルを変更した後、以下を実行してみましょう。

```
/simplify
```

変更したコードの品質改善提案が表示されます。

### ステップ 3：サブエージェントを作成する

```
/agents
```

コードレビュー専門のサブエージェントを作成してみましょう。作成後、実際にコードレビューを依頼してみてください。

### ステップ 4：バッチ処理スキルを試す

```
/batch
```

複数のファイルに対して一括で処理を実行してみましょう。たとえば「すべての JavaScript ファイルに JSDoc コメントを追加する」といった指示を試してみてください。

## まとめ
- サブエージェントは特定のタスクに特化した子エージェントで、コンテキストの分離と並列実行が可能
- `/agents` コマンドでサブエージェントを管理できる
- スキルは Claude Code の拡張機能で、`/skills` で一覧確認、スラッシュコマンドで呼び出す
- プラグインはスキル・エージェント・Hooks・MCP をまとめたパッケージ
- SubagentStop Hook でサブエージェントの出力を自動検証できる
