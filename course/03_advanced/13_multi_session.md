# マルチセッションと並列作業

## この記事で学ぶこと
- 複数セッションの同時実行と Git worktree による分離
- 効率的な並列作業のパターン
- セッション管理のコマンドとベストプラクティス

## 前提知識
- Git / GitHub 連携の基本（→ [Git / GitHub 連携の基本](06_git_basics.md)）
- Claude Code のセッション概念の理解

## 本文

### なぜ並列作業が必要なのか

開発作業では、複数のタスクを同時に進めたい場面が頻繁にあります。

- バグ修正をしながら、新機能の実装も進めたい
- テストの作成を待つ間に、ドキュメントの更新を行いたい
- 複数のファイルを独立して並行処理したい

Claude Code では、複数のセッションを同時に実行することで、これらの並列作業が可能です。

### Git Worktree による安全な並列作業

複数のセッションが同じリポジトリで同時に作業すると、ファイルの競合が発生します。Claude Code はこの問題を **Git worktree**（ワークツリー）で解決します。

Git worktree とは、1 つのリポジトリに対して**複数の作業ディレクトリ**を作成する Git の機能です。各ワークツリーは独立したブランチで動作するため、互いの作業が干渉しません。

```
リポジトリ（.git）
├── worktree-1/ (feature/auth ブランチ) ← セッション A が作業
├── worktree-2/ (fix/login-bug ブランチ) ← セッション B が作業
└── worktree-3/ (docs/api-spec ブランチ) ← セッション C が作業
```

**Claude Code の自動 worktree 作成：**

デスクトップアプリで新しいセッションを開始すると、Claude Code は自動的に worktree を作成して分離します。手動で worktree を管理する必要はありません。

### セッション管理のコマンド

**新しいセッションの開始：**

デスクトップアプリでは、新しいタブまたはウィンドウを開くことで新しいセッションが開始されます。

CLI の場合：
```bash
# 新しいセッションを開始
claude

# バックグラウンドでセッションを開始
claude -p "テストを全て実行して修正して" &
```

**セッションの一覧と再開：**
```
/resume
```

過去のセッション一覧が表示され、選択して再開できます。

**セッションの分岐：**
```
/branch feature-experiment
```

現在のセッションの状態を保持したまま、新しいブランチとして分岐します。異なるアプローチを試すときに便利です。

**バックグラウンドタスクの管理：**
```
/tasks
```

バックグラウンドで実行中のタスク一覧を表示します。

**セッションをバックグラウンドに送る：**

`Ctrl+B` で現在のタスクをバックグラウンドに送り、新しい指示を入力できます。

### 並列作業のパターン

**パターン 1：機能開発 + テスト作成の並列化**

```
セッション A：「ユーザー認証機能を実装して」
セッション B：「既存のAPIエンドポイントのテストを書いて」
```

セッション A が認証機能を実装している間に、セッション B が別の部分のテストを作成します。

**パターン 2：フロントエンド + バックエンドの同時開発**

```
セッション A：「ログインフォームのUIコンポーネントを作成して」
セッション B：「認証APIのエンドポイントを実装して」
```

**パターン 3：調査 + 実装の並行**

```
セッション A（planモード）：「このバグの原因を調査して報告して」
セッション B：「ドキュメントの更新を進めて」
```

Plan モードのセッションで調査を行いながら、別のセッションで実装を進めます。

### バックグラウンドタスクの活用

長時間かかるタスク（テストの全件実行、大規模なリファクタリングなど）は、バックグラウンドに送って別の作業を続けられます。

```
1. 「テストを全件実行して、失敗したものを修正して」と指示
2. Ctrl+B でバックグラウンドに送る
3. 新しい指示を入力して別の作業を開始
4. /tasks でバックグラウンドタスクの進捗を確認
```

### セッション名の管理

複数のセッションを使う場合、名前をつけて管理すると混乱を防げます。

```
/rename auth-feature
```

セッション一覧（`/resume`）で、どのセッションがどの作業をしていたかが一目でわかります。

### worktree に関する追加設定

`.worktreeinclude` ファイルを使うと、通常 `.gitignore` で除外されているファイルを worktree に含めることができます。

```
# .worktreeinclude
.env.example
config/local.example.json
```

### 注意点とベストプラクティス

1. **セッション数の上限に注意**：同時に実行するセッションが多すぎると、マシンのリソース（メモリ、CPU）を消費します。通常は 2〜3 セッションが適切です

2. **トークン消費の管理**：並列セッションはそれぞれ独立してトークンを消費します。`/cost` コマンドで各セッションの消費量を確認しましょう

3. **マージの順序を考慮する**：並列で開発したブランチをマージする際、コンフリクトが発生する可能性があります。依存関係のあるブランチは順番にマージしましょう

4. **`/compact` で文脈を圧縮する**：長時間のセッションでは、`/compact` コマンドで会話履歴を圧縮し、トークン消費を抑えましょう

## やってみよう（ハンズオン）

### ステップ 1：セッション名を設定する

現在のセッションに名前をつけましょう。

```
/rename main-work
```

### ステップ 2：バックグラウンドタスクを試す

簡単なタスクをバックグラウンドに送ってみましょう。

1. 「プロジェクト内のすべての TODO コメントを探して一覧にして」と指示
2. `Ctrl+B` でバックグラウンドに送る
3. 別の指示を入力して作業を続ける
4. `/tasks` で進捗を確認

### ステップ 3：セッションの分岐を試す

```
/branch experiment-a
```

分岐したセッションで実験的な変更を行い、元のセッションに戻ってみましょう。

```
/resume
```

## まとめ
- Claude Code は複数セッションの同時実行をサポートし、Git worktree で自動的にファイルを分離する
- `Ctrl+B` でタスクをバックグラウンドに送り、並行して別の作業を進められる
- `/resume` で過去のセッションを再開、`/branch` でセッションを分岐できる
- 並列セッションは 2〜3 が適切。トークン消費とマシンリソースに注意する
- セッションに名前をつけて管理すると、複数の作業を整理しやすい
