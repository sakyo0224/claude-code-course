## 本文

### Slack 連携

Slack 連携を設定すると、Claude Code から直接 Slack にメッセージを送ったり、チャンネルの情報を取得したりできます。

**リモートコネクタでの設定（推奨）：**

デスクトップアプリの場合：
1. プロンプト入力欄の「+」ボタンをクリック
2. 「Connect an app」→「Slack」を選択
3. Slack ワークスペースの OAuth 認証を完了

CLI の場合：
```
/install-slack-app
```

**利用できる主な操作：**
- チャンネルへのメッセージ送信
- チャンネルのメッセージ検索
- チャンネル一覧の取得
- スレッドへの返信

**活用シナリオ 1：デプロイ通知の自動化**
```
デプロイが完了したら、Slackの#deploymentsチャンネルに以下の内容で通知して：
- デプロイ日時
- 変更内容のサマリー
- 影響を受けるサービス
```

**活用シナリオ 2：日次レポートの自動投稿**
```
今日のGitコミットログを要約して、Slackの#dev-dailyチャンネルに日次レポートとして投稿して
```

### Notion 連携

Notion 連携を設定すると、Claude Code からドキュメントの作成・更新・検索が可能になります。

**リモートコネクタでの設定：**
1. デスクトップアプリの「+」ボタン → 「Connect an app」→「Notion」
2. Notion ワークスペースの OAuth 認証を完了
3. Claude Code にアクセスを許可するページを選択

**利用できる主な操作：**
- ページの作成・更新
- データベースの検索・フィルタリング
- ページ内容の取得
- コメントの追加

**活用シナリオ 1：技術仕様書の自動生成**
```
この関数の実装内容を分析して、Notionの「技術仕様書」データベースに新しいページを作成して。
以下の項目を含めて：
- 機能名
- 概要
- API仕様
- データフロー図（テキストベース）
```

**活用シナリオ 2：タスク管理との連携**
```
Notionの「スプリントタスク」データベースから未完了のタスクを取得して、
優先度の高い順に並べて表示して
```

### Google Calendar 連携

Google Calendar 連携を設定すると、Claude Code からスケジュールの確認・作成・更新ができます。

**リモートコネクタでの設定：**
1. デスクトップアプリの「+」ボタン → 「Connect an app」→「Google Calendar」
2. Google アカウントの OAuth 認証を完了

**利用できる主な操作：**
- 予定の一覧表示
- 新しい予定の作成
- 予定の更新・削除
- 空き時間の検索
- 出席応答

**活用シナリオ 1：開発スケジュールの管理**
```
今週の予定を確認して、コーディングに使える空き時間をリストアップして
```

**活用シナリオ 2：リリーススケジュールの自動登録**
```
次のスプリントリリースの予定を作成して。
日時：来週金曜 15:00-16:00
タイトル：Sprint 12 リリース
参加者：dev-team@example.com
説明：変更内容のサマリーを含めて
```

### 複数の MCP サーバーを組み合わせたワークフロー

MCP の真価は、複数のサービスを組み合わせたときに発揮されます。

**ワークフロー例 1：PR マージ → Slack 通知 → Notion 更新**

```
PR #42 をマージして、完了したら以下を実行して：
1. Slackの#releasesチャンネルにマージ内容を通知
2. Notionの「リリースログ」に変更内容を記録
3. Google Calendarにリリース確認の予定を30分後に作成
```

Claude Code はこの指示を受けて、順番に各 MCP サーバーのツールを呼び出します。

**ワークフロー例 2：障害対応の自動化**

```
エラーログを分析して、以下を実行して：
1. 原因と対策をまとめたレポートを作成
2. Slackの#incident-responseチャンネルに障害報告を投稿
3. Notionの「障害履歴」データベースに記録を追加
4. 対応ミーティングの予定をGoogle Calendarに作成
```

### MCP サーバーの設定一覧の確認

すべての MCP サーバーの設定を確認するには、settings.json を直接確認するか、以下のコマンドを使います。

```
/mcp
```

### トラブルシューティング

| 問題 | 原因 | 対策 |
|------|------|------|
| MCP サーバーに接続できない | 認証の期限切れ | 再度 OAuth 認証を行う |
| ツールが見つからない | サーバーが起動していない | `/mcp` で状態を確認、再起動 |
| 権限エラー | アクセス範囲の不足 | 連携サービス側でアクセス権限を確認・追加 |
| 応答が遅い | ネットワークの問題 | インターネット接続を確認 |

### セキュリティのベストプラクティス

1. **最小権限の原則**：MCP コネクタには必要最小限のスコープ（権限範囲）のみを許可する
2. **定期的な認証情報の見直し**：使わなくなった連携は無効化する
3. **機密情報の取り扱い**：Slack やNotion で機密情報を扱う場合は、Claude Code の操作ログを確認する習慣をつける
4. **テスト環境での検証**：本番の Slack ワークスペースに直接メッセージを送る前に、テストチャンネルで動作を確認する

## やってみよう（ハンズオン）

### ステップ 1：Slack 連携を設定する

1. デスクトップアプリで Slack コネクタを接続するか、CLI で `/install-slack-app` を実行
2. 以下のコマンドで接続を確認：
```
Slackのチャンネル一覧を表示して
```

### ステップ 2：テストメッセージを送信する

```
Slackの#testチャンネルに「Claude Code からのテスト送信です」と投稿して
```

テストチャンネルがない場合は、自分だけが見えるチャンネルを作成してテストしてください。

### ステップ 3：複合ワークフローを試す

複数のサービスが接続されている場合、以下のような複合指示を試してみましょう。

```
今日のGitコミット履歴を要約して、Slackの#dev-dailyに投稿して
```

## まとめ
- Slack / Notion / Google Calendar はリモート MCP コネクタで簡単に接続できる
- 自然言語で指示するだけで、各サービスの操作が可能
- 複数の MCP サーバーを組み合わせることで、サービス横断のワークフローを自動化できる
- セキュリティに注意し、最小権限の原則を守る
- テスト環境で検証してから本番環境に適用する
