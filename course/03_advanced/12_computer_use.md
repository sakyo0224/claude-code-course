## 本文

### Computer Use とは

Computer Use（コンピュータ操作）は、Claude Code がデスクトップ上のアプリケーションを**画面認識を通じて直接操作**する機能です。マウスのクリック、キーボードの入力、画面のスクロールなど、人間がパソコンで行う操作を Claude Code が代行できます。

通常の Claude Code はファイル操作やターミナルコマンドを通じて作業しますが、Computer Use を使うと GUI（グラフィカルユーザーインターフェース）を持つアプリケーションも操作対象になります。

### 動作の仕組み

Computer Use は以下のサイクルで動作します。

```
1. スクリーンショットの撮影
   ↓
2. 画像認識による画面内容の理解
   ↓
3. 操作の判断（クリック位置、入力内容の決定）
   ↓
4. マウス/キーボード操作の実行
   ↓
5. 1に戻る（操作結果を確認）
```

このサイクルを繰り返すことで、複雑な GUI 操作を実行します。

### 対応プラットフォーム

| プラットフォーム | 対応状況 | 備考 |
|----------------|---------|------|
| macOS | 対応済み | Accessibility / Screen Recording 権限の設定が必要 |
| Windows | 対応済み（2026年4月3日〜） | 権限設定は不要 |

**macOS での権限設定：**
1. システム設定 → プライバシーとセキュリティ → アクセシビリティ
2. Claude Code デスクトップアプリを許可リストに追加
3. システム設定 → プライバシーとセキュリティ → 画面収録
4. Claude Code デスクトップアプリを許可リストに追加

### アプリケーションごとの権限管理

Computer Use では、アプリケーションごとに 3 段階の権限を設定できます。

| 権限レベル | 説明 | 適したアプリ |
|-----------|------|-------------|
| **View only**（閲覧のみ） | スクリーンショットの確認のみ。操作は不可 | 機密情報を含むアプリ |
| **Click only**（クリックのみ） | クリック操作のみ可能。テキスト入力は不可 | 設定画面、ナビゲーション |
| **Full control**（完全制御） | すべての操作が可能 | 開発ツール、テキストエディタ |

**設定方法：**
デスクトップアプリの設定画面 → Computer Use → アプリケーションごとの権限を選択

### Chrome 拡張連携

ブラウザ操作には、スクリーンショットベースの Computer Use よりも高精度な方法があります。Chrome 拡張を使うと、DOM（ページの構造）を直接認識して操作できます。

**Chrome 拡張のセットアップ：**
```
/chrome
```

このコマンドで Chrome 拡張の設定ガイドが表示されます。

**Chrome 拡張のメリット：**
- スクリーンショットより高精度な要素認識
- テキスト入力が正確
- ページの読み込み待ちを適切に処理
- 複数タブの管理が可能

### Computer Use の活用例

**例 1：Figma デザインの確認と実装**
```
Figmaで開いているデザインを確認して、同じレイアウトのReactコンポーネントを作成して
```

**例 2：スプレッドシートの操作**
```
Excelで開いている売上データを確認して、グラフを作成して
```

**例 3：ブラウザでの動作確認**
```
開発サーバーをChromeで開いて、ログインフォームが正しく動作するか確認して
```

**例 4：IDE の操作**
```
VS Codeで開いているファイルのデバッグを開始して、ブレークポイントの変数値を確認して
```

### トークン消費に関する注意

Computer Use は**トークン消費が非常に大きい**機能です。スクリーンショットの画像処理により、1 回のキャプチャで約 1,334 トークン（1000x1000 ピクセルの場合）を消費します。

**トークンを節約するコツ：**
1. **可能な限りテキストベースの操作を優先する**：ファイル操作やコマンド実行で代替できる作業は、Computer Use を使わない
2. **操作の範囲を限定する**：「ブラウザで全ページをチェックして」ではなく、「ログインページのフォームを確認して」と具体的に指示する
3. **解像度の考慮**：高解像度のディスプレイではスクリーンショットのサイズが大きくなり、トークン消費が増加する

### 自動検証（Auto-verify）

Computer Use と連携した自動検証機能があります。コードを編集した後、自動で以下のチェックを実行できます。

- ライブプレビューのスクリーンショット撮影
- DOM の検査（エラーチェック）
- コンソールエラーの確認
- レイアウトの崩れの検出

この機能は、開発サーバー（`npm run dev` など）が起動している状態で動作します。

### セキュリティ上の注意

Anthropic は以下を推奨しています。

- **機密データを扱うアプリケーションと併用しないこと**：パスワードマネージャー、銀行アプリなどが画面に表示されている状態での使用は避ける
- **操作範囲を限定すること**：必要なアプリのみに権限を付与する
- **操作内容を確認する習慣をつけること**：特に Full control モードでは、Claude がどのような操作を行ったか確認する

## やってみよう（ハンズオン）

### ステップ 1：Computer Use の権限を設定する（macOS の場合）

1. システム設定 → プライバシーとセキュリティ → アクセシビリティ で Claude を許可
2. システム設定 → プライバシーとセキュリティ → 画面収録 で Claude を許可

### ステップ 2：簡単な操作を試す

テキストエディタ（メモ帳やテキストエディット）を開いた状態で、以下を試してください。

```
テキストエディタに「Hello from Claude Code!」と入力して
```

### ステップ 3：Chrome 拡張を設定する

```
/chrome
```

表示されるガイドに従って Chrome 拡張をインストールし、簡単なブラウザ操作を試してみましょう。

```
Chromeで https://example.com を開いて、ページのタイトルを教えて
```

## まとめ
- Computer Use は画面認識を通じてデスクトップアプリケーションを操作する機能
- アプリごとに View only / Click only / Full control の 3 段階で権限を設定できる
- Chrome 拡張を使うとブラウザ操作がより高精度になる
- トークン消費が大きいため、テキストベースの操作で代替できる場合はそちらを優先する
- 機密データを扱うアプリとの併用は避け、操作範囲を限定する
