## 本文

### リモートセッションとは

リモートセッションは、Claude Code のタスクを**Anthropic のクラウド環境で実行**する機能です。ローカルのパソコンではなくクラウドで動作するため、パソコンを閉じても、スリープしても、作業が継続します。

**ローカルセッションとの比較：**

| 項目 | ローカルセッション | リモートセッション |
|------|-------------------|-------------------|
| 実行環境 | 自分のパソコン | Anthropic クラウド |
| パソコンの状態 | 起動している必要がある | 閉じてもOK |
| 実行時間 | パソコンの稼働時間に依存 | 長時間タスクに対応 |
| ファイルアクセス | ローカルファイルに直接アクセス | リポジトリのクローンを使用 |
| 複数リポジトリ | 1 つのリポジトリ | 複数リポジトリを同時に扱える |

### リモートセッションの開始方法

**方法 1：デスクトップアプリの「Continue in」メニュー**

ローカルで作業中のセッションを、リモートに移行できます。

1. セッション中に「Continue in」メニューを開く
2. 「Remote」を選択
3. セッションがクラウドに移行され、ローカルの操作は不要になる

**方法 2：最初からリモートで開始**

デスクトップアプリの新しいセッション作成時に、実行環境として「Remote」を選択します。

**方法 3：CLI から**

```bash
# リモートセッションの開始（具体的なコマンドはバージョンにより異なる場合があります）
claude --remote
```

### リモートセッションの活用シーン

**シーン 1：大規模なリファクタリング**

数時間かかるような大規模な変更を、夜間にリモートで実行します。

```
プロジェクト全体のTypeScript移行を行って。
1. .jsファイルを.tsに変換
2. 型定義を追加
3. テストが全て通ることを確認
4. 完了したらPRを作成して
```

朝起きたときには PR が作成されています。

**シーン 2：CI/CD の補完**

テストが失敗した PR を自動で修正するリモートセッションを設定します。

**シーン 3：複数リポジトリにまたがる変更**

リモートセッションでは複数のリポジトリを同時に扱えるため、マイクロサービス間の整合性を保った変更が可能です。

### Remote Control（リモートコントロール）

Remote Control は、外部からローカルの Claude Code セッションを操作する機能です。

```
/remote-control
```

または短縮形：
```
/rc
```

これにより、claude.ai（Web）やモバイルアプリから、ローカルで実行中の Claude Code セッションに指示を送れます。

**Remote Control の流れ：**
1. ローカルで `/remote-control` を実行
2. 接続用のコードが表示される
3. claude.ai またはモバイルアプリでコードを入力して接続
4. 外部から指示を送信

### Dispatch（ディスパッチ）

Dispatch は、**スマートフォンからタスクを送信**する機能です。外出先からでも、Claude Code にタスクを依頼できます。

**使い方：**
1. スマートフォンの Claude アプリまたは Web ブラウザで claude.ai にアクセス
2. Dispatch 機能を使ってタスクを送信
3. タスクはリモートセッションとして実行される
4. 完了通知をスマートフォンで受け取る

**活用例：**
- 通勤中に「昨日のバグを修正して PR を作成して」と送信
- 会議中に思いついた改善案を「TODO として追加して」と送信
- 外出先から「テストを全件実行して結果を報告して」と確認

### SSH セッション

リモートサーバー上で Claude Code を直接実行することもできます。

```bash
# リモートサーバーに SSH で接続
ssh user@server.example.com

# サーバー上で Claude Code を起動
claude
```

**SSH セッションの活用：**
- 本番環境に近い環境でのテスト
- GPU が必要な機械学習タスク
- 特定のネットワーク環境が必要な作業

### セッションの継続と復帰

リモートセッションの状態は保存されるため、後からいつでも確認・復帰できます。

```
/resume
```

リモートセッションも一覧に表示されるため、ローカルセッションと同様に再開できます。

### IDE との連携

ローカルセッションを IDE（VS Code など）に移行することもできます。

```
/desktop
```

または「Continue in」メニューから IDE を選択します。これにより、Claude Code での作業を IDE の環境で引き継げます。

### コスト管理

リモートセッションはクラウドリソースを消費するため、コスト管理が重要です。

```
/cost
```

各セッションのトークン消費量を確認できます。

**リモートセッションのコスト最適化：**
- 具体的な指示を出すことで、不要な探索を減らす
- タスクの範囲を明確にし、完了条件を指定する
- `/compact` で文脈を圧縮してからリモートに移行する

## やってみよう（ハンズオン）

### ステップ 1：Remote Control を試す

```
/remote-control
```

表示される接続コードを控え、別のデバイス（スマートフォンやブラウザの claude.ai）から接続してみましょう。

### ステップ 2：ローカルセッションをリモートに移行する

1. ローカルで簡単なタスクを開始（例：「READMEを更新して」）
2. 「Continue in」メニューから「Remote」を選択
3. パソコンを閉じても作業が継続することを確認
4. `/resume` でリモートセッションの結果を確認

### ステップ 3：コスト確認の習慣をつける

```
/cost
```

セッション終了時にコストを確認する習慣をつけましょう。

## まとめ
- リモートセッションは Anthropic クラウドで実行され、パソコンを閉じても作業が続く
- 「Continue in」メニューでローカルからリモートに簡単に移行できる
- Remote Control で外部デバイスからローカルセッションを操作できる
- Dispatch でスマートフォンからタスクを送信できる
- 複数リポジトリの同時操作や長時間タスクに特に有効
- コスト管理に注意し、具体的な指示でトークン消費を最適化する
