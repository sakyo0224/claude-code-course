# オススメコマンド・テクニック50選

## この記事で学ぶこと

- Claude Code で使える便利なコマンドとテクニックを50個知る
- 場面に応じて最適な操作方法を選べるようになる
- すぐに試せる具体的な操作手順を身につける

## はじめに

Claude Code には、日常的に使える便利なコマンドやテクニックがたくさん用意されています。しかし、数が多いためにすべてを把握するのは大変です。
この記事では、特に実用的なものを10カテゴリ・50個に厳選してご紹介いたします。気になるものから一つずつ試してみてください。

## 基本操作コマンド

### 1. /help — 使い方を表示する

**使いどころ：** 操作方法が分からないとき
**操作方法：**
```
/help
```
と入力するだけ

**ポイント：** 利用可能なスラッシュコマンドの一覧も表示されますので、困ったらまずこれを試しましょう

### 2. /status — 現在の状態を確認する

**使いどころ：** 接続中のモデルや認証状態を確認したいとき
**操作方法：**
```
/status
```
と入力します

**ポイント：** 現在使用中のモデル名、認証方法、プロジェクトのパスなどが一覧で表示されます

### 3. /clear — 会話をリセットする

**使いどころ：** 話題を切り替えたいとき、コンテキストが長くなりすぎたとき
**操作方法：**
```
/clear
```
と入力します

**ポイント：** これまでの会話履歴がすべてクリアされます。トークン消費を抑えたいときにも有効です

### 4. /compact — 会話を要約して圧縮する

**使いどころ：** 会話が長くなってきたが、文脈は維持したいとき
**操作方法：**
```
/compact
```
と入力します。オプションでカスタム指示を追加することもできます

**ポイント：** 会話の要点を保ちながらトークン消費を大幅に削減できます。`/compact 日本語のコードに関する部分だけ残して` のように指示も可能です

### 5. /model — 使用モデルを切り替える

**使いどころ：** 別のモデルで回答を試したいとき
**操作方法：**
```
/model
```
と入力して、表示される一覧から選択します

**ポイント：** Opus、Sonnet、Haikuなど、タスクの難易度やコストに応じて使い分けることができます

### 6. /cost — トークン消費量を確認する

**使いどころ：** このセッションでどのくらいコストがかかっているか知りたいとき
**操作方法：**
```
/cost
```
と入力します

**ポイント：** 入力・出力トークン数と概算コストが表示されます。使い過ぎ防止に定期的に確認しましょう

### 7. /login — アカウントにログインする

**使いどころ：** 初回セットアップ時や、認証が切れたとき
**操作方法：**
```
/login
```
と入力して、画面の指示に従います

**ポイント：** ブラウザが自動で開き、Anthropicアカウントで認証できます

### 8. /logout — アカウントからログアウトする

**使いどころ：** 別のアカウントに切り替えたいとき、共有端末を使い終わったとき
**操作方法：**
```
/logout
```
と入力します

**ポイント：** セキュリティのため、共有環境では作業後にログアウトしておくことをおすすめします

## セッション管理

### 9. /resume — 過去のセッションを再開する

**使いどころ：** 前回の作業の続きをしたいとき
**操作方法：**
```
/resume
```
と入力すると、過去のセッション一覧が表示されます

**ポイント：** セッションIDを指定して直接再開することもできます。前回の文脈を引き継いで作業を継続できます

### 10. /rewind — 直前の操作を取り消す

**使いどころ：** 直前の応答をやり直したいとき
**操作方法：**
```
/rewind
```
と入力します

**ポイント：** Claude の直前のターン（応答とそれに伴うファイル変更）を取り消して、やり直すことができます

### 11. /diff — 変更差分を確認する

**使いどころ：** Claude がどのファイルをどう変更したか確認したいとき
**操作方法：**
```
/diff
```
と入力します

**ポイント：** Git形式の差分表示で、変更箇所が一目で分かります。コミット前の確認に便利です

### 12. /branch — ブランチを切り替える

**使いどころ：** 作業用ブランチに切り替えたいとき
**操作方法：**
```
/branch ブランチ名
```
と入力します

**ポイント：** Git操作に不慣れでも、簡単にブランチの作成・切り替えができます

### 13. Escキー2回で巻き戻し

**使いどころ：** 素早く直前の操作を取り消したいとき
**操作方法：** Escキーを素早く2回押します

**ポイント：** `/rewind` と同じ効果ですが、キーボードだけで操作できるため素早く対応できます。誤って実行してしまった処理をすぐに戻したいときに重宝します

### 14. /context — コンテキストの管理

**使いどころ：** 現在の会話で参照されているファイルやコンテキストを確認・管理したいとき
**操作方法：**
```
/context
```
と入力します

**ポイント：** どのファイルが会話のコンテキストに含まれているかを確認でき、不要なものを除外することもできます

## 計画・実行モード

### 15. /plan — 計画モードで作業する

**使いどころ：** 大きなタスクを実行する前に、まず計画を立てたいとき
**操作方法：**
```
/plan このプロジェクトにログイン機能を追加したい
```
のように入力します

**ポイント：** コードを変更する前に、実施手順や影響範囲を整理してくれます。いきなり実装に入るよりも安全です

### 16. Shift+Tabで計画モード切替

**使いどころ：** 入力中に素早く計画モードと通常モードを切り替えたいとき
**操作方法：** プロンプト入力中に Shift+Tab を押します

**ポイント：** 入力欄の左側にモード表示が切り替わります。計画モードではファイル変更を行わず、方針だけを提案してくれます

### 17. /fast — 高速モードに切り替える

**使いどころ：** 簡単な質問やちょっとした作業で、素早く回答が欲しいとき
**操作方法：**
```
/fast
```
と入力します

**ポイント：** より高速なモデル（Haiku等）に切り替わり、応答速度が上がります。単純な質問には十分な品質です

### 18. /auto — 自動承認モードに切り替える

**使いどころ：** ファイル変更の確認を毎回求められるのが煩わしいとき
**操作方法：**
```
/auto
```
と入力します

**ポイント：** ファイルの読み書きやコマンド実行を自動的に承認します。信頼できる作業に限って使いましょう

### 19. /bypass — 権限チェックをスキップする

**使いどころ：** 特定の操作で毎回許可を求められるのを一時的にスキップしたいとき
**操作方法：**
```
/bypass
```
と入力します

**ポイント：** セキュリティに関わる設定ですので、内容を十分に理解した上で使用してください。不用意な使用は避けましょう

## ファイル操作テクニック

### 20. 画像をドラッグ＆ドロップ

**使いどころ：** スクリーンショットやデザイン画像を見せて指示したいとき
**操作方法：** 画像ファイルをターミナルのClaude Code入力欄にドラッグ＆ドロップします

**ポイント：** 「この画像のようなUIを作って」「このエラー画面を解決して」など、視覚的な指示が可能になります

### 21. @ファイル名でコンテキスト追加

**使いどころ：** 特定のファイルを参照させたいとき
**操作方法：** プロンプト内で
```
@src/index.ts
```
のようにファイルパスを記述します

**ポイント：** 明示的にファイルを指定することで、Claudeが正確にそのファイルの内容を理解した上で回答してくれます

### 22. PDFを読ませる

**使いどころ：** PDF文書の内容について質問したり、要約させたいとき
**操作方法：**
```
@document.pdf この文書を要約して
```
のように指示します

**ポイント：** 技術文書や仕様書をPDFのまま読み込ませて、内容について質問できます。日本語PDFにも対応しています

### 23. CSVを分析させる

**使いどころ：** データの傾向分析や集計をしたいとき
**操作方法：**
```
@data.csv このデータの傾向を分析して
```
のように指示します

**ポイント：** CSVファイルを読み込んで、統計的な分析やグラフ用のコード生成なども依頼できます

### 24. フォルダ一括処理の指示

**使いどころ：** フォルダ内の複数ファイルに対して同じ処理をしたいとき
**操作方法：**
```
src/componentsフォルダ内のすべてのファイルにTypeScriptの型注釈を追加して
```
のように指示します

**ポイント：** Claude が自動的にフォルダ内のファイルを走査し、一つずつ処理してくれます。大量のファイルがある場合は段階的に進めるよう指示すると安全です

### 25. 複数ファイルの比較

**使いどころ：** 2つ以上のファイルの違いを確認したいとき
**操作方法：**
```
@file1.ts と @file2.ts の違いを教えて
```
のように指示します

**ポイント：** 単なるdiff以上に、意味的な違いや影響範囲まで解説してくれます

## プロンプトテクニック

### 26. ステップバイステップ指示

**使いどころ：** 複雑なタスクを正確に実行してほしいとき
**操作方法：**
```
以下の手順で進めてください：1. まず〜 2. 次に〜 3. 最後に〜
```
のように番号付きで指示します

**ポイント：** 手順を明確に分けることで、抜け漏れなく作業が進みます。特に大きな変更を行うときに効果的です

### 27. 出力形式の指定

**使いどころ：** 回答の形式をコントロールしたいとき
**操作方法：**
```
JSON形式で出力して
```
```
表形式でまとめて
```
```
箇条書きで5つ挙げて
```
のように形式を指定します

**ポイント：** 出力形式を指定すると、そのままコピー＆ペーストで使える形で回答が返ってきます

### 28. 「まず計画を見せて」

**使いどころ：** いきなりコードを変更されたくないとき
**操作方法：**
```
ログイン機能を追加したい。まず計画を見せて、実装はしないで
```
と指示します

**ポイント：** 方針を確認してから実装に進められるため、手戻りを防げます。大きな変更ほど効果的です

### 29. 「3案出して」

**使いどころ：** 複数のアプローチを比較検討したいとき
**操作方法：**
```
この問題の解決方法を3案出して、それぞれのメリット・デメリットも添えて
```
と指示します

**ポイント：** 1つの方法に絞らず、選択肢を比較することで最適な判断ができます

### 30. 「前回の続きから」

**使いどころ：** 中断した作業を再開したいとき
**操作方法：**
```
/resume
```
でセッションを復元した後、
```
前回の続きから作業を進めて
```
と指示します

**ポイント：** セッションを復元すれば、前回の会話の文脈を引き継いだ状態で作業を再開できます

### 31. ペルソナ設定

**使いどころ：** 特定の立場や視点でレビューしてほしいとき
**操作方法：**
```
あなたはセキュリティの専門家です。このコードをレビューしてください
```
と指示します

**ポイント：** ペルソナを設定すると、その視点に特化した回答が得られます。「初心者」「上級者」「テスター」など、目的に応じて使い分けましょう

### 32. 制約条件の伝え方

**使いどころ：** 守ってほしい条件やルールがあるとき
**操作方法：**
```
以下の制約を守ってください：- 外部ライブラリは使わない - ファイルは3つ以内 - 100行以内
```
のように明記します

**ポイント：** 制約を先に伝えることで、やり直しの回数が大幅に減ります。CLAUDE.mdに書いておけば毎回伝える必要がなくなります

## カスタマイズ

### 33. CLAUDE.mdにルールを書く

**使いどころ：** プロジェクト固有のルールをClaudeに常に守らせたいとき
**操作方法：** プロジェクトルートに `CLAUDE.md` ファイルを作成し、ルールや指示を記述します

**ポイント：** 「回答は日本語で」「テストは必ず書く」「このフレームワークを使う」など、毎回伝えたい指示を書いておくと自動的に読み込まれます

### 34. settings.jsonの基本

**使いどころ：** Claude Codeの動作設定を細かく調整したいとき
**操作方法：**
```
~/.claude/settings.json
```
を編集します

**ポイント：** 許可するコマンド、デフォルトのモデル、テーマなどを設定できます。プロジェクトごとに `.claude/settings.json` を配置することも可能です

### 35. /installでスキル追加

**使いどころ：** Claude Codeに新しいスキルや機能を追加したいとき
**操作方法：**
```
/install スキル名
```
と入力します

**ポイント：** コミュニティが作成したスキルを追加して、Claude Codeの機能を拡張できます

### 36. テーマ変更

**使いどころ：** ターミナルの見た目を自分好みにしたいとき
**操作方法：**
```
/theme
```
と入力して、表示される一覧から選択します

**ポイント：** ダークテーマ、ライトテーマなど複数のテーマが用意されています。長時間作業する方は目に優しいテーマを選びましょう

### 37. 許可設定のカスタマイズ

**使いどころ：** 毎回の確認ダイアログを減らしたいとき
**操作方法：** settings.jsonの `allowedTools` にツール名を追加するか、実行時に「常に許可」を選択します

**ポイント：** 信頼できる操作は事前に許可しておくことで、作業の流れが中断されにくくなります。ただし、破壊的な操作の許可は慎重に行いましょう

## 自動化・効率化

### 38. Hooksで保存時に自動処理

**使いどころ：** ファイル変更時にリントやフォーマットを自動実行したいとき
**操作方法：** settings.jsonの `hooks` セクションに処理を定義します（例：`"afterWrite": ["npx prettier --write $FILE"]`）

**ポイント：** ファイル保存のたびに自動でコード整形やチェックが走るため、品質を保ちやすくなります

### 39. /scheduleで定期実行

**使いどころ：** 定期的にタスクを実行したいとき（例：毎日のレポート生成）
**操作方法：**
```
/schedule
```
コマンドでcron形式のスケジュールを設定します

**ポイント：** リモートエージェントとして定期実行されるため、手動で起動する必要がありません

### 40. /loopで繰り返し

**使いどころ：** 一定間隔で同じ処理を繰り返したいとき
**操作方法：**
```
/loop 5m /status
```
のように、間隔とコマンドを指定します

**ポイント：** デプロイの監視やビルド状態の確認など、定期的なチェックに便利です

### 41. パイプで入力を渡す

**使いどころ：** 他のコマンドの出力をClaudeに渡して処理させたいとき
**操作方法：**
```
cat error.log | claude "このエラーログを分析して"
```
のようにパイプで渡します

**ポイント：** シェルの他のコマンドと組み合わせることで、非常に柔軟な使い方ができます

### 42. バッチ処理の指示パターン

**使いどころ：** 大量の同じ種類の作業を一括で処理したいとき
**操作方法：**
```
src/配下のすべての.jsファイルを.tsに変換して、型注釈も追加して
```
のように一括指示します

**ポイント：** 「すべての〜に対して〜する」というパターンで指示すると、Claudeが自動的に対象を列挙して順番に処理してくれます

## 外部連携

### 43. /mcpでサーバー確認

**使いどころ：** 接続中のMCPサーバーを確認したいとき
**操作方法：**
```
/mcp
```
と入力します

**ポイント：** 現在接続されているMCP（Model Context Protocol）サーバーの一覧とステータスが表示されます

### 44. MCP接続の追加

**使いどころ：** 外部ツールやデータベースとClaudeを連携させたいとき
**操作方法：**
```
claude mcp add サーバー名 コマンド
```
でMCPサーバーを追加します

**ポイント：** GitHub、データベース、ファイルシステムなど、さまざまな外部サービスと連携できます。settings.jsonに記述して恒久的に設定することも可能です

### 45. Slack/Notionへの出力

**使いどころ：** Claudeの作業結果をSlackやNotionに直接送りたいとき
**操作方法：** 対応するMCPサーバーを設定した上で、
```
この結果をSlackの#generalチャンネルに投稿して
```
のように指示します

**ポイント：** MCP経由で連携することで、Claudeから直接外部サービスにデータを送信できます。事前にMCPサーバーの設定が必要です

## トラブルシューティング

### 46. 応答が止まったとき

**使いどころ：** Claudeが長時間応答しない、フリーズしたように見えるとき
**操作方法：** まず Ctrl+C で現在の処理を中断します。それでも改善しない場合は `/clear` で会話をリセットします

**ポイント：** ネットワーク接続の確認も合わせて行いましょう。`/status` で接続状態を確認することもできます

### 47. トークン上限に達したとき

**使いどころ：** 「コンテキストウィンドウの上限に近づいています」と表示されたとき
**操作方法：**
```
/compact
```
で会話を圧縮するか、
```
/clear
```
でリセットして要点だけ伝え直します

**ポイント：** 長い会話を続けるとトークンが消費されていきます。こまめに `/compact` を使うか、適切なタイミングで `/clear` することを習慣にしましょう

### 48. 意図しない変更を戻すとき

**使いどころ：** Claudeがファイルを予期しない形で変更してしまったとき
**操作方法：** Escキー2回押し、または `/rewind` で直前の操作を取り消します。Gitを使っている場合は `/diff` で差分を確認してから `git checkout -- ファイル名` で戻すこともできます

**ポイント：** 大きな変更を行う前には、Gitでコミットしておくと安心です。いつでも元に戻せる状態を保ちましょう

## 上級テクニック

### 49. サブエージェントの活用

**使いどころ：** 複雑なタスクを並列で効率的に処理したいとき
**操作方法：** Claude Codeは内部的にサブエージェントを起動して、ファイル検索や分析を並行処理します。
```
このプロジェクト全体のセキュリティ監査をして
```
のような大きなタスクを投げると自動的に活用されます

**ポイント：** ユーザーが明示的に指定する必要はありませんが、タスクを大きな単位で依頼すると、Claudeが自動的にサブエージェントを使って効率的に処理してくれます

### 50. Computer Useの起動

**使いどころ：** ブラウザ操作やGUIアプリの操作をClaudeに任せたいとき
**操作方法：**
```
このWebサイトにアクセスして、スクリーンショットを撮って
```
のように指示すると、Computer Use機能が起動します

**ポイント：** 画面上のボタンクリックやフォーム入力など、GUI操作を自動化できます。現在はベータ機能のため、複雑な操作は段階的に指示することをおすすめします

## まとめ

- まずは基本コマンド（`/help`、`/clear`、`/compact`）を覚えるだけで、日常の操作が格段にスムーズになります
