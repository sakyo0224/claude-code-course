# 覚えておきたい基本スラッシュコマンド

## この記事で学ぶこと
- スラッシュコマンドとは何かを理解する
- 初心者が覚えるべき基本コマンドを把握する
- 各コマンドの使いどころを知る

## 前提知識
- [パーミッションモード入門](11_permission_modes_intro.md)

## 本文

### スラッシュコマンドとは

スラッシュコマンドとは、入力欄に `/`（スラッシュ）から始まるキーワードを入力することで、特定の機能を呼び出す仕組みです。

普通の会話文とは異なり、Claude Codeの操作や設定を直接制御できます。入力欄に `/` と入力すると、利用可能なコマンドの一覧が表示されるので、そこから選ぶこともできます。

### 初心者が覚えるべき10のコマンド

Claude Codeにはたくさんのスラッシュコマンドがありますが、最初に覚えるべきものを10個に厳選しました。

---

**1. /help — 困ったらまずこれ**

```
/help
```

使い方の一覧やヘルプ情報が表示されます。「何ができるんだっけ？」と思ったら、まずこのコマンドを使いましょう。

---

**2. /clear — 会話をリセット**

```
/clear
```

現在の会話履歴をすべてクリアし、新しいセッションを開始します。話題を変えたいときに使います。`/reset` や `/new` でも同じ動作です。

---

**3. /resume — 過去のセッションを再開**

```
/resume
```

過去のセッション一覧が表示され、選択したセッションの続きから作業を再開できます。`/continue` でも同じ動作です。

---

**4. /compact — 会話を圧縮する**

```
/compact
```

長くなった会話を要約して圧縮します。会話が長くなると処理が遅くなったり、トークン（利用量）を多く消費したりするため、こまめに圧縮するのがおすすめです。

特定の内容を残したい場合は、引数をつけることもできます。

```
/compact 提案書の内容は残して圧縮してください
```

---

**5. /rewind — 巻き戻し**

```
/rewind
```

会話やファイルの変更を、好きなポイントまで巻き戻します。「さっきの変更をなかったことにしたい」というときに使います。Escキーを2回押しても同じ機能を呼び出せます。

---

**6. /plan — 計画モードで作業を始める**

```
/plan 報告書のテンプレートを作成する
```

ファイルを変更せず、まず作業計画を立ててもらいます。重要な作業の前に「どんな手順で進めるか」を確認したいときに使います。

---

**7. /cost — トークン使用量を確認**

```
/cost
```

現在のセッションでどれだけトークン（利用量）を消費したかを表示します。利用量が気になるときにチェックしましょう。

---

**8. /model — AIモデルを変更**

```
/model
```

使用するAIモデルを切り替えます。通常はデフォルトのままで問題ありませんが、より高性能なモデルを使いたいときや、軽量なモデルで十分なときに切り替えられます。

---

**9. /diff — 変更箇所を確認**

```
/diff
```

Claudeが行ったファイルの変更箇所を、見やすい形式で表示します。「何が変わったの？」と確認したいときに使います。

---

**10. /copy — 回答をコピー**

```
/copy
```

Claudeの最新の回答をクリップボードにコピーします。回答をメールやドキュメントに貼り付けたいときに便利です。

数字をつけると、直近の回答から指定した数だけ遡ってコピーできます。

```
/copy 3
```

---

### コマンドの探し方

すべてのコマンドを覚える必要はありません。以下の方法でいつでもコマンドを探せます。

- **`/` を入力** ：利用可能なコマンドの一覧が表示される
- **`/help` を実行** ：ヘルプ情報が表示される
- **`/powerup` を実行** ：機能を学べるインタラクティブなレッスンが始まる

### よく使うコマンドの組み合わせ

実際の作業では、以下のような流れでコマンドを組み合わせることが多いです。

**新しい作業を始めるとき：**
1. `/clear` で前の会話をリセット
2. `/plan ○○を作成する` で作業計画を確認
3. 計画に問題なければ、実際の作業を指示

**作業を中断・再開するとき：**
1. 作業を中断（アプリを閉じてOK）
2. 次に開いたとき `/resume` で前のセッションを再開
3. 会話が長くなっていたら `/compact` で圧縮

**結果に不満があるとき：**
1. `/rewind` で巻き戻し
2. 別の指示で再度試す

## やってみよう（ハンズオン）

### ステップ1：コマンド一覧を表示してみましょう

Claude Codeの入力欄に `/` と入力してみてください（Enterは押さない）。

表示されるコマンドの一覧をざっと眺めて、どんなコマンドがあるか確認してみましょう。

### ステップ2：基本コマンドを実際に使ってみましょう

以下の手順を順番に試してみてください。

1. 何か質問をして回答をもらう

```
日本の祝日を3つ教えてください。
```

2. 回答をコピーする

```
/copy
```

3. メモ帳などに貼り付けて、正しくコピーされたか確認

4. トークン使用量を確認する

```
/cost
```

5. 会話をクリアする

```
/clear
```

### ステップ3：/compact を体験してみましょう

1. 長めの会話をしてみましょう（3〜5回やり取り）

```
日本の四季について教えてください。
```

```
特に秋の魅力を詳しく教えてください。
```

```
秋の旬の食べ物を10個リストアップしてください。
```

2. 会話を圧縮してみましょう

```
/compact
```

圧縮前後で、それまでの会話の文脈が保たれているか確認してみてください。

## まとめ
- スラッシュコマンドは `/` から始まるキーワードで機能を呼び出す仕組み
- まず覚えるべきは `/help`、`/clear`、`/resume`、`/compact`、`/rewind` の5つ
- `/` と入力するだけでコマンド一覧が見られるので、すべて暗記する必要はない
- 作業の流れに応じてコマンドを組み合わせると効率的
