## 本文

### この記事の位置づけ

この記事は、初級セクションの総まとめです。ここまで学んできた操作方法やコツを活用して、実際にビジネス文書を作成・編集してみましょう。

以下の3つの課題に取り組みます。

1. 会議の議事録を作成する
2. 提案書のたたき台を作る
3. 既存の文書を校正・改善する

### 課題1：会議の議事録を作成する

会議のメモから、正式な議事録を作成してもらいましょう。

**シナリオ：** あなたは営業部のメンバーで、先ほどの部内ミーティングのメモを取りました。このメモから正式な議事録を作成する必要があります。

**ステップ1：メモの内容を準備する**

以下の内容をClaude Codeに渡します。テキストをそのまま入力欄にコピー＆ペーストしてください。

```
以下の会議メモから、正式な議事録を作成してください。

【会議メモ】
日時：4月7日 10時〜11時
場所：第3会議室
参加者：部長の鈴木、課長の田中、山本、佐藤、自分（中村）

・今月の売上見込み → 田中さんから報告。目標の95%ペース。あと5%は新規案件で補う方針
・A社の大型案件について → 山本さんが4/10にプレゼン予定。資料は4/9までに部長チェック
・新しい営業ツールの導入 → 佐藤さんが来週デモを手配。全員参加してほしいとのこと
・次回ミーティング → 4/14 同じ時間

議事録の形式は以下にしてください：
- 「議事録」というタイトル
- 日時、場所、参加者の情報
- 議題ごとに「議題」「内容」「決定事項」「担当者・期限」を整理
- 最後に「次回予定」を記載
```

**ステップ2：結果を確認する**

Claudeが作成した議事録を確認します。修正が必要な箇所があれば、追加の指示を出しましょう。

```
ありがとうございます。以下の点を修正してください。
- 参加者の表記を「鈴木部長」のように役職付きにしてください
- A社の案件について、「プレゼン資料はパワーポイントで20枚以内」という条件を追記してください
```

**ステップ3：ファイルとして保存する**

満足のいく内容になったら、ファイルとして保存します。

```
この議事録を「議事録_20260407_営業部ミーティング.txt」としてデスクトップに保存してください。
```

### 課題2：提案書のたたき台を作る

新しいプロジェクトの提案書のたたき台を作成してもらいましょう。

**シナリオ：** 社内のペーパーレス化プロジェクトについて、上司に提出する提案書を作成します。

**ステップ1：計画を立てる**

まずは計画モードで、どんな構成にするか相談しましょう。

```
/plan 社内のペーパーレス化を推進する提案書のたたき台を作成したいです。上司（部長）に提出する社内向けの提案書です。A4で2〜3枚程度のボリュームを想定しています。どのような構成にすべきか提案してください。
```

**ステップ2：提案書を作成する**

構成に問題がなければ、実際に作成を指示します。

```
その構成で提案書を作成してください。以下の情報を盛り込んでください。

- 現状の課題：月間の紙の使用量が約5,000枚、印刷コストが月15万円
- 提案内容：クラウドストレージの導入、ペーパーレス会議の推進、電子承認の導入
- 期待効果：紙の使用量を70%削減、年間コスト120万円の削減
- スケジュール：3ヶ月で段階的に導入
- 必要予算：初期費用50万円、月額ランニングコスト3万円
```

**ステップ3：改善を重ねる**

出力された提案書に対して、改善の指示を出しましょう。

```
以下の点を改善してください。
1. 冒頭にエグゼクティブサマリー（要約）を100字以内で追加
2. コスト比較の部分を表形式に変更
3. 想定されるリスクと対策のセクションを追加
```

### 課題3：既存の文書を校正・改善する

ここでは、ファイルを渡して校正・改善してもらう練習をします。

**ステップ1：校正用のテスト文書を作成する**

以下の内容を「テスト文書.txt」として保存しておきます（わざと問題のある文章にしてあります）。

```
お客様各位

平素は格別のお引き立てを頂き、誠にありがとうございます。

さて、このたび弊社では、新しいサービスをリリースする事になりました。
このサービスは、お客様の業務効率を大幅に改善するものでして、
多くのお客様にご利用頂ければと考えております。

つきましては、下記の通りご案内申し上げます。

サービス名：ビジネスアシストPro
開始日：2026年5月1日
料金：月額10,000円（税別）
対象：法人のお客様

ご不明な点がございましたら、お気軽にお問い合わせ下さい。

以上
```

**ステップ2：文書をClaude Codeに渡して校正を依頼する**

「テスト文書.txt」をドラッグ＆ドロップするか、内容を直接入力して、以下の指示を送ります。

```
この文書を校正してください。以下の観点でチェックをお願いします。
1. 誤字脱字
2. 表記の統一（漢字/ひらがなの使い分け）
3. ビジネス文書としての適切さ
4. より良い表現への改善提案

修正箇所と修正理由を一覧で示した後、修正済みの完成版を出力してください。
```

**ステップ3：修正結果を確認してファイルに反映する**

修正内容に納得できたら、ファイルに保存しましょう。

```
修正済みの内容で「テスト文書_修正済み.txt」としてデスクトップに保存してください。
```

### 実践のポイント

これらの課題を通じて、以下のポイントを意識してみてください。

- **段階的に進める** ：一度にすべてを完璧にしようとせず、まずたたき台を作り、その後改善していく
- **具体的に指示する** ：「良い感じに」ではなく、何をどう変えてほしいか具体的に伝える
- **確認を怠らない** ：AIの出力は必ず人間の目で確認する。特に数字や固有名詞は要チェック
- **トークンを意識する** ：必要に応じて `/compact` で圧縮し、話題が変わったら `/clear` でリセット

## まとめ
- Claude Codeは議事録、提案書、文書校正など、さまざまなビジネス文書の作成に活用できる
- 「計画→作成→改善→保存」の流れで段階的に進めるのが効果的
- 具体的な指示と人間による確認が、質の高い文書を作るカギ
- ここまで学んだスラッシュコマンドやショートカットを活用して効率よく作業しましょう
