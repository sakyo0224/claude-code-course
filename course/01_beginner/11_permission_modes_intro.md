## 本文

### パーミッションモードとは

パーミッションモードは、Claude Codeがファイルの編集やコマンドの実行をする際に、「どこまで自動でやるか」を制御する仕組みです。

Claude Codeはパソコン上のファイルを直接操作できるため、意図しない変更が行われないように、「実行前に確認を求める」仕組みが用意されています。この確認の度合いを調整するのがパーミッションモードです。

自動車にたとえると、「マニュアル運転」から「完全自動運転」まで、自動化のレベルを自分で選べるようなイメージです。

### 5つのモード

Claude Codeには5つのパーミッションモードがあります。

**1. Ask permissions（確認モード）— 初心者におすすめ**

ファイルの編集やコマンドの実行のたびに、毎回確認を求められます。

- 「このファイルを編集してよいですか？」
- 「このコマンドを実行してよいですか？」

と表示されるので、内容を確認してから「許可する」か「拒否する」を選べます。初期設定ではこのモードになっています。

**2. Auto accept edits（編集自動承認モード）**

ファイルの編集は自動的に行われ、コマンドの実行時だけ確認を求められます。

文書の編集作業を多く行う方で、毎回の確認が煩わしいと感じたときに切り替えると便利です。

**3. Plan mode（計画モード）**

ファイルの読み取りと分析はできますが、ファイルの変更やコマンドの実行はまったく行いません。

「まずはどんな作業をするか計画だけ立ててほしい」というときに使います。実際の変更を行う前に、Claudeの計画を確認できるので安心です。

**4. Auto mode（自動モード）**

安全性チェックを行いながら、ほとんどの操作を自動的に実行します。

操作に慣れてきて、毎回の確認が不要だと感じたときに使います。ただし、破壊的な操作（重要なファイルの削除など）の前には確認が入ります。

**5. Bypass permissions（権限省略モード）**

すべての確認を省略し、完全に自動で実行します。

このモードは、テスト環境やサンドボックス（隔離された安全な環境）でのみ使うことが推奨されています。通常の作業では使わないでください。

### モードの比較表

| モード | ファイル編集 | コマンド実行 | おすすめの場面 |
|--------|------------|------------|--------------|
| Ask permissions | 確認あり | 確認あり | 初心者、重要なファイルを扱うとき |
| Auto accept edits | 自動 | 確認あり | 文書編集を多く行うとき |
| Plan mode | 不可 | 不可 | 作業計画を立てるとき |
| Auto mode | 自動 | 自動（安全チェック付き） | 操作に慣れてきたら |
| Bypass permissions | 自動 | 自動（チェックなし） | テスト環境のみ |

※ CI/CD向けのdontAskモードについては上級セクションで解説します。

### モードの切り替え方法

パーミッションモードは、以下の方法で切り替えられます。

**方法1：キーボードショートカット**

Shift＋Tab キーを押すと、モードの切り替えができます。

**方法2：設定画面から**

`/config` と入力して設定画面を開き、パーミッションモードの項目を変更します。

### 初心者へのおすすめ

初めてClaude Codeを使う方は、以下の使い方をおすすめします。

1. **まずは「Ask permissions（確認モード）」で始める** ：初期設定のままで、すべての操作を確認しながら進めましょう
2. **慣れてきたら「Auto accept edits」に切り替える** ：ファイル編集の確認が煩わしくなったら、このモードに切り替えましょう
3. **大事な作業の前は「Plan mode」を活用する** ：重要なファイルを扱うときは、まず計画モードでClaudeの作業計画を確認してから、実行に移しましょう

### 確認メッセージへの対応方法

確認モードでは、以下のような確認メッセージが表示されます。

```
Claude wants to edit file: /Users/tanaka/Documents/提案書.txt
Allow? [Yes/No/Always]
```

- **Yes** ：この1回だけ許可する
- **No** ：拒否する
- **Always** ：このタイプの操作を今後すべて許可する

迷ったときは「Yes」を選ぶのが安全です。「Always」は、同じ種類の操作を何度も行うことがわかっている場合に選びましょう。

## やってみよう（ハンズオン）

### ステップ1：現在のモードを確認しましょう

Claude Codeを開いて、画面のどこかに現在のパーミッションモードが表示されているか確認してみましょう。

### ステップ2：計画モードを試してみましょう

以下の指示を送って、計画モードを体験してみましょう。

```
/plan デスクトップに「自己紹介.txt」というファイルを作成してほしいのですが、どのような手順で進めますか？
```

Claudeが計画を提示してくれますが、実際にファイルは作成されません。計画モードの安心感を体感してください。

### ステップ3：確認モードでファイルを作成してみましょう

計画モードを抜けて、実際にファイルを作成してみましょう。

```
デスクトップに「テスト_パーミッション.txt」というファイルを作成してください。内容は「パーミッションモードのテストです」としてください。
```

確認メッセージが表示されたら、内容を読んでから「Yes」を選択しましょう。ファイルが作成されることを確認してください。

## まとめ
- パーミッションモードはClaude Codeの操作の自動化レベルを制御する仕組み
- 5つのモードがあり、初心者には「Ask permissions（確認モード）」がおすすめ
- Shift＋Tab で素早く切り替えられる
- 大事な作業の前は「Plan mode」で計画を確認すると安心
- 確認メッセージが出たら、内容をよく読んでから許可/拒否を選ぶ
