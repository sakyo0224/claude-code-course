# セッション管理の基本

- 会話履歴の管理方法を知る

## 本文

### セッションとは

Claude Codeでは、一連の会話のまとまりを「セッション」と呼びます。

たとえば、「提案書を作成する」という作業を始めてから完成させるまでの一連のやり取りが、1つのセッションです。日常会話にたとえると、1つの話題について話し始めてから終わるまでが1つのセッション、と考えていただければ結構です。

セッション内では、それまでの会話の内容をClaudeが覚えているので、「さっき作った文書の2段落目を修正して」のような、前の会話を踏まえた指示ができます。

### 新しいセッションを始める

新しい話題や作業を始めるときは、新しいセッションを開始するのがおすすめです。

**方法1：/clear コマンドを使う**

入力欄に以下のように入力してEnterキーを押します。

```
/clear
```

これで会話履歴がクリアされ、まっさらな状態で新しい会話を始められます。`/reset` や `/new` と入力しても同じ動作をします。

**方法2：アプリのメニューから**

デスクトップアプリのメニューから「New Session」（新しいセッション）を選択する方法もあります。

### 過去のセッションを再開する

以前の作業を途中から続けたいときは、過去のセッションを再開できます。

```
/resume
```

このコマンドを入力すると、過去のセッションの一覧が表示されます。続けたいセッションを選択すると、前回の会話の続きから作業を再開できます。`/continue` と入力しても同じ動作をします。

### セッションに名前をつける

セッションが増えてくると、どれがどの作業だったかわからなくなることがあります。そんなときは、セッションに名前をつけておきましょう。

```
/rename 提案書作成_A社向け
```

わかりやすい名前をつけておくと、後から探しやすくなります。

### セッションの分岐

作業の途中で「別のアプローチも試してみたい」と思ったとき、セッションを分岐させることができます。

```
/branch 別案
```

これは、ゲームの「セーブデータを複製する」ようなイメージです。元のセッションを残したまま、別の方向で作業を進められます。

### 会話の巻き戻し

「さっきの指示の結果がイマイチだったから、やり直したい」というとき、会話を巻き戻すことができます。

```
/rewind
```

このコマンドを使うと、会話やファイルの変更を好きなポイントまで巻き戻せます。ゲームのセーブデータをロードするような感覚で使えます。Escキーを2回押しても同じ画面に入れます。

巻き戻しには3つの選択肢があります。
- 「コードと会話の両方を復元」：ファイルの変更も会話も元に戻す
- 「会話のみ復元」：会話だけ巻き戻し、ファイルの変更はそのまま
- 「コードのみ復元」：ファイルだけ元に戻し、会話はそのまま

### 会話のエクスポート

セッションの内容を外部に保存したい場合は、エクスポート機能を使います。

```
/export
```

会話の内容がテキストファイルとして保存されます。議事録やメモとして残しておきたいときに便利です。

### セッション管理のコツ

- **1つの話題につき1セッション** ：異なる話題を同じセッションで続けると、Claudeが混乱することがあります。話題が変わったら新しいセッションを始めましょう
- **こまめに名前をつける** ：後で探しやすいように、作業内容がわかる名前をつけておきましょう
- **長くなったら圧縮する** ：会話が長くなると処理が遅くなることがあります。`/compact` コマンドで会話を圧縮できます（次の記事で詳しく解説します）

## やってみよう（ハンズオン）

### ステップ1：セッションを開始して名前をつけましょう

1. Claude Codeを開いて、以下の指示を送ります

```
今日の天気について教えてください。
```

2. 回答を確認したら、セッションに名前をつけます

```
/rename テスト_天気の会話
```

### ステップ2：新しいセッションを始めましょう

1. 会話をクリアします

```
/clear
```

2. 別の話題で会話を始めます

```
おすすめのビジネス書を3冊紹介してください。
```

3. このセッションにも名前をつけましょう

```
/rename テスト_おすすめ本
```

### ステップ3：前のセッションに戻ってみましょう

1. 過去のセッション一覧を開きます

```
/resume
```

2. 「テスト_天気の会話」を選択して、前の会話に戻れることを確認しましょう

3. 戻れたら、会話の続きとして以下を送ってみます

```
さっき教えてもらった天気の情報を、3行にまとめてください。
```

前の会話の内容を覚えているかどうか、確認してみましょう。

## まとめ
- セッションは「一連の会話のまとまり」のこと
- `/clear` で新しいセッションを開始、`/resume` で過去のセッションを再開
- `/rename` でわかりやすい名前をつけておくと後から探しやすい
- `/rewind` で会話やファイルの変更を巻き戻せる
- 1つの話題につき1セッションを心がけると効率的
