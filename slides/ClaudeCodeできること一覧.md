# Claude Code でできること — 能力カタログ

> このページは「Claude Code で何ができるか」を業務シーン別にまとめた一覧です。
> 気になる項目から読んでください。

## 難易度の目安

| マーク | 意味 |
|--------|------|
| ★☆☆ | 入力と一言の指示だけで完了。プログラミング不要 |
| ★★☆ | 設定ファイルへの記入が必要。難しくはない |
| ★★★ | 仕組みの理解が必要。IT担当者と一緒に設定 |

---

## 1. 文書・テキスト処理

### 文章の要約・整理 ★☆☆
長い文書を短くまとめる。要点を箇条書きにする。

**具体例：**
- 10ページの会議録 → 1ページの要点まとめ
- 取引先から届いた長い提案書 → 3点の重要ポイント
- ウェブ記事の内容 → 社内向け要約メモ

**どう使うか：**
```
「このファイルを読んで、3点に要約してください」
```

---

### 文章の品質改善 ★☆☆
誤字脱字の修正、文体の統一、敬語への変換。

**具体例：**
- 部下が書いた報告書 → 役員向けの丁寧な文体に
- 社内向けに書いた文書 → 顧客向けの表現に変換
- 箇条書きのメモ → 読みやすいビジネス文書に整形

**どう使うか：**
```
「この文章を、取引先に送れる丁寧な敬語に書き直してください」
```

---

### 翻訳 ★☆☆
英語 ↔ 日本語の翻訳。ニュアンスを保った自然な表現。

**具体例：**
- 英語の契約書の概要を日本語で把握
- 日本語のプレスリリースを英語に翻訳
- 英語の製品マニュアルを日本語の手順書に変換

---

### テンプレートからの文書生成 ★☆☆
雛形と入力データを渡すだけで、完成した文書を生成。

**具体例：**
- 見積書テンプレート ＋ 品目リスト → 完成した見積書
- 報告書テンプレート ＋ 今月のデータ → 月次報告書
- 提案書の骨子 → 文章を肉付けした完成版

---

### 複数ファイルの一括編集 ★★☆
フォルダ内の複数ファイルに、同じルールで変更を適用。

**具体例：**
- 100本の商品説明文の語尾を一括で統一
- フォルダ内の全議事録に「承認済み」スタンプを追記
- 複数の報告書ファイルの書式を統一

**時間削減：** 100ファイル × 5分 → 一括実行で数分

---

## 2. データ処理・分析

### CSVデータの集計・分析 ★☆☆
Excelのような集計を、指示を出すだけで実行。

**具体例：**
- 全国5店舗の売上CSV → 店舗別・月別の集計表
- 在庫データ → 発注が必要な品目リスト
- アンケート結果CSV → 回答傾向の分析サマリー

**どう使うか：**
```
「このCSVを読んで、店舗別の売上トップ5を出してください」
```

---

### レポートの自動生成 ★☆☆
データを渡すだけで、グラフ（テキスト表）と考察つきのレポートを生成。

**具体例：**
- 月次売上データ → 前月比・前年比つきの月次レポート
- 顧客アンケート結果 → 課題と改善提案つきのレポート
- 経費データ → 部門別・カテゴリ別の経費レポート

**時間削減：** 毎月4〜5時間の作業 → 数十分に短縮

---

### PDFからの情報抽出 ★☆☆
複数のPDFから必要な項目だけを取り出して一覧表に。

**具体例：**
- 毎月届く請求書（PDF）50件 → 取引先名・金額・期日の一覧表
- 契約書PDFの束 → 契約金額・期間・更新日の抽出
- 会議資料（PDF）10冊 → 決定事項だけを集めた一覧

**時間削減：** 月8時間の手入力作業 → ほぼゼロに

---

### 比較表の作成 ★☆☆
複数の選択肢を比べた表を自動生成。

**具体例：**
- 5つのクラウドサービスの料金・機能比較表
- 候補3社の見積もり比較表（優劣コメントつき）
- 新旧の規程・契約内容の変更点一覧

---

## 3. コミュニケーション支援

### 個別メールの一括生成 ★☆☆
顧客リストを渡すと、1件ずつ内容を変えたメールを大量生成。

**具体例：**
- 50社への案内メール（業種・前回の接点に合わせて文面を変える）
- 未払い請求のリマインドメール（金額・期日を個別に記載）
- 顧客向け年賀状・挨拶メールの一括作成

**時間削減：** 50通 × 6分 = 5時間 → 一括生成で数分

---

### 議事録の自動作成 ★☆☆
会議メモや録音の文字起こしを渡すと、整形された議事録に。

**具体例：**
- 箇条書きの会議メモ → 決定事項・アクションアイテムつきの議事録
- 会議の文字起こし → 参加者・議題・決定事項が整理された形式
- 複数回の会議メモ → プロジェクト経緯をまとめた1本のドキュメント

---

### 企画書・提案書の作成 ★☆☆
キーワードやアイデアの断片から、構成された文書の骨子を生成。

**具体例：**
- 「新製品のターゲット・特徴・価格帯」を渡す → 企画書の骨子
- アイデアの箇条書き → 論理的に整理されたプレゼン構成
- 上司からの要件メモ → 社内稟議書の下書き

---

## 4. 調査・リサーチ

### 競合調査と比較分析 ★☆☆
集めた情報を整理・比較し、考察つきのレポートを作成。

**具体例：**
- 5社のWebサイトから収集した情報 → 機能・料金・評判の比較レポート
- 競合他社のプレスリリース → トレンド分析と自社への影響考察
- 市場調査データ → 機会と脅威の整理（SWOT分析）

**時間削減：** 丸1日かかる調査・資料作成 → 半日以下に

---

### 情報の収集・整理 ★☆☆
散在した情報をまとめて、意思決定しやすい形に整理。

**具体例：**
- 各部門から届いたバラバラな報告 → 一元管理された報告書
- 過去の会議資料 → プロジェクトの経緯サマリー
- 各種アンケート回答 → 傾向と課題の整理

---

## 5. 自動化・効率化

### ルールの恒久設定（CLAUDE.md） ★★☆
一度書けば毎回説明不要。Claude Code がずっと守り続ける。

**具体例：**
- 「常に敬語で、1000字以内で書く」
- 「うちの会社は製造業で、製品はXXXです」
- 「ファイルを作るときは必ずバックアップを取る」

**効果：** 毎回同じ前置きを話す時間がゼロになる

---

### 作業完了時の自動処理（Hooks） ★★★
ファイルを保存した瞬間・作業が終わった直後に自動でスクリプトが走る。

**具体例：**
- 報告書を保存 → 自動で上司にメール通知
- ファイルを編集 → 自動でバックアップを作成
- 処理完了 → 自動でSlackに結果を投稿

---

### 定期タスクの自動実行（スケジュール） ★★★
パソコンが起動していなくても、決まった時間に自動で動く。

**具体例：**
- 毎朝9時：業界ニュースを収集→要約→メールで受信
- 毎週月曜：先週の数値を集計してレポート生成
- 毎月1日：月次のデータバックアップ

---

## 6. 外部サービス連携（MCP）

### Slack 連携 ★★★
Claude Code から Slack に自動投稿・通知。

**具体例：**
- 処理完了時に指定チャンネルに結果を自動投稿
- 毎日の業務サマリーを朝イチで自動送信
- エラー発生時に担当者にアラート

---

### Notion 連携 ★★★
作成した文書やデータを Notion のページに自動保存。

**具体例：**
- 議事録を自動生成して Notion のデータベースに保存
- 分析レポートを Notion のページとして作成
- タスクリストを Notion に自動追加

---

### Google Calendar 連携 ★★★
スケジュールの確認や登録を Claude Code 経由で行う。

**具体例：**
- 「今週の空き時間を確認して、打ち合わせ候補を3つ提案」
- 会議の議事録に記載されたアクション期限を自動でカレンダーに登録

---

## 7. デスクトップ操作の自動化（Computer Use）

### GUI アプリの自動操作 ★★★
Claude Code が画面を認識して、マウス・キーボードを代わりに操作。

**具体例：**
- レガシーシステムへのデータ入力作業の自動化
- Webブラウザを操作して情報を収集・フォームを送信
- 複数アプリをまたいだ定型的なコピペ作業

**注意：** 操作前に必ず確認を挟む設定を推奨

---

## 8. チーム・組織での活用

### ルールの共有と統一品質 ★★☆
CLAUDE.md をチームで共有すれば、全員が同じ基準で作業できる。

**具体例：**
- 部門全体の文書スタイルを統一（フォント・表記ルール）
- 新人が入っても即日から同じルールで動く
- 外部委託先とのやり取りルールを共有

---

### 複数エージェントの並列作業 ★★★
複数の Claude Code が役割分担して同時進行。

**具体例：**
- 「調査係」「分析係」「執筆係」が同時に動いて、1時間のレポート作業を20分に
- 大量ファイルを複数エージェントが分割処理

---

### 監査ログとコンプライアンス対応 ★★★
何をいつ・どのファイルに実行したか、すべて記録に残す。

**具体例：**
- 個人情報ファイルへのアクセス履歴を自動記録
- 承認なしでの特定フォルダ操作をブロック
- 変更履歴を監査レポートに自動まとめ

---

## まとめ：Claude Code のポジション

```
┌─────────────────────────────────────────────┐
│                 Claude Code                  │
│                                              │
│  ✅ ファイルを直接読み書きできる            │
│  ✅ 複数ファイルを一括処理できる            │
│  ✅ ルールを覚えて毎回適用できる            │
│  ✅ 外部サービスと連携できる                │
│  ✅ 定期的に自動実行できる                  │
│  ✅ チームで設定を共有できる                │
│                                              │
│  ❌ インターネット上の情報を自律収集 (一部) │
│  ❌ リアルタイムデータの取得 (設定が必要)   │
└─────────────────────────────────────────────┘
```

**一言でいうと：**
> 「決まったパターンの業務を、ファイル操作を含めて丸ごと任せられるAIアシスタント」
