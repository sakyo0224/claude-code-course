# セミナー発表資料 設計書（v2）

## この設計書の使い方

- 発表資料の全体像を俯瞰する
- セクションの順番・取捨選択を検討する際の判断材料にする
- プロンプト例・スクリーンショット・演習の過不足を確認する
- 成果物（資料・スライド・台本）の整合性をチェックする

---

## 全体構造

```
オープニング → 知識 → 準備 → 安心装備 → 体験① → 体験② → 体験③ → CLAUDE.md → まとめ → 質疑
```

**参加者の状態変化：**

| 段階 | 状態 | 判定基準 |
|------|------|---------|
| オープニング後 | 「何が得られるか」がわかる | 今日のゴール3つを言える |
| 知識パート後 | 「なるほど」 | チャット型とエージェント型の違いを一文で説明できる |
| 準備後 | 「準備できた」 | アプリが起動し、データ学習オフが完了している |
| 安心装備後 | 「失敗しても大丈夫」 | /plan と /rewind の存在を知っている |
| 体験①後 | 「動かせた！」 | Claudeに話しかけて返事を受け取れた |
| 体験②後 | 「ファイルも扱える！」 | ファイルを渡して議事録を作れた |
| 体験③後 | 「自分の仕事で使える！」 | 自分の業務テーマで1つ試せた |
| CLAUDE.md後 | 「もっと便利にできる」 | 引き継ぎ資料の比喩で仕組みを理解した |
| まとめ後 | 「明日からやろう」 | 3ステップが明確 |

## 時間配分

| 種類 | 時間 | 割合 |
|------|------|------|
| 講義 | 16分 | 25% |
| デモ | 15分 | 23% |
| 演習 | 19分 | 29% |
| 質疑・バッファ | 15分 | 23% |
| **合計** | **65分** | **100%** |

**設計方針：** 「聞くだけ」が10分以上続かないよう、デモ・演習・問いかけを挟む。

## コンテンツ要素の分類と優先度

### 知識ブロック（読んで理解するもの）

| 要素 | 優先度 | 理由 |
|------|--------|------|
| チャット型 vs エージェント型 | **必須** | 核心。Claude Codeの価値を理解する前提 |
| デスクトップアプリの特徴 | **必須** | 今日使うツールの説明 |
| Claude Cowork との違い | 補足 | 知らなくても使える。質疑で聞かれたら答える程度 |
| 3つのプラットフォーム比較 | 削除 | 「今日使うのはこれ」の一言で十分 |

### 手順ブロック（見て実行するもの）

| 要素 | 優先度 |
|------|--------|
| インストール（3 STEP） | **必須** |
| データ学習オフ（2 STEP） | **必須** |
| /plan の使い方 | **必須**（体験の前に） |
| /rewind の使い方 | **必須**（体験の前に） |

### 体験ブロック（自分で試すもの）

| 演習 | 時間 | 種類 | 難易度 |
|------|------|------|--------|
| ①はじめの一歩 | 4分 | そのまま入力 | 非常に簡単 |
| ②ファイルを渡す | 5分 | そのまま入力＋ファイル操作 | 簡単 |
| ③自分の仕事で試す | 5分 | 穴埋め＋自由入力 | 普通 |

### 行動ブロック（次の一歩を示すもの）

| 要素 | 優先度 |
|------|--------|
| 明日からの3ステップ | **必須** |
| CLAUDE.md の紹介 | **必須**（独立セクションで） |

## 情報の依存関係

```
AI・Claudeの定義
  └─ チャット型 vs エージェント型 の理解
       └─ 「今日使うのはClaude Codeデスクトップアプリ」
            ├─ インストール → データ学習オフ（準備完了）
            │    ├─ /plan, /rewind を知る（★安心した状態で体験へ）
            │    │    ├─ 演習①：話しかける（初回体験）
            │    │    │    └─ 演習②：ファイルを渡す（エージェント体験）
            │    │    │         └─ 演習③：自分の仕事で試す（実践）
            │    │    │              └─ 共有タイム
            │    │    └─ [省略可能] Claude Cowork の説明
            │    └─ [省略可能] 3プラットフォーム比較
            └─ CLAUDE.md（体験後に「もっと便利に」として紹介）
                 └─ 明日からの3ステップ（行動促進）
```

凡例：`[省略可能]` = 時間が足りなければカットしても本筋に影響しない

## 旧構造からの変更履歴

| 課題 | 旧 | 新 | 効果 |
|------|-----|-----|------|
| 情報過多 | PART1に比較表3つ | 比較表1つに圧縮 | 触る前の知識を最小限に |
| 安心ブロックの位置 | PART5（最後） | セクション4（体験の前） | 安心した状態で手を動かせる |
| CLAUDE.mdが唐突 | まとめに突然登場 | 独立セクション（体験の後） | 「もっと便利にする方法」として自然に導入 |
| 演習の少なさ | 2回（PART4のみ） | 3回＋共有タイム | 段階的に難易度を上げて体験を積み上げ |
| 重複 | 本文と付録で/plan,/rewindが重複 | セクション4で集中的に扱い、付録は早見表のみ | 冗長さを解消 |
| Coworkの不正確な記述 | 「全部できます」 | 「用途が異なるツール」 | 事実に即した表現に修正 |
