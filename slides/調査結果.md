### 確認済みの事実

#### 1. Claude Code デスクトップアプリ版の機能と特徴

- （信頼度：高）Claude Code はデスクトップアプリ版・ターミナル版・IDE統合版・ブラウザ版の4形態で提供されている
- （信頼度：高）デスクトップアプリ版では、PCのローカルファイルへの直接アクセス（読み書き・保存）が可能
- （信頼度：高）デスクトップアプリ版では複数のセッションを並行実行できる（例：1つ目がバグ修正、2つ目がリサーチ、3つ目がドキュメント更新）
- （信頼度：高）2026年3月にコンピュータ操作機能（画面上のファイルを開く、クリック、ナビゲーション）がCoworkとClaude Codeに追加された
- （信頼度：高）2026年4月3日にWindows対応が正式に追加された（Pro・Maxプラン向け）
- （信頼度：高）デスクトップアプリはmacOSとWindowsに対応（Linuxは未対応）

**ブラウザ版（claude.ai）との具体的な違い：**
- （信頼度：高）ブラウザ版にはファイルシステムへの直接アクセス機能がない。ファイルは.zipでアップロードする必要がある
- （信頼度：高）デスクトップアプリはMCPサーバー接続に対応しており、ローカルファイル・データベース・外部ツールと連携可能
- （信頼度：高）デスクトップアプリにはCowork（自律エージェントモード）が搭載されている
- （信頼度：高）デスクトップアプリはキーボードショートカットでどのアプリからでも呼び出せる
- （信頼度：高）デスクトップアプリではドラッグ＆ドロップでファイルを渡せる

**料金プラン：**
- （信頼度：高）Proプラン：月額$20（年払いの場合は月額$17）
- （信頼度：高）Proプランに含まれるもの：Claude Code（ターミナル版含む）、ファイル作成・コード実行、無制限プロジェクト、Google Workspace連携、リモートMCPコネクタ、拡張推論モデルへのアクセス
- （信頼度：高）Proプランの利用制限：5時間ウィンドウあたり約45メッセージ（メッセージの長さや複雑さにより変動）
- （信頼度：高）Max 5xプラン：月額$100（Proの5倍の利用量）、Max 20xプラン：月額$200（Proの20倍の利用量）
- （信頼度：高）Proプランでアクセスできるモデル：Opus 4.5、Sonnet 4.5、Haiku 4.5、Sonnet 4.6

#### 2. 主要コマンド

- （信頼度：高）**`/plan`** — Claude を読み取り専用モードにする。ファイルの読み取りとコードベースの分析はできるが、変更はできない。提案される変更はすべて「計画」として提示され、明示的な承認が必要
- （信頼度：高）**`/rewind`** — 会話やコードの状態を任意のチェックポイントまで巻き戻す。Escキーを2回押してもリワインドモードに入れる。3つの選択肢がある：「コードと会話の両方を復元」「会話のみ復元」「コードのみ復元」
- （信頼度：高）**`/clear`** — チャット履歴とコンテキストウィンドウを完全にクリアする。新しいセッションを開始するのと同じ効果
- （信頼度：高）**`/help`** — 使い方の一覧を表示する。`/` と入力するだけでも利用可能なコマンド一覧が表示される

**初心者に必要なコマンドについて：**
- （信頼度：高）上記4つのコマンドは初心者が知るべき最も基本的なコマンドとして十分。他にも `/compact`（コンテキスト圧縮）、`/model`（モデル切替）、`/cost`（コスト確認）などがあるが、初回セミナーでは上記4つで十分

#### 3. CLAUDE.md の仕組み

- （信頼度：高）CLAUDE.md はClaude起動時に最初に読み込まれ、システムプロンプトの一部として会話全体を通じて保持される
- （信頼度：高）階層構造で管理される：
  - **グローバル：** `~/.claude/CLAUDE.md` — すべてのプロジェクトに共通で適用される個人設定
  - **プロジェクトルート：** プロジェクトのルートディレクトリに配置する `CLAUDE.md` — 最も一般的な使い方
  - **サブディレクトリ：** フォルダごとに個別のルールを設定可能
- （信頼度：高）現在のディレクトリに近いファイルほど優先度が高い（後に読み込まれるため、モデルがより注意を払う）
- （信頼度：高）典型的な記載内容：プロジェクト構成、ビルド/テストコマンド、コーディング規約、避けるべきこと、デプロイ設定、チームの好み（PR形式、コミットメッセージのスタイルなど）

#### 4. データプライバシー設定

- （信頼度：高）設定場所：Privacy Settings（プライバシー設定画面）から変更可能
- （信頼度：高）「Allow Anthropic to use my data for model training」のトグルで、データを学習に使うかどうかを選択できる。いつでも変更可能
- （信頼度：高）**オンにした場合：** データ保持期間が5年に延長され、新しいClaudeモデルの学習に使用される。Free、Pro、Maxアカウントが対象（Claude Code利用時のデータも含む）
- （信頼度：高）**オフにした場合：** データ保持期間は従来通り30日。学習には使用されない
- （信頼度：高）商用プラン（Team、Enterprise、API、Amazon Bedrock、Google Cloud Vertex API）にはこの設定は適用されない（そもそも学習に使われない）
- （信頼度：高）設定変更後やアカウント削除後は、今後のモデル学習からデータが除外される
- （信頼度：高）Anthropicは機密データのフィルタリングや匿名化の自動処理を行っており、第三者にデータを販売することはない

#### 5. Claude Cowork との違い

- （信頼度：高）**Cowork の現在のステータス：リサーチプレビュー（試験運用中）。** 2026年1月23日にProプラン向けに公開。Team・Enterpriseプランでも利用可能。正式リリース（GA）の時期は未発表
- （信頼度：高）Anthropicは「機密データを扱うアプリケーションと併用しないこと」を推奨している
- （信頼度：高）**対象ユーザーの違い：** Cowork = 非開発者向けの業務ワークフロー自動化（マーケティング、営業、財務、法務、PM）、Claude Code = 汎用AIアシスタント（開発だけでなく幅広い業務に対応）
- （信頼度：高）**動作の違い：** Cowork はデスクトップアプリ内で動作し、画面上のアプリを操作（クリック、入力など）する。Claude Codeはファイルの読み書きとコマンド実行で作業する
- （信頼度：高）**セキュリティ：** Coworkは仮想マシン内で隔離されて動作するため安全性が高い。Claude Codeはより開放的
- （信頼度：高）**トークン効率：** Coworkはスクリーンショットや画像処理などの裏方作業があるため、利用枠の消費が早い
- （信頼度：高）**費用：** どちらも有料プラン（Pro以上）に含まれており、追加費用はなし
- （信頼度：高）Coworkには11の業務別プラグイン（マーケティング、営業、財務、法務、PM等）がある

### 要確認の情報

- ⚠️ **セミナー記事の料金表記「月額$20（約3,000円）」について：** $20は正しいが、2026年4月時点の為替レートでは約3,000円よりも高い可能性がある（為替変動による）。年払い（月額$17）の選択肢も言及すべき。（理由：為替レートは変動するため、発表時の最新レートを確認する必要がある）

- ⚠️ **セミナー記事の「Claude Coworkでできることは Claude Code でも全部できます」について：** これは不正確な可能性が高い。Coworkは画面操作（クリック、ナビゲーション）ができ、11の業務別プラグインがある。Claude Codeはファイル操作とコマンド実行が中心。正確には「用途が異なる」と表現すべき。（理由：公式情報では両者は対象ユーザーも機能も異なるツールとして位置づけられている）

- ⚠️ **セミナー記事の「Cowork：ファイルアクセスに範囲に制限あり」について：** Coworkは仮想マシン内で隔離されて動作するため「制限あり」は概ね正しいが、最近のアップデートでデスクトップ全体の操作が可能になっている。表現を見直す余地がある。（理由：2026年3-4月のアップデートで機能が大幅に拡充されている）

- ⚠️ **セミナー記事のプライバシー設定URL `claude.ai/settings/data-privacy-controls` について：** 公式プライバシーセンターは `privacy.claude.com` だが、設定画面のURLがこのパスで正しいかは実際にアクセスして確認が必要。（理由：URLの正確性は発表で非常に重要）

- ⚠️ **セミナー記事の3つのプラットフォームの表で「ブラウザ版：AIエージェント △ 限定的」について：** ブラウザ版のclaude.aiでもある程度のエージェント的機能（Artifacts、Project機能など）が使えるため、「限定的」の表現が適切かは検討の余地がある。（理由：ブラウザ版も機能拡充が進んでいる）

- ⚠️ **`/plan` コマンドの挙動について：** セミナー記事では「作業前に手順を提案してくれる」と記載されているが、正確には「読み取り専用モードにして、変更を計画として提示し、承認を求める」動作。単に手順を提案するだけでなく、ファイル変更の実行を制限する機能がある点を正確に伝えるべき。（理由：初心者向けの説明としてはセミナー記事の表現でも十分だが、正確性を期すならば補足が望ましい）

- ⚠️ **ダウンロードURL `claude.com/download` について：** 検索結果でも確認されたURLだが、実際にアクセスして現在もこのURLが有効か、当日までに確認が必要。（理由：リダイレクト先が変わっている可能性がある）
