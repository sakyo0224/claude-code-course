# /token-saver

コンテキスト肥大化を防ぎ、トークン消費を最小化する。

## モード判定

引数またはユーザー発言から判定。

| モード | トリガー |
|--------|---------|
| `handoff` | 「リセット」「引き継ぎ」「まとめて」 |
| `audit` | 「監査」「見直し」「CLAUDE.md」 |

判定不能時は使い方を1行で案内: 「`/token-saver handoff` で引き継ぎ、`/token-saver audit` でCLAUDE.md監査ができます」

---

## handoff — セッション引き継ぎ

会話コンテキストを要約し、次セッションに貼れる引き継ぎドキュメントを即座に生成する。

出力フォーマット:

```
## セッション引き継ぎ（{日付}）
### 完了済み
### 進行中・次にやること
### 決定事項・前提
### 関連ファイル
### 注意点
```

生成後「修正があれば指摘してください」と1回だけ確認。

---

## audit — CLAUDE.md 監査

グローバルとプロジェクトの CLAUDE.md を読み込み、トークン効率の観点で診断する。AIが自明に判断できる内容・冗長な記述・重複を特定し、結果と修正案をまとめて1回で提示する。

CLAUDE.md が存在しない場合は「見つかりません」と報告して終了。

### 出力

```
## CLAUDE.md 監査結果
- 現在: 約XXXトークン → 効率化後: 約XXXトークン（XX%削減）
- 効率化箇所:
  1. [行XX-XX] 内容 → 理由
  2. ...

## 提案する変更（diff）
[効率化前 → 効率化後]
```

出力後「この変更を適用しますか？（対象番号 / all / skip）」と1回確認。承認後に編集実行。
